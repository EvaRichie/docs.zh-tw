<span data-ttu-id="a681b-106">某些類型會被視為 *基本類型*，例如各種大小的布林型別 `bool` 和整數和浮點數類型，包括位元組和字元的類型。</span><span class="sxs-lookup"><span data-stu-id="a681b-106">Some types are considered *primitive types*, such as the Boolean type `bool` and integral and floating point types of various sizes, which include types for bytes and characters.</span></span> <span data-ttu-id="a681b-107">這些類型會在 [基本類型](basic-types.md)中描述。</span><span class="sxs-lookup"><span data-stu-id="a681b-107">These types are described in [Primitive Types](basic-types.md).</span></span>

<span data-ttu-id="a681b-112">函數的類型包含參數類型和傳回類型的相關資訊。</span><span class="sxs-lookup"><span data-stu-id="a681b-112">The type of a function includes information about the parameter types and return type.</span></span>

<span data-ttu-id="a681b-113">.NET Framework 是物件類型、介面類別型、委派類型和其他專案的來源。</span><span class="sxs-lookup"><span data-stu-id="a681b-113">The .NET Framework is the source of object types, interface types, delegate types, and others.</span></span> <span data-ttu-id="a681b-114">您可以定義自己的物件類型，就像在任何其他 .NET 語言中一樣。</span><span class="sxs-lookup"><span data-stu-id="a681b-114">You can define your own object types just as you can in any other .NET language.</span></span>

<span data-ttu-id="a681b-115">此外，F # 程式碼可以定義別名（名稱為「 *類型縮寫*」），這是類型的替代名稱。</span><span class="sxs-lookup"><span data-stu-id="a681b-115">Also, F# code can define aliases, which are named *type abbreviations*, that are alternative names for types.</span></span> <span data-ttu-id="a681b-116">當類型未來可能會變更，而且您想要避免變更相依于型別的程式碼時，您可以使用類型縮寫。</span><span class="sxs-lookup"><span data-stu-id="a681b-116">You might use type abbreviations when the type might change in the future and you want to avoid changing the code that depends on the type.</span></span> <span data-ttu-id="a681b-117">或者，您可以使用類型縮寫做為類型的易記名稱，使程式碼更容易閱讀和瞭解。</span><span class="sxs-lookup"><span data-stu-id="a681b-117">Or, you might use a type abbreviation as a friendly name for a type that can make code easier to read and understand.</span></span>
