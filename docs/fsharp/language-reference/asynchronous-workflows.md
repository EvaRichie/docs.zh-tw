<span data-ttu-id="4dbbc-104">本文描述 F # 中的支援以非同步方式執行計算，也就是不會封鎖其他工作的執行。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-104">This article describes support in F# for performing computations asynchronously, that is, without blocking execution of other work.</span></span> <span data-ttu-id="4dbbc-105">例如，您可以使用非同步計算來撰寫應用程式，這些應用程式的 Ui 會在應用程式執行其他工作時，仍能回應使用者。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-105">For example, asynchronous computations can be used to write applications that have UIs that remain responsive to users as the application performs other work.</span></span>

<span data-ttu-id="4dbbc-118">在非同步工作流程中，某些運算式和作業是同步的，有些則是較長的計算，其設計目的是要以非同步方式傳回結果。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-118">In an asynchronous workflow, some expressions and operations are synchronous, and some are longer computations that are designed to return a result asynchronously.</span></span> <span data-ttu-id="4dbbc-119">當您以非同步方式呼叫方法，而不是使用一般系結時 `let` ，就會使用 `let!` 。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-119">When you call a method asynchronously, instead of an ordinary `let` binding, you use `let!`.</span></span> <span data-ttu-id="4dbbc-120">的作用 `let!` 是在執行計算時，讓執行在其他計算或執行緒上繼續執行。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-120">The effect of `let!` is to enable execution to continue on other computations or threads as the computation is being performed.</span></span> <span data-ttu-id="4dbbc-121">在系結的右側傳回之後 `let!` ，其餘的非同步工作流程會繼續執行。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-121">After the right side of the `let!` binding returns, the rest of the asynchronous workflow resumes execution.</span></span>

<span data-ttu-id="4dbbc-122">下列程式碼顯示與之間的 `let` 差異 `let!` 。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-122">The following code shows the difference between `let` and `let!`.</span></span> <span data-ttu-id="4dbbc-123">使用的程式程式碼 `let` 只會建立異步計算做為物件，您稍後可以使用來執行，例如 `Async.StartImmediate` 或 [`Async.RunSynchronously`](https://fsharp.github.io/fsharp-core-docs/reference/fsharp-control-fsharpasync.html#RunSynchronously) 。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-123">The line of code that uses `let` just creates an asynchronous computation as an object that you can run later by using, for example, `Async.StartImmediate` or [`Async.RunSynchronously`](https://fsharp.github.io/fsharp-core-docs/reference/fsharp-control-fsharpasync.html#RunSynchronously).</span></span> <span data-ttu-id="4dbbc-124">使用的程式程式碼 `let!` 會開始計算，然後執行緒會暫停，直到結果可用為止，此時會繼續執行。</span><span class="sxs-lookup"><span data-stu-id="4dbbc-124">The line of code that uses `let!` starts the computation, and then the thread is suspended until the result is available, at which point execution continues.</span></span>

