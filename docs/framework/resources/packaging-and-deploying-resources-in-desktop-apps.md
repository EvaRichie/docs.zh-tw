## <a name="the-resource-fallback-process"></a><span data-ttu-id="49148-130">資源後援處理序</span><span class="sxs-lookup"><span data-stu-id="49148-130">The Resource Fallback Process</span></span>

<span data-ttu-id="49148-131">封裝和部署資源的中樞和支點模型會使用後援程序來找出適當的資源。</span><span class="sxs-lookup"><span data-stu-id="49148-131">The hub and spoke model for packaging and deploying resources uses a fallback process to locate appropriate resources.</span></span> <span data-ttu-id="49148-132">如果應用程式要求的當地語系化資源無法使用，通用語言執行平台 (CLR) 就會在文化特性的階層架構中搜尋最接近使用者應用程式所要求的適當後援資源，而擲回例外狀況只會做為最後手段。</span><span class="sxs-lookup"><span data-stu-id="49148-132">If an application requests a localized resource  that is unavailable, the common language runtime searches the hierarchy of cultures for an appropriate fallback resource that most closely matches the user's application's request, and throws an exception only as a last resort.</span></span> <span data-ttu-id="49148-133">在階層的每個層級，如果找到適當的資源，則執行階段會使用它。</span><span class="sxs-lookup"><span data-stu-id="49148-133">At each level of the hierarchy, if an appropriate resource is found, the runtime uses it.</span></span> <span data-ttu-id="49148-134">如果找不到資源，則會繼續搜尋下一個層級。</span><span class="sxs-lookup"><span data-stu-id="49148-134">If the resource is not found, the search continues at the next level.</span></span>

### <a name="net-framework-resource-fallback-process"></a><span data-ttu-id="49148-136">.NET Framework 資源後援處理序</span><span class="sxs-lookup"><span data-stu-id="49148-136">.NET Framework resource fallback process</span></span>
