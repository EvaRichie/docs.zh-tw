 <span data-ttu-id="f2704-132">假設您的物件具有 STA 執行緒，則您需要執行緒之間的通訊方法，並驗證您位於正確的執行緒。</span><span class="sxs-lookup"><span data-stu-id="f2704-132">Given that you have objects with STA threading, you need a way to communicate between threads, and validate that you are on the correct thread.</span></span> <span data-ttu-id="f2704-133">其中具有發送器的角色。</span><span class="sxs-lookup"><span data-stu-id="f2704-133">Herein lies the role of the dispatcher.</span></span> <span data-ttu-id="f2704-134">發送器是基本訊息發送系統，具有多個已設定優先權的佇列。</span><span class="sxs-lookup"><span data-stu-id="f2704-134">The dispatcher is a basic message dispatching system, with multiple prioritized queues.</span></span> <span data-ttu-id="f2704-135">訊息範例包括未經處理的輸入通知 (滑鼠移動)、架構功能 (版面配置) 或使用者命令 (執行此方法)。</span><span class="sxs-lookup"><span data-stu-id="f2704-135">Examples of messages include raw input notifications (mouse moved), framework functions (layout), or user commands (execute this method).</span></span> <span data-ttu-id="f2704-136">藉由衍生自 <xref:System.Windows.Threading.DispatcherObject> ，您可以建立具有 STA 行為的 CLR 物件，並在建立時給予發送器的指標。</span><span class="sxs-lookup"><span data-stu-id="f2704-136">By deriving from <xref:System.Windows.Threading.DispatcherObject>, you create a CLR object that has STA behavior, and will be given a pointer to a dispatcher at creation time.</span></span>
