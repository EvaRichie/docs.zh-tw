# <a name="alignment-margins-and-padding-overview"></a>對齊、邊界和填補概觀
<xref:System.Windows.FrameworkElement>類別會公開數個用來精確定位子項目的屬性。 本主題討論四個最重要的屬性： <xref:System.Windows.FrameworkElement.HorizontalAlignment%2A> 、 <xref:System.Windows.FrameworkElement.Margin%2A> 、 <xref:System.Windows.Controls.Border.Padding%2A> 和 <xref:System.Windows.FrameworkElement.VerticalAlignment%2A> 。 一定要了解這些屬性的作用，因為它們提供控制 [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] 應用程式中項目位置的基礎。

## <a name="understanding-alignment-properties"></a>了解對齊屬性  
 <xref:System.Windows.FrameworkElement.HorizontalAlignment%2A>和 <xref:System.Windows.FrameworkElement.VerticalAlignment%2A> 屬性會描述子專案應如何放在父項目的已配置版面配置空間內。 藉由一起使用這些屬性，您可以精確定位子項目。 例如，的子專案 <xref:System.Windows.Controls.DockPanel> 可以指定四種不同的水準對齊方式： <xref:System.Windows.HorizontalAlignment.Left> 、 <xref:System.Windows.HorizontalAlignment.Right> 或 <xref:System.Windows.HorizontalAlignment.Center> ，或以 <xref:System.Windows.HorizontalAlignment.Stretch> 填滿可用空間。 類似的值可供垂直定位之用。  
  
> [!NOTE]
> 明確設定專案 <xref:System.Windows.FrameworkElement.Height%2A> 上的和 <xref:System.Windows.FrameworkElement.Width%2A> 屬性會優先于 <xref:System.Windows.HorizontalAlignment.Stretch> 屬性值。 嘗試設定 <xref:System.Windows.FrameworkElement.Height%2A> 、和的值，會 <xref:System.Windows.FrameworkElement.Width%2A> <xref:System.Windows.FrameworkElement.HorizontalAlignment%2A> `Stretch` 導致 `Stretch` 忽略要求。

## <a name="understanding-margin-properties"></a>了解邊界屬性  
 <xref:System.Windows.FrameworkElement.Margin%2A>屬性會描述元素及其子系或對等之間的距離。 <xref:System.Windows.FrameworkElement.Margin%2A>值可以是統一的，方法是使用類似的語法 `Margin="20"` 。 使用此語法時，會將 <xref:System.Windows.FrameworkElement.Margin%2A> 20 個裝置獨立圖元的統一套用至元素。 <xref:System.Windows.FrameworkElement.Margin%2A>值也可以採用四個相異值的形式，每個值都會描述不同的邊界，以套用至左側、頂端、右側和底部（依該順序），例如 `Margin="0,10,5,25"` 。 適當地使用 <xref:System.Windows.FrameworkElement.Margin%2A> 屬性，可讓您非常精確地控制元素的轉譯位置，以及其鄰近專案和子系的呈現位置。

> 非零的邊界會在專案的和之外套用 <xref:System.Windows.FrameworkElement.ActualWidth%2A> 空間 <xref:System.Windows.FrameworkElement.ActualHeight%2A> 。
