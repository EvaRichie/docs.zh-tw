# <a name="ihostgcmanagerthreadisblockingforsuspension-method"></a>IHostGCManager::ThreadIsBlockingForSuspension 方法

通知主機發出方法呼叫的執行緒，即將封鎖垃圾收集。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT ThreadIsBlockingForSuspension ();  
```  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`ThreadIsBlockingForSuspension` 傳回成功。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime (CLR) 尚未載入至進程，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫已超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|當封鎖的執行緒或光纖正在等候時，已取消事件。|  
|E_FAIL|發生未知的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  

 CLR 通常會 `ThreadIsBlockForSuspension` 在準備垃圾收集時呼叫方法，讓主機有機會重新排程未受管理工作的執行緒。  
  
> [!IMPORTANT]
> 主機只能在呼叫之後重新排程工作 `ThreadIsBlockingForSuspension` 。 執行時間呼叫 [SuspensionStarting](ihostgcmanager-suspensionstarting-method.md)之後，主機不能重新排程工作。
