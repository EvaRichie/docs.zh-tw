# <a name="iclrtaskyieldtask-method"></a>ICLRTask::YieldTask 方法
要求 common language runtime （CLR）放置目前[ICLRTask](iclrtask-interface.md)實例所代表的工作，並將處理器時間提供給其他工作使用。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT YieldTask ();  
```  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|Description|  
|-------------|-----------------|  
|S_OK|`YieldTask`已成功傳回。|  
|HOST_E_CLRNOTAVAILABLE|CLR 尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 就無法在進程內使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  
 主機呼叫 `YieldTask` 以要求其他工作或進程的處理器資源。 這個方法主要是為了讓長時間執行的程式碼放棄 CPU 時間。 執行時間會嘗試將目前實例所代表的工作，放 `ICLRTask` 在可以產生處理時間的狀態中，但不保證會成功。
