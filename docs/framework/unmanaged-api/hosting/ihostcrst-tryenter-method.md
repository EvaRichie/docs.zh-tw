# <a name="ihostcrsttryenter-method"></a>IHostCrst::TryEnter 方法
嘗試輸入目前[IHostCrst](ihostcrst-interface.md)實例所表示的重要區段。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT TryEnter (  
    [in]  DWORD  option,  
    [out] BOOL   *pbSucceeded  
);  
```  
  
## <a name="parameters"></a>參數  
 `option`  
 在其中一個[WAIT_OPTION](wait-option-enumeration.md)值，指出當作業封鎖時，主機應採取的動作。  
  
 `pbSucceeded`  
 [out] `true`如果可以輸入關鍵區段，則為，否則為 `false` 。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`TryEnter`已成功傳回。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime （CLR）尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 就無法在進程內使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  
 `TryEnter`會立即傳回，並指出呼叫執行緒是否已進入重要區段。 這個方法會鏡像 Wind32 `TryEnterCriticalSection` 函數。
