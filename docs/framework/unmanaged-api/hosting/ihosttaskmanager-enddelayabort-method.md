# <a name="ihosttaskmanagerenddelayabort-method"></a>IHostTaskManager::EndDelayAbort 方法

在先前對 [IHostTaskManager：： BeginDelayAbort](ihosttaskmanager-begindelayabort-method.md)的呼叫之後，通知主機 managed 程式碼即將結束，而無法中止目前的工作。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT EndDelayAbort ();  
```  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`EndDelayAbort` 傳回成功。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime (CLR) 尚未載入至進程，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫已超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|當封鎖的執行緒或光纖正在等候時，已取消事件。|  
|E_FAIL|發生未知的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
|E_UNEXPECTED|`EndDelayAbort` 呼叫時沒有對應的呼叫 `BeginDelayAbort` 。|  
  
## <a name="remarks"></a>備註  

 在呼叫之前，CLR 會在目前的工作上進行對應的呼叫 `BeginDelayAbort` `EndDelayAbort` 。 如果沒有這類對應的呼叫，主機的 [IHostTaskManager](ihosttaskmanager-interface.md) 應該會傳回 E_UNEXPECTED `EndDelayAbort` ，且不應該採取任何動作。
