# <a name="ihostsecuritycontextcapture-method"></a>IHostSecurityContext::Capture 方法

取得從呼叫[IHostSecurityManager：： GetSecurityCoNtext](ihostsecuritymanager-getsecuritycontext-method.md)傳回之[IHostSecurityCoNtext](ihostsecuritycontext-interface.md)實例的複製。  
  
## <a name="syntax"></a>語法  
  
```cpp
HRESULT Capture (  
    [out] IHostSecurityContext** ppClonedContext  
);  
```  
  
## <a name="parameters"></a>參數  

 `ppClonedContext`  
 擴展要捕捉之物件的複本位址指標 `IHostSecurityContext` 。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`Capture` 傳回成功。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime (CLR) 尚未載入至進程，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫已超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|當封鎖的執行緒或光纖正在等候時，已取消事件。|  
|E_FAIL|發生未知的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  

 從傳回的介面指標 `Capture` 是所捕獲內容的複本。 當這項資訊跨非同步程式碼點移動時，其存留期會與進行呼叫的指標分隔開來。 因此，可以釋放原始指標。
