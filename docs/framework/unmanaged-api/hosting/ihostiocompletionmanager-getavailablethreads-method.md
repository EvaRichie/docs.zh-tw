# <a name="ihostiocompletionmanagergetavailablethreads-method"></a>IHostIoCompletionManager::GetAvailableThreads 方法

取得目前未處理要求之主機所管理的執行緒總數的 i/o 完成執行緒數目。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT GetAvailableThreads (  
    [out] DWORD *pdwAvailableIoCompletionThreads  
);  
```  
  
## <a name="parameters"></a>參數  

 `pdwAvailableIoCompletionThreads`  
 擴展主機管理的 i/o 完成執行緒數目指標，目前可供服務要求使用。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`GetAvailableThreads` 傳回成功。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime (CLR) 尚未載入至進程，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫已超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|當封鎖的執行緒或光纖正在等候時，已取消事件。|  
|E_FAIL|發生未知的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
|E_NOTIMPL|主機未提供的實作為 `GetAvailableThreads` 。|  
  
## <a name="remarks"></a>備註  

 主機可能會想要對 i/o 完成執行緒集區的大小進行獨佔控制，例如執行、效能或擴充性之類的原因。 因此，主機不需要執行 `GetAvailableThreads` 。 在此情況下，主機應該會從此方法傳回 E_NOTIMPL。
