# <a name="iclrassemblyidentitymanagergetclrassemblyreferencelist-method"></a>ICLRAssemblyIdentityManager::GetCLRAssemblyReferenceList 方法
從提供的部分元件識別清單中，取得[ICLRAssemblyReferenceList](iclrassemblyreferencelist-interface.md)實例的介面指標。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT  GetCLRAssemblyReferenceList (  
    [in] LPCWSTR *ppwzAssemblyReferences,  
    [in] DWORD    dwNumOfReferences,  
    [out] ICLRAssemblyReferenceList **ppReferenceList  
);  
```  
  
## <a name="parameters"></a>參數  
 `ppwzAssemblyReferences`  
 在以 null 結束的字串陣列，格式為 "name，property = value ..."，指定部分元件身分識別的清單。  
  
 `dwNumOfReferences`  
 在中的專案數 `ppwzAssemblyReferences` 。  
  
 `ppReferenceList`  
 脫銷`ICLRAssemblyReferenceList`物件的介面指標，其中包含所指定元件清單的元件識別資料 `ppwzAssemblyReferences` 。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|說明|  
|-------------|-----------------|  
|S_OK|此方法已成功傳回。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime （CLR）尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 如果方法傳回 E_FAIL，就無法在進程內使用 CLR。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|
