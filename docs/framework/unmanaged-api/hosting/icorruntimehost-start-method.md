# <a name="icorruntimehoststart-method"></a>ICorRuntimeHost::Start 方法

啟動 common language runtime (CLR) 。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT Start ();  
```  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|作業成功。|  
|S_FALSE|作業無法完成。|  
|E_FAIL|發生未知的嚴重失敗。 如果方法傳回 E_FAIL，則 CLR 在進程中已無法再使用。 對任何裝載 Api 的後續呼叫都會傳回 HOST_E_CLRNOTAVAILABLE。|  
|HOST_E_CLRNOTAVAILABLE|CLR 尚未載入至進程，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
  
## <a name="remarks"></a>備註  

 通常不需要呼叫 `Start` 方法，因為 CLR 會在第一次執行 managed 程式碼的要求時自動啟動。
