# <a name="ihostthreadpoolmanagerqueueuserworkitem-method"></a>IHostThreadPoolManager::QueueUserWorkItem 方法
將函式排入佇列以進行執行，並指定物件，其中包含該函數要使用的資料。 當執行緒變成可用時，就會執行此函式。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT QueueUserWorkItem (  
    [in] LPTHREAD_START_ROUTINE Function,  
    [in] PVOID Context,  
    [in] ULONG Flags  
);  
```  
  
## <a name="parameters"></a>參數  
 `Function`  
 在函式指標，表示要執行的函式。  
  
 `Context`  
 在包含要使用之資料的物件 `Function` 。  
  
 `Flags`  
 在其中一個旗標值（如 Win32 方法所定義 `QueueUserWorkItem` ），可控制執行。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`QueueUserWorkItem`已成功傳回。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime （CLR）尚未載入進程中，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|已封鎖的執行緒或光纖在等候時取消了事件。|  
|E_FAIL|發生不明的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 就無法在進程內使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  
 `QueueUserWorkItem`將工作專案排入執行緒集區中的背景工作執行緒佇列。 其簽章和參數類型與對應的 Win32 函式（具有相同的名稱）相同。 如需詳細資訊，請參閱 Windows 平臺檔。
