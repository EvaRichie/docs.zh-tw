# <a name="ihosttasksetclrtask-method"></a>IHostTask::SetCLRTask 方法

將 `ICLRTask` 實例與目前的 [IHostTask](ihosttask-interface.md) 實例產生關聯。  
  
## <a name="syntax"></a>語法  
  
```cpp  
HRESULT SetCLRTask (  
    [in] ICLRTask *pCLRTask  
);  
```  
  
## <a name="parameters"></a>參數  

 `pCLRTask`  
 在要 `ICLRTask` 與目前實例相關聯之實例的介面指標 `IHostTask` 。  
  
## <a name="return-value"></a>傳回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|`SetCLRTask` 傳回成功。|  
|HOST_E_CLRNOTAVAILABLE|Common language runtime (CLR) 尚未載入至進程，或 CLR 處於無法執行 managed 程式碼或成功處理呼叫的狀態。|  
|HOST_E_TIMEOUT|呼叫已超時。|  
|HOST_E_NOT_OWNER|呼叫端沒有擁有鎖定。|  
|HOST_E_ABANDONED|當封鎖的執行緒或光纖正在等候時，已取消事件。|  
|E_FAIL|發生未知的嚴重失敗。 當方法傳回 E_FAIL 時，CLR 在進程內將無法再使用。 對裝載方法的後續呼叫會傳回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>備註  

 CLR 會呼叫， `SetCLRTask` 以將 `ICLRTask` 實例與目前的 `IHostTask` 實例產生關聯，而此實例是由對 [IHostTaskManager：： CreateTask](ihosttaskmanager-createtask-method.md)的呼叫所建立。
