## <a name="pool-creation-and-assignment"></a><span data-ttu-id="9aeed-130">集區的建立及指派</span><span class="sxs-lookup"><span data-stu-id="9aeed-130">Pool Creation and Assignment</span></span>

 <span data-ttu-id="9aeed-136">在下列 C# 範例中，會建立三個新的 <xref:System.Data.SqlClient.SqlConnection> 物件，但是只需要兩個連接集區來管理它們。</span><span class="sxs-lookup"><span data-stu-id="9aeed-136">In the following C# example, three new <xref:System.Data.SqlClient.SqlConnection> objects are created, but only two connection pools are required to manage them.</span></span> <span data-ttu-id="9aeed-137">請注意，第一個及第二個連接字串的不同之處在於指派給 `Initial Catalog` 的值不同。</span><span class="sxs-lookup"><span data-stu-id="9aeed-137">Note that the first and second connection strings differ by the value assigned for `Initial Catalog`.</span></span>  
  
```csharp

  "Integrated Security=SSPI;Initial Catalog=Northwind"))  
    {  
        connection.Open();
        // Pool A is created.  
    }

  "Integrated Security=SSPI;Initial Catalog=pubs"))  
    {  
        connection.Open();
        // Pool B is created because the connection strings differ.  
    }

  "Integrated Security=SSPI;Initial Catalog=Northwind"))  
    {  
        connection.Open();
        // The connection string matches pool A.

```

 <span data-ttu-id="9aeed-167">如果連接是與已消失的伺服器連接，即使連接共用器尚未偵測到已嚴重損毀的連接並將其標記為無效，此連接還是會從集區中建立。</span><span class="sxs-lookup"><span data-stu-id="9aeed-167">If a connection exists to a server that has disappeared, this connection can be drawn from the pool even if the connection pooler has not detected the severed connection and marked it as invalid.</span></span> <span data-ttu-id="9aeed-168">這是因為檢查連接是否仍然有效的額外負荷抵銷了集區的優點，它導致了與伺服器之間的往返通訊。</span><span class="sxs-lookup"><span data-stu-id="9aeed-168">This is the case because the overhead of checking that the connection is still valid would eliminate the benefits of having a pooler by causing another round trip to the server to occur.</span></span> <span data-ttu-id="9aeed-169">發生此情況時，第一次嘗試使用該連接時將偵測到連接已嚴重損毀，並擲回例外狀況。</span><span class="sxs-lookup"><span data-stu-id="9aeed-169">When this occurs, the first attempt to use the connection will detect that the connection has been severed, and an exception is thrown.</span></span>

## <a name="transaction-support"></a><span data-ttu-id="9aeed-175">異動支援</span><span class="sxs-lookup"><span data-stu-id="9aeed-175">Transaction Support</span></span>

## <a name="pool-fragmentation"></a><span data-ttu-id="9aeed-187">集區片段</span><span class="sxs-lookup"><span data-stu-id="9aeed-187">Pool Fragmentation</span></span>  

 <span data-ttu-id="9aeed-188">集區片段是許多 Web 應用程式中的常見問題，這些應用程式可能會建立大量集區，並且直至處理序結束後才釋放它們。</span><span class="sxs-lookup"><span data-stu-id="9aeed-188">Pool fragmentation is a common problem in many Web applications where the application can create a large number of pools that are not freed until the process exits.</span></span> <span data-ttu-id="9aeed-189">這會使大量連接保持開啟狀態並消耗記憶體，導致效能降低。</span><span class="sxs-lookup"><span data-stu-id="9aeed-189">This leaves a large number of connections open and consuming memory, which results in poor performance.</span></span>  
  
### <a name="pool-fragmentation-due-to-integrated-security"></a><span data-ttu-id="9aeed-190">整合安全性導致的集區片段</span><span class="sxs-lookup"><span data-stu-id="9aeed-190">Pool Fragmentation Due to Integrated Security</span></span>
