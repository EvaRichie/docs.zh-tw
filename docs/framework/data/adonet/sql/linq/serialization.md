## <a name="overview"></a><span data-ttu-id="0adc8-109">概觀</span><span class="sxs-lookup"><span data-stu-id="0adc8-109">Overview</span></span>  

 <span data-ttu-id="0adc8-110">所產生的程式碼 [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] 預設會提供延後載入功能。</span><span class="sxs-lookup"><span data-stu-id="0adc8-110">The code generated by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] provides deferred loading capabilities by default.</span></span> <span data-ttu-id="0adc8-111">在視需要透明載入資料的中介層中，延後載入非常方便。</span><span class="sxs-lookup"><span data-stu-id="0adc8-111">Deferred loading is very convenient on the mid-tier for transparent loading of data on demand.</span></span> <span data-ttu-id="0adc8-112">但是，由於不論是否需要延後載入，序列化程式都會觸發延後載入，所以序列化會有問題。</span><span class="sxs-lookup"><span data-stu-id="0adc8-112">However, it is problematic for serialization, because the serializer triggers deferred loading whether deferred loading is intended or not.</span></span> <span data-ttu-id="0adc8-113">事實上，序列化物件後，在所有傳出延後載入的參考之下的遞移封閉 (Transitive Closure) 也已序列化。</span><span class="sxs-lookup"><span data-stu-id="0adc8-113">In effect, when an object is serialized, its transitive closure under all outbound defer-loaded references is serialized.</span></span>  
  
 <span data-ttu-id="0adc8-114">[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)]序列化功能會解決這個問題，主要是透過兩種機制：</span><span class="sxs-lookup"><span data-stu-id="0adc8-114">The [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] serialization feature addresses this problem, primarily through two mechanisms:</span></span>  
  
- <span data-ttu-id="0adc8-115">用於關閉延後載入的 <xref:System.Data.Linq.DataContext> 模式 (<xref:System.Data.Linq.DataContext.ObjectTrackingEnabled%2A>)。</span><span class="sxs-lookup"><span data-stu-id="0adc8-115">A <xref:System.Data.Linq.DataContext> mode for turning off deferred loading (<xref:System.Data.Linq.DataContext.ObjectTrackingEnabled%2A>).</span></span> <span data-ttu-id="0adc8-116">如需詳細資訊，請參閱<xref:System.Data.Linq.DataContext>。</span><span class="sxs-lookup"><span data-stu-id="0adc8-116">For more information, see <xref:System.Data.Linq.DataContext>.</span></span>  
  
- <span data-ttu-id="0adc8-117">程式碼產生參數，用在產生的實體上產生 <xref:System.Runtime.Serialization.DataContractAttribute?displayProperty=nameWithType> 和 <xref:System.Runtime.Serialization.DataMemberAttribute?displayProperty=nameWithType> 屬性 (Attribute)。</span><span class="sxs-lookup"><span data-stu-id="0adc8-117">A code-generation switch to generate <xref:System.Runtime.Serialization.DataContractAttribute?displayProperty=nameWithType> and <xref:System.Runtime.Serialization.DataMemberAttribute?displayProperty=nameWithType> attributes on generated entities.</span></span> <span data-ttu-id="0adc8-118">這個方面 (包含序列化之下延後載入類別的行為) 為本主題的主旨。</span><span class="sxs-lookup"><span data-stu-id="0adc8-118">This aspect, including the behavior of defer-loading classes under serialization, is the major subject of this topic.</span></span>  
  
### <a name="definitions"></a><span data-ttu-id="0adc8-119">定義</span><span class="sxs-lookup"><span data-stu-id="0adc8-119">Definitions</span></span>  
  
- <span data-ttu-id="0adc8-120">*DataContract 序列化程式*： Windows Communication Framework 所使用的預設序列化程式， (.NET Framework 3.0 或更新版本的 WCF) 元件。</span><span class="sxs-lookup"><span data-stu-id="0adc8-120">*DataContract serializer*: Default serializer used by the Windows Communication Framework (WCF) component of the .NET Framework 3.0 or later versions.</span></span>  
  
- <span data-ttu-id="0adc8-121">*單向序列化*：類別的序列化版本，其中只包含單向關聯屬性 (，以避免迴圈) 。</span><span class="sxs-lookup"><span data-stu-id="0adc8-121">*Unidirectional serialization*: The serialized version of a class that contains only a one-way association property (to avoid a cycle).</span></span> <span data-ttu-id="0adc8-122">依照慣例，主索引鍵-外部索引鍵關聯性之父端上的屬性 (Poperty) 會標記為即將序列化。</span><span class="sxs-lookup"><span data-stu-id="0adc8-122">By convention, the property on the parent side of a primary-foreign key relationship is marked for serialization.</span></span> <span data-ttu-id="0adc8-123">雙向關聯中的另一端則不會序列化。</span><span class="sxs-lookup"><span data-stu-id="0adc8-123">The other side in a bidirectional association is not serialized.</span></span>  
  
     <span data-ttu-id="0adc8-124">單向序列化是 [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] 唯一支援的序列化類型。</span><span class="sxs-lookup"><span data-stu-id="0adc8-124">Unidirectional serialization is the only type of serialization supported by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)].</span></span>
