## <a name="considerations-for-working-with-a-streaming-provider"></a><span data-ttu-id="6e614-175">使用資料流處理提供者的考量</span><span class="sxs-lookup"><span data-stu-id="6e614-175">Considerations for Working with a Streaming Provider</span></span>

<span data-ttu-id="6e614-176">下列是當您實作資料流處理提供者以及從資料服務存取媒體資源時，所要考量的事項。</span><span class="sxs-lookup"><span data-stu-id="6e614-176">The following are things to consider when you implement a streaming provider and when you access media resources from a data service.</span></span>

- <span data-ttu-id="6e614-179">POST 要求不能用來建立新的媒體連結項目。</span><span class="sxs-lookup"><span data-stu-id="6e614-179">A POST request cannot be used to create a new media link entry.</span></span> <span data-ttu-id="6e614-180">您必須改為發出 POST 要求來建立新的媒體資源，而資料服務會使用預設值建立新的媒體連結項目。</span><span class="sxs-lookup"><span data-stu-id="6e614-180">Instead, you must issue a POST request to create a new media resource, and the data service creates a new media link entry with default values.</span></span> <span data-ttu-id="6e614-181">後續的 MERGE 或 PUT 要求可以更新這個新的實體。</span><span class="sxs-lookup"><span data-stu-id="6e614-181">This new entity can be updated by a subsequent MERGE or PUT request.</span></span> <span data-ttu-id="6e614-182">您也可以考慮快取實體，並在處置工具中進行更新，例如將屬性值設定為 POST 要求中 Slug 標頭的值。</span><span class="sxs-lookup"><span data-stu-id="6e614-182">You may also consider caching the entity and make updates in the disposer, such as setting the property value to the value of the Slug header in the POST request.</span></span>
