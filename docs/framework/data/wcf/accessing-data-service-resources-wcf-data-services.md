## <a name="addressing-resources"></a><span data-ttu-id="0a006-107">設定資源存取位址</span><span class="sxs-lookup"><span data-stu-id="0a006-107">Addressing Resources</span></span>  
 <span data-ttu-id="0a006-108">在 OData 中，您可以使用 URI 來處理資料模型所公開的任何資料。</span><span class="sxs-lookup"><span data-stu-id="0a006-108">In OData, you address any data exposed by the data model by using a URI.</span></span> <span data-ttu-id="0a006-109">例如，下列 URI 會傳回 Customer 實體集的摘要，其中包含 Customer 實體類型之所有執行個體的項目：</span><span class="sxs-lookup"><span data-stu-id="0a006-109">For example, the following URI returns a feed that is the Customers entity set, which contains entries for all instances of the Customer entity type:</span></span>

 <span data-ttu-id="0a006-110">實體還包含稱為實體索引鍵的特殊屬性。</span><span class="sxs-lookup"><span data-stu-id="0a006-110">Entities have special properties called entity keys.</span></span> <span data-ttu-id="0a006-111">實體索引鍵用於以唯一方式識別實體集中的單一實體。</span><span class="sxs-lookup"><span data-stu-id="0a006-111">An entity key is used to uniquely identify a single entity in an entity set.</span></span> <span data-ttu-id="0a006-112">這樣可以讓您定址實體集中某個實體型別的特殊執行個體。</span><span class="sxs-lookup"><span data-stu-id="0a006-112">This enables you to address a specific instance of an entity type in the entity set.</span></span> <span data-ttu-id="0a006-113">例如，下列 URI 會傳回 Customer 實體類型之特定執行個體的項目，此執行個體的索引鍵值為 `ALFKI`：</span><span class="sxs-lookup"><span data-stu-id="0a006-113">For example, the following URI returns an entry for a specific instance of the Customer entity type that has a key value of `ALFKI`:</span></span>

 <span data-ttu-id="0a006-114">您也可以個別定址實體執行個體的基本和複雜屬性。</span><span class="sxs-lookup"><span data-stu-id="0a006-114">Primitive and complex properties of an entity instance can also be individually addressed.</span></span> <span data-ttu-id="0a006-115">例如，下列 URI 會傳回 XML 項目，其中包含特定 Customer 的 `ContactName` 屬性值：</span><span class="sxs-lookup"><span data-stu-id="0a006-115">For example, the following URI returns an XML element that contains the `ContactName` property value for a specific Customer:</span></span>

 <span data-ttu-id="0a006-116">當您在之前的 URI 中包含 `$value` 端點時，回應訊息中只會傳回基本屬性的值。</span><span class="sxs-lookup"><span data-stu-id="0a006-116">When you include the `$value` endpoint in the previous URI, only the value of the primitive property is returned in the response message.</span></span> <span data-ttu-id="0a006-117">下列範例只會傳回 "Maria Anders" 字串而沒有 XML 項目：</span><span class="sxs-lookup"><span data-stu-id="0a006-117">The following example returns only the string "Maria Anders" without the XML element:</span></span>

 <span data-ttu-id="0a006-118">資料模型中會以關聯性定義實體之間的關係。</span><span class="sxs-lookup"><span data-stu-id="0a006-118">Relationships between entities are defined in the data model by associations.</span></span> <span data-ttu-id="0a006-119">這些關聯可讓您定址相關實體，其方式是使用實體執行個體的導覽屬性。</span><span class="sxs-lookup"><span data-stu-id="0a006-119">These associations enable you to address related entities by using navigation properties of an entity instance.</span></span> <span data-ttu-id="0a006-120">導覽屬性可在多對一關聯性中傳回單一相關實體，或是在一對多關聯性中傳回一組相關實體。</span><span class="sxs-lookup"><span data-stu-id="0a006-120">A navigation property can return either a single related entity, in the case of a many-to-one relationship, or a set of related entities, in the case of a one-to-many relationship.</span></span> <span data-ttu-id="0a006-121">例如，下列 URI 會傳回一個摘要，它是與特定 Customer 相關之所有 Order 的集合：</span><span class="sxs-lookup"><span data-stu-id="0a006-121">For example, the following URI returns a feed that is the set of all the Orders that are related to a specific Customer:</span></span>

 <span data-ttu-id="0a006-122">關聯性 (通常是雙向關聯性) 是由一對導覽屬性來表示。</span><span class="sxs-lookup"><span data-stu-id="0a006-122">Relationships, which are usually bi-directional, are represented by a pair of navigation properties.</span></span> <span data-ttu-id="0a006-123">下列 URI 會傳回特定 Order 實體所屬之 Customer 實體的參考，與之前範例所示的關聯性相反：</span><span class="sxs-lookup"><span data-stu-id="0a006-123">As the reverse of the relationship shown in the previous example, the following URI returns a reference to the Customer entity to which a specific Order entity belongs:</span></span>

 <span data-ttu-id="0a006-124">OData 也可以讓您根據查詢運算式的結果來定址資源。</span><span class="sxs-lookup"><span data-stu-id="0a006-124">OData also enables you to address resources based on the results of query expressions.</span></span> <span data-ttu-id="0a006-125">這樣可以讓您根據經過評估的運算式篩選資源集。</span><span class="sxs-lookup"><span data-stu-id="0a006-125">This makes it possible to filter sets of resources based on an evaluated expression.</span></span> <span data-ttu-id="0a006-126">例如，下列 URI 會篩選資源，只傳回自 1997 年 9 月 22 日後配送且屬於指定 Customer 的 Orders。</span><span class="sxs-lookup"><span data-stu-id="0a006-126">For example, the following URI filters the resources to return only the Orders for the specified Customer that have shipped since September 22, 1997:</span></span>

 <span data-ttu-id="0a006-127">如需詳細資訊，請參閱[OData： URI 慣例](https://www.odata.org/documentation/odata-version-2-0/uri-conventions/)。</span><span class="sxs-lookup"><span data-stu-id="0a006-127">For more information, see [OData: URI Conventions](https://www.odata.org/documentation/odata-version-2-0/uri-conventions/).</span></span>
