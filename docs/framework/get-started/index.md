## <a name="what-is-net-framework"></a><span data-ttu-id="f7968-107">什麼是 .NET Framework？</span><span class="sxs-lookup"><span data-stu-id="f7968-107">What is .NET Framework?</span></span>

<span data-ttu-id="f7968-108">.NET Framework 是適用于 Windows 的受控執行環境，可為執行中的應用程式提供各種服務。</span><span class="sxs-lookup"><span data-stu-id="f7968-108">.NET Framework is a managed execution environment for Windows that provides a variety of services to its running apps.</span></span> <span data-ttu-id="f7968-109">其由兩個主要元件組成：通用語言執行平台 (CLR) 和 .NET Framework 類別庫，前者是負責處理執行中應用程式的執行引擎，後者提供通過測試、可重複使用的程式碼程式庫，讓開發人員可從自己的應用程式中呼叫。</span><span class="sxs-lookup"><span data-stu-id="f7968-109">It consists of two major components: the common language runtime (CLR), which is the execution engine that handles running apps, and the .NET Framework Class Library, which provides a library of tested, reusable code that developers can call from their own apps.</span></span> <span data-ttu-id="f7968-110">.NET Framework 提供給執行中應用程式的服務包括下列各項：</span><span class="sxs-lookup"><span data-stu-id="f7968-110">The services that .NET Framework provides to running apps include the following:</span></span>

- <span data-ttu-id="f7968-111">記憶體管理。</span><span class="sxs-lookup"><span data-stu-id="f7968-111">Memory management.</span></span> <span data-ttu-id="f7968-112">在許多程式設計語言中，程式設計人員負責配置和釋放記憶體，以及處理物件存留期。</span><span class="sxs-lookup"><span data-stu-id="f7968-112">In many programming languages, programmers are responsible for allocating and releasing memory and for handling object lifetimes.</span></span> <span data-ttu-id="f7968-113">在 .NET Framework 應用程式中，CLR 會代表應用程式提供這些服務。</span><span class="sxs-lookup"><span data-stu-id="f7968-113">In .NET Framework apps, the CLR provides these services on behalf of the app.</span></span>

- <span data-ttu-id="f7968-114">一般型別系統。</span><span class="sxs-lookup"><span data-stu-id="f7968-114">A common type system.</span></span> <span data-ttu-id="f7968-115">在傳統的程式語言中，基本型別是由編譯器所定義，這會讓跨語言互通性變得很複雜。</span><span class="sxs-lookup"><span data-stu-id="f7968-115">In traditional programming languages, basic types are defined by the compiler, which complicates cross-language interoperability.</span></span> <span data-ttu-id="f7968-116">在 .NET Framework 中，基本類型是由 .NET Framework 類型系統所定義，而且是以 .NET Framework 為目標的所有語言通用。</span><span class="sxs-lookup"><span data-stu-id="f7968-116">In .NET Framework, basic types are defined by the .NET Framework type system and are common to all languages that target .NET Framework.</span></span>

- <span data-ttu-id="f7968-117">廣泛類別庫。</span><span class="sxs-lookup"><span data-stu-id="f7968-117">An extensive class library.</span></span> <span data-ttu-id="f7968-118">程式設計人員不再需要撰寫大量的程式碼來處理常見的低階程式設計作業，而能夠使用 .NET Framework 類別庫中可立即存取的類型程式庫及其成員來完成這項工作。</span><span class="sxs-lookup"><span data-stu-id="f7968-118">Instead of having to write vast amounts of code to handle common low-level programming operations, programmers use a readily accessible library of types and their members from the .NET Framework Class Library.</span></span>

- <span data-ttu-id="f7968-119">開發架構和技術。</span><span class="sxs-lookup"><span data-stu-id="f7968-119">Development frameworks and technologies.</span></span> <span data-ttu-id="f7968-120">.NET Framework 包含特定應用程式開發領域的程式庫，例如 web 應用程式的 ASP.NET、適用于資料存取的 ADO.NET、服務導向應用程式的 Windows Communication Foundation，以及適用于 Windows 桌面應用程式的 Windows Presentation Foundation。</span><span class="sxs-lookup"><span data-stu-id="f7968-120">.NET Framework includes libraries for specific areas of app development, such as ASP.NET for web apps, ADO.NET for data access, Windows Communication Foundation for service-oriented apps, and Windows Presentation Foundation for Windows desktop apps.</span></span>

- <span data-ttu-id="f7968-121">語言互通性。</span><span class="sxs-lookup"><span data-stu-id="f7968-121">Language interoperability.</span></span> <span data-ttu-id="f7968-122">以 .NET Framework 的語言編譯器會發出名為通用中繼語言 (CIL) 的中繼程式碼，而該程式碼會在執行時間由 common Language runtime 進行編譯。</span><span class="sxs-lookup"><span data-stu-id="f7968-122">Language compilers that target .NET Framework emit an intermediate code named Common Intermediate Language (CIL), which, in turn, is compiled at runtime by the common language runtime.</span></span> <span data-ttu-id="f7968-123">有了此功能，使用某一種語言撰寫的常式就能供其他語言存取，而程式設計人員也能使用自己慣用的語言專心建立應用程式。</span><span class="sxs-lookup"><span data-stu-id="f7968-123">With this feature, routines written in one language are accessible to other languages, and programmers focus on creating apps in their preferred languages.</span></span>

- <span data-ttu-id="f7968-124">版本相容性。</span><span class="sxs-lookup"><span data-stu-id="f7968-124">Version compatibility.</span></span> <span data-ttu-id="f7968-125">在罕見的情況下，使用特定版本的 .NET Framework 開發的應用程式會在較新版本上執行，而不需要修改。</span><span class="sxs-lookup"><span data-stu-id="f7968-125">With rare exceptions, apps that are developed by using a particular version of .NET Framework run without modification on a later version.</span></span>

- <span data-ttu-id="f7968-126">並存執行。</span><span class="sxs-lookup"><span data-stu-id="f7968-126">Side-by-side execution.</span></span> <span data-ttu-id="f7968-127">.NET Framework 可讓多個版本的 common language runtime 存在於同一部電腦上，以協助解決版本衝突。</span><span class="sxs-lookup"><span data-stu-id="f7968-127">.NET Framework helps resolve version conflicts by allowing multiple versions of the common language runtime to exist on the same computer.</span></span> <span data-ttu-id="f7968-128">這表示應用程式的多個版本可以並存，而且應用程式可以在其建立所在的 .NET Framework 版本上執行。</span><span class="sxs-lookup"><span data-stu-id="f7968-128">This means that multiple versions of apps can coexist and that an app can run on the version of .NET Framework with which it was built.</span></span> <span data-ttu-id="f7968-129">並存執行適用於.NET Framework 版本群組 1.0/1.1、2.0/3.0/3.5 和 4/4.5.x/4.6.x/4.7.x/4.8。</span><span class="sxs-lookup"><span data-stu-id="f7968-129">Side-by-side execution applies to the .NET Framework version groups 1.0/1.1, 2.0/3.0/3.5, and 4/4.5.x/4.6.x/4.7.x/4.8.</span></span>

- <span data-ttu-id="f7968-130">多目標。</span><span class="sxs-lookup"><span data-stu-id="f7968-130">Multitargeting.</span></span> <span data-ttu-id="f7968-131">將目標設為 [.NET Standard](../../standard/net-standard.md)，開發人員就能建立可在該標準版本所支援的多個 .NET Framework 平台上運作的類別庫。</span><span class="sxs-lookup"><span data-stu-id="f7968-131">By targeting [.NET Standard](../../standard/net-standard.md), developers create class libraries that work on multiple .NET Framework platforms supported by that version of the standard.</span></span> <span data-ttu-id="f7968-132">例如，以 .NET Standard 2.0 為目標的程式庫可供以 .NET Framework 4.6.1、.NET Core 2.0 和 UWP 10.0.16299 為目標的應用程式使用。</span><span class="sxs-lookup"><span data-stu-id="f7968-132">For example, libraries that target .NET Standard 2.0 can be used by apps that target .NET Framework 4.6.1, .NET Core 2.0, and UWP 10.0.16299.</span></span>
