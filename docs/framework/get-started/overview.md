## <a name="features-of-the-common-language-runtime"></a><span data-ttu-id="6f36f-130">Common Language Runtime 的功能</span><span class="sxs-lookup"><span data-stu-id="6f36f-130">Features of the common language runtime</span></span>

<span data-ttu-id="6f36f-131">Common Language Runtime 負責管理記憶體、執行緒執行、程式碼執行、程式碼安全驗證、編譯 (Compilation) 和其他系統服務。</span><span class="sxs-lookup"><span data-stu-id="6f36f-131">The common language runtime manages memory, thread execution, code execution, code safety verification, compilation, and other system services.</span></span> <span data-ttu-id="6f36f-132">這些功能都內建到在 Common Language Runtime 上執行的 Managed 程式碼中。</span><span class="sxs-lookup"><span data-stu-id="6f36f-132">These features are intrinsic to the managed code that runs on the common language runtime.</span></span>

<span data-ttu-id="6f36f-133">就安全性而言，Managed 元件會根據若干因素而被授予不同程度的信任，這些因素包括元件的原始出處 (例如網際網路、企業網路或本機電腦)。</span><span class="sxs-lookup"><span data-stu-id="6f36f-133">Regarding security, managed components are awarded varying degrees of trust, depending on a number of factors that include their origin (such as the Internet, enterprise network, or local computer).</span></span> <span data-ttu-id="6f36f-134">這表示即使是在相同作用中的應用程式中使用，Managed 元件可能可以也可能無法執行檔案存取作業、註冊存取作業或其他易受影響的功能。</span><span class="sxs-lookup"><span data-stu-id="6f36f-134">This means that a managed component might or might not be able to perform file-access operations, registry-access operations, or other sensitive functions, even if it's used in the same active app.</span></span>

<span data-ttu-id="6f36f-135">Runtime 也會藉由實作嚴格的型別和程式碼驗證基礎架構，也就是一般型別系統 (CTS)，強制執行程式碼的加強性。</span><span class="sxs-lookup"><span data-stu-id="6f36f-135">The runtime also enforces code robustness by implementing a strict type-and-code-verification infrastructure called the common type system (CTS).</span></span> <span data-ttu-id="6f36f-136">CTS 確保所有 Managed 程式碼都能夠自我描述。</span><span class="sxs-lookup"><span data-stu-id="6f36f-136">The CTS ensures that all managed code is self-describing.</span></span> <span data-ttu-id="6f36f-137">不同的 Microsoft 和協力廠商語言編譯器會產生符合 CTS 的 Managed 程式碼。</span><span class="sxs-lookup"><span data-stu-id="6f36f-137">The various Microsoft and third-party language compilers generate managed code that conforms to the CTS.</span></span> <span data-ttu-id="6f36f-138">這表示 Managed 程式碼不但能夠使用其他 Managed 型別和執行個體，同時還能嚴格強制執行型別精確度和型別安全。</span><span class="sxs-lookup"><span data-stu-id="6f36f-138">This means that managed code can consume other managed types and instances, while strictly enforcing type fidelity and type safety.</span></span>

<span data-ttu-id="6f36f-139">此外，Runtime 的 Managed 環境排除許多常見的軟體問題。</span><span class="sxs-lookup"><span data-stu-id="6f36f-139">In addition, the managed environment of the runtime eliminates many common software issues.</span></span> <span data-ttu-id="6f36f-140">例如，Runtime 能夠自動處理物件配置，並管理對物件的參考，而且在不再用它們時加以釋放。</span><span class="sxs-lookup"><span data-stu-id="6f36f-140">For example, the runtime automatically handles object layout and manages references to objects, releasing them when they are no longer being used.</span></span> <span data-ttu-id="6f36f-141">這種自動記憶體管理解決了兩個最常見的應用程式錯誤：記憶體流失和無效的記憶體參考。</span><span class="sxs-lookup"><span data-stu-id="6f36f-141">This automatic memory management resolves the two most common app errors, memory leaks and invalid memory references.</span></span>

<span data-ttu-id="6f36f-142">Runtime 也提升開發人員的產能。</span><span class="sxs-lookup"><span data-stu-id="6f36f-142">The runtime also accelerates developer productivity.</span></span> <span data-ttu-id="6f36f-143">例如，程式設計人員可以用自己選擇的開發語言撰寫應用程式，但仍充分利用其他開發人員以其他語言所撰寫的執行階段、類別庫和元件。</span><span class="sxs-lookup"><span data-stu-id="6f36f-143">For example, programmers write apps in their development language of choice yet take full advantage of the runtime, the class library, and components written in other languages by other developers.</span></span> <span data-ttu-id="6f36f-144">選擇以執行階段為目標來開發編譯器的廠商都可作到這一點。</span><span class="sxs-lookup"><span data-stu-id="6f36f-144">Any compiler vendor who chooses to target the runtime can do so.</span></span> <span data-ttu-id="6f36f-145">以 .NET Framework 為目標的語言編譯器可以將 .NET Framework 的功能提供給以該語言所撰寫的現有程式碼使用，大幅簡化現有應用程式的移轉程序。</span><span class="sxs-lookup"><span data-stu-id="6f36f-145">Language compilers that target the .NET Framework make the features of the .NET Framework available to existing code written in that language, greatly easing the migration process for existing apps.</span></span>

<span data-ttu-id="6f36f-146">Runtime 是為未來軟體所設計的，但它也支援目前和過去的軟體。</span><span class="sxs-lookup"><span data-stu-id="6f36f-146">While the runtime is designed for the software of the future, it also supports software of today and yesterday.</span></span> <span data-ttu-id="6f36f-147">Managed 和 Unmanaged 程式碼的互通性 (Interoperability)，讓開發人員繼續使用必要的 COM 元件和 DLL。</span><span class="sxs-lookup"><span data-stu-id="6f36f-147">Interoperability between managed and unmanaged code enables developers to continue to use necessary COM components and DLLs.</span></span>

<span data-ttu-id="6f36f-148">Runtime 是為增強效能所設計的。</span><span class="sxs-lookup"><span data-stu-id="6f36f-148">The runtime is designed to enhance performance.</span></span> <span data-ttu-id="6f36f-149">雖然 Common Language Runtime 提供許多標準的執行階段服務，但未曾解譯 Managed 程式碼。</span><span class="sxs-lookup"><span data-stu-id="6f36f-149">Although the common language runtime provides many standard runtime services, managed code is never interpreted.</span></span> <span data-ttu-id="6f36f-150">透過一項稱為 Just-In-Time (JIT) 編譯的功能，所有的 Managed 程式碼都可使用執行所在系統的原生機器語言執行。</span><span class="sxs-lookup"><span data-stu-id="6f36f-150">A feature called just-in-time (JIT) compiling enables all managed code to run in the native machine language of the system on which it's executing.</span></span> <span data-ttu-id="6f36f-151">同時，記憶體管理員移除分散的記憶體的可能性，增加記憶體參考位置，以進一步提高效能。</span><span class="sxs-lookup"><span data-stu-id="6f36f-151">Meanwhile, the memory manager removes the possibilities of fragmented memory and increases memory locality-of-reference to further increase performance.</span></span>

<span data-ttu-id="6f36f-152">最後，執行階段可由高效能的伺服器端應用程式裝載，例如 Microsoft SQL Server 和 Internet Information Services (IIS)。</span><span class="sxs-lookup"><span data-stu-id="6f36f-152">Finally, the runtime can be hosted by high-performance, server-side apps, such as Microsoft SQL Server and Internet Information Services (IIS).</span></span> <span data-ttu-id="6f36f-153">這個基礎架構讓您使用 Managed 程式碼撰寫商務邏輯的同時，仍能夠享受到由業界最佳、可支援執行階段主應用程式的企業伺服器所提供的超高效能。</span><span class="sxs-lookup"><span data-stu-id="6f36f-153">This infrastructure enables you to use managed code to write your business logic, while still enjoying the superior performance of the industry's best enterprise servers that support runtime hosting.</span></span>
