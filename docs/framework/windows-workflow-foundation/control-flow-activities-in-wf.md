# <a name="control-flow-activities-in-wf"></a><span data-ttu-id="1a7bd-103">WF 中的控制流程活動</span><span class="sxs-lookup"><span data-stu-id="1a7bd-103">Control Flow Activities in WF</span></span>

<span data-ttu-id="1a7bd-104">[!INCLUDE[netfx_current_long](../../../includes/netfx-current-long-md.md)] 提供數種活動，可用於控制工作流程內的執行流程。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-104">The [!INCLUDE[netfx_current_long](../../../includes/netfx-current-long-md.md)] provides several activities for controlling flow of execution within a workflow.</span></span> <span data-ttu-id="1a7bd-105">其中一些活動 (例如 `Switch` 和) 會像 `If` 是 Visual c # 等程式設計環境中的方式來實行流程式控制制結構，而其他 (例如 `Pick`) 模型新的程式設計結構。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-105">Some of these activities (such as `Switch` and `If`) implement flow control structures similar to those in programming environments such as Visual C#, while others (such as `Pick`) model new programming structures.</span></span>  
  
 <span data-ttu-id="1a7bd-106">請注意，雖然 `Parallel` 和 `ParallelForEach` 之類的活動會排定多個同時執行的子活動，但只會使用單一執行緒來進行工作流程。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-106">Note that while activities such as the `Parallel` and `ParallelForEach` activities schedule multiple child activities for execution simultaneously, only a single thread is used for a workflow.</span></span> <span data-ttu-id="1a7bd-107">這些活動的每個子活動會循序執行，在先前的活動完成或閒置之前，後續的活動均不會執行。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-107">Each child activity of these activities executes sequentially and successive activities do not execute until previous activities either complete or go idle.</span></span> <span data-ttu-id="1a7bd-108">因此，最適合使用這些活動的應用程式即為必須交錯執行數個可能封鎖之活動的應用程式。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-108">As a result, these activities are most useful for applications in which several potentially blocking activities must execute in an interleaved fashion.</span></span> <span data-ttu-id="1a7bd-109">如果這些活動中沒有任何子活動閒置，`Parallel` 活動會像 `Sequence` 活動般執行，而 `ParallelForEach` 活動會像 `ForEach` 活動般執行。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-109">If none of the child activities of these activities go idle, a `Parallel` activity executes just like a `Sequence` activity, and a `ParallelForEach` activity executes just like a `ForEach` activity.</span></span> <span data-ttu-id="1a7bd-110">然而，如果使用非同步活動 (例如衍生自 <xref:System.Activities.AsyncCodeActivity> 的活動) 或傳訊活動，控制項會轉移至下一個分支，而子活動會等候其訊息被接收，或其非同步工作完成。</span><span class="sxs-lookup"><span data-stu-id="1a7bd-110">If, however, asynchronous activities (such as activities that derive from <xref:System.Activities.AsyncCodeActivity>) or messaging activities are used, control will pass to the next branch while the child activity waits for its message to be received or its asynchronous work to be completed.</span></span>
