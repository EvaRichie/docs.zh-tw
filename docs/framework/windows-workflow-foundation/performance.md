### <a name="tracking"></a><span data-ttu-id="8c718-490">追蹤</span><span class="sxs-lookup"><span data-stu-id="8c718-490">Tracking</span></span>
 <span data-ttu-id="8c718-491">工作流程追蹤可以用來追蹤工作流程的進度。</span><span class="sxs-lookup"><span data-stu-id="8c718-491">Workflow tracking can be used to track the progress of a workflow.</span></span>  <span data-ttu-id="8c718-492">追蹤事件中包含的資訊是由追蹤設定檔所決定。</span><span class="sxs-lookup"><span data-stu-id="8c718-492">The information that is included in the tracking events is determined by a tracking profile.</span></span>  <span data-ttu-id="8c718-493">追蹤設定檔越複雜，追蹤所耗用的資源就愈多。</span><span class="sxs-lookup"><span data-stu-id="8c718-493">The more complex the tracking profile, the more expensive tracking becomes.</span></span>

 <span data-ttu-id="8c718-494">WF3 隨附 SQL 架構追蹤服務。</span><span class="sxs-lookup"><span data-stu-id="8c718-494">WF3 shipped with a SQL-based tracking service.</span></span>  <span data-ttu-id="8c718-495">這項服務可以在批次和非批次模式下運作。</span><span class="sxs-lookup"><span data-stu-id="8c718-495">This service could work in batched and non-batched modes.</span></span>  <span data-ttu-id="8c718-496">在非批次模式中，追蹤事件會直接寫入資料庫。</span><span class="sxs-lookup"><span data-stu-id="8c718-496">In non-batched mode, tracking events are written directly to the database.</span></span>  <span data-ttu-id="8c718-497">在批次模式中，追蹤事件會收集到與工作流程執行個體狀態相同的批次中。</span><span class="sxs-lookup"><span data-stu-id="8c718-497">In batched mode, tracking events are collected into the same batch as the workflow instance state.</span></span>  <span data-ttu-id="8c718-498">針對最廣泛的工作流程設計，批次模式可提供最佳效能。</span><span class="sxs-lookup"><span data-stu-id="8c718-498">The batched mode has the best performance for the widest range of workflow designs.</span></span>  <span data-ttu-id="8c718-499">不過，如果工作流程執行許多活動而未保存，而且這些活動都會加以追蹤，則批次處理可能對效能產生負面影響。</span><span class="sxs-lookup"><span data-stu-id="8c718-499">However, batching can have a negative performance impact if the workflow runs many activities without persisting and those activities are tracked.</span></span>  <span data-ttu-id="8c718-500">這種情況常發生在迴圈中，而避免這種情況發生的最佳方式，就是設計大型迴圈來包含保存點。</span><span class="sxs-lookup"><span data-stu-id="8c718-500">This would commonly happen in loops and the best way to avoid this scenario is to design large loops to contain a persistence point.</span></span>  <span data-ttu-id="8c718-501">在迴圈中導入保存點同樣可能對效能造成負面影響，因此務必計算各項的成本並找出平衡。</span><span class="sxs-lookup"><span data-stu-id="8c718-501">Introducing a persistence point into a loop can negatively affect performance as well so it is important to measure the costs of each and come up with a balance.</span></span>
