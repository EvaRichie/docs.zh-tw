## <a name="multiple-base-addresses"></a><span data-ttu-id="2c9d3-110">多個基底位址</span><span class="sxs-lookup"><span data-stu-id="2c9d3-110">Multiple Base Addresses</span></span>  
 <span data-ttu-id="2c9d3-111">這項功能僅適用于裝載于 IIS 的 WCF 服務。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-111">This feature is only available to WCF services that are hosted under IIS.</span></span> <span data-ttu-id="2c9d3-112">預設不會啟用此功能。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-112">This feature is not enabled by default.</span></span> <span data-ttu-id="2c9d3-113">若要啟用它，您必須將 `multipleSiteBindingsEnabled` 屬性加入 Web.config 檔中的 <`serviceHostingEnvironment`> 元素，並將它設定為 `true` ，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-113">To enable it you must add the `multipleSiteBindingsEnabled` attribute to the <`serviceHostingEnvironment`> element in your Web.config file and set it to `true`, as shown in the following example.</span></span>  
  
```xml  
<serviceHostingEnvironment multipleSiteBindingsEnabled="true"/>  
```  
  
 <span data-ttu-id="2c9d3-114">在 IIS 下裝載 WCF 服務時，IIS 會根據包含應用程式之虛擬目錄的 URI 為您建立一個基底位址。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-114">When hosting a WCF service under IIS, IIS creates one base address for you based on the URI to the virtual directory that contains the application.</span></span> <span data-ttu-id="2c9d3-115">您可以使用 Internet Information Services 管理員，加入其他使用相同通訊協定的基底位址，以便在網站中加入一個或多個繫結。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-115">You can add additional base addresses that use the same protocol by using Internet Information Services Manager to add one or more bindings to your Web site.</span></span> <span data-ttu-id="2c9d3-116">為每個繫結指定通訊協定 (HTTP 或 HTTPS)、IP 位址、連接埠和主機名稱。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-116">For each binding specify a protocol (HTTP or HTTPS), an IP address, a port, and a host name.</span></span> <span data-ttu-id="2c9d3-117">如需有關使用 Internet Information Services 管理員的詳細資訊，請參閱 [Iis 管理員 (iis 7) ](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc753842(v=ws.10))。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-117">For more information about using Internet Information Services Manager, see [IIS Manager (IIS 7)](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc753842(v=ws.10)).</span></span> <span data-ttu-id="2c9d3-118">如需將系結加入至網站的詳細資訊，請參閱 [建立網站 (IIS 7) ](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc772350(v=ws.10))</span><span class="sxs-lookup"><span data-stu-id="2c9d3-118">For more information about adding bindings to a site, see [Create a Web Site (IIS 7)](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc772350(v=ws.10))</span></span>  
  
 <span data-ttu-id="2c9d3-119">針對相同網站指定多個基底位址，會影響 WCF 說明頁面的內容、匯入架構，以及服務所產生的 WSDL/MEX 資訊。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-119">Specifying multiple base addresses for the same site affects the content of the WCF Help page, importing schema, and the WSDL/MEX information generated by the service.</span></span> <span data-ttu-id="2c9d3-120">WCF 說明頁面會顯示用來產生可與服務通訊之 WCF 用戶端的命令列。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-120">The WCF Help page displays the command line to use to generate a WCF client that can communicate with the service.</span></span> <span data-ttu-id="2c9d3-121">此命令列僅包含在 IIS 繫結中指定的第一個網站位址。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-121">This command line contains only the first address specified in the IIS binding for the Web site.</span></span> <span data-ttu-id="2c9d3-122">同樣地，在匯入結構時，只會使用在 IIS 繫結中指定的第一個基底位址。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-122">Similarly when importing schema, only the first base address specified in the IIS binding is used.</span></span> <span data-ttu-id="2c9d3-123">WSDL 和 MEX 資料包含所有在 IIS 繫結中指定的基底位址。</span><span class="sxs-lookup"><span data-stu-id="2c9d3-123">WSDL and MEX data contain all the base addresses specified in the IIS bindings.</span></span>
