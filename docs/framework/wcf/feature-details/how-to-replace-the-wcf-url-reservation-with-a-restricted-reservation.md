 <span data-ttu-id="9aec2-109">作為預設作業系統設定的一部分，Windows Communication Foundation (WCF) 會為埠80建立可全域存取的保留，讓所有使用者都能執行使用雙重 HTTP 系結進行雙工通訊的應用程式。</span><span class="sxs-lookup"><span data-stu-id="9aec2-109">As part of the default operating system configuration, Windows Communication Foundation (WCF) creates a globally accessible reservation for port 80 to enable all users to run applications that use a dual HTTP binding for duplex communication.</span></span> <span data-ttu-id="9aec2-110">此保留項目的 ACL 是攻所有人使用，因此，系統管理員無法明確地允許或不允許在一個 URL 或一組 URL 上接聽的權限。</span><span class="sxs-lookup"><span data-stu-id="9aec2-110">Because the ACL on this reservation is for everyone, administrators cannot explicitly allow or disallow permission to listen on a URL or set of URLs.</span></span> <span data-ttu-id="9aec2-111">本主題說明如何刪除此保留項目，以及如何使用受限的 ACL 重新建立保留項目。</span><span class="sxs-lookup"><span data-stu-id="9aec2-111">This topic explains how to delete this reservation and how to re-create the reservation with a restricted ACL.</span></span>

```output
            SDDL: D:(A;;GX;;;WD)  
```

 <span data-ttu-id="9aec2-114">保留專案是由 WCF 應用程式使用 HTTP 雙重系結進行雙工通訊時所使用的 URL 範本所組成。</span><span class="sxs-lookup"><span data-stu-id="9aec2-114">The reservation consists of a URL template used when a WCF application is using an HTTP dual binding for duplex communication.</span></span> <span data-ttu-id="9aec2-115">當透過 HTTP 雙重系結進行通訊時，此表單的 Url 會用於 WCF 服務，以將訊息傳送回 WCF 用戶端。</span><span class="sxs-lookup"><span data-stu-id="9aec2-115">URLs of this form are used for a WCF service to send messages back to the WCF client when communicating over a HTTP dual binding.</span></span> <span data-ttu-id="9aec2-116">所有人都會得到在 URL 上接聽的權限，但沒有委派接聽其他處理序的權限。</span><span class="sxs-lookup"><span data-stu-id="9aec2-116">Everyone is given permission to listen on the URL but not to delegate listening to another process.</span></span> <span data-ttu-id="9aec2-117">最後，ACL 會以 Security Descriptor Definition Language (SSDL) 描述。</span><span class="sxs-lookup"><span data-stu-id="9aec2-117">Finally, the ACL is described in Security Descriptor Definition Language (SSDL).</span></span> <span data-ttu-id="9aec2-118">如需 SSDL 的詳細資訊，請參閱 [ssdl](/windows/win32/secauthz/security-descriptor-definition-language)</span><span class="sxs-lookup"><span data-stu-id="9aec2-118">For more information about SSDL, see [SSDL](/windows/win32/secauthz/security-descriptor-definition-language)</span></span>
