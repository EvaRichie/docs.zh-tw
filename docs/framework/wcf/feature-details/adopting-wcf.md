# <a name="adopt-windows-communication-foundation"></a><span data-ttu-id="f9232-102">採用 Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="f9232-102">Adopt Windows Communication Foundation</span></span>

<span data-ttu-id="f9232-103">您可以選擇使用 Windows Communication Foundation （WCF）進行新的開發，同時繼續維護使用 ASP.NET 開發的現有應用程式。</span><span class="sxs-lookup"><span data-stu-id="f9232-103">You can choose to use Windows Communication Foundation (WCF) for new development while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="f9232-104">因為 WCF 的目的是為了方便與使用 .NET Framework 在任何案例中建立的應用程式進行通訊，所以它可以做為解決各種不同軟體通訊問題的標準工具，使其無法 ASP.NET。</span><span class="sxs-lookup"><span data-stu-id="f9232-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="f9232-105">新的 WCF 應用程式可以部署在與現有 ASP.NET Web 服務相同的電腦上。</span><span class="sxs-lookup"><span data-stu-id="f9232-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="f9232-106">如果現有的 ASP.NET Web 服務使用2.0 版之前的 .NET Framework 版本，您可以使用 ASP.NET IIS 註冊工具，選擇性地將 .NET Framework 2.0 部署到要裝載新 WCF 應用程式的 IIS 應用程式。</span><span class="sxs-lookup"><span data-stu-id="f9232-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="f9232-107">該工具記載于[ASP.NET Iis 註冊工具（Aspnet_regiis .exe）](https://docs.microsoft.com/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90))，並在 iis 6.0 管理主控台內建使用者介面。</span><span class="sxs-lookup"><span data-stu-id="f9232-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://docs.microsoft.com/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90)), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="f9232-108">WCF 可以藉由將設定為在 ASP.NET 相容性模式中執行的 WCF 服務新增至 IIS 中現有的 ASP.NET Web 服務應用程式，來將新功能加入至現有的 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="f9232-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="f9232-109">由於 ASP.NET 的相容性模式，新 WCF 服務的程式碼可以使用類別來存取和更新與預先存在的 ASP.NET 碼相同的應用程式狀態資訊 <xref:System.Web.HttpContext> 。</span><span class="sxs-lookup"><span data-stu-id="f9232-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="f9232-110">應用程式也可以共用相同的類別庫。</span><span class="sxs-lookup"><span data-stu-id="f9232-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="f9232-111">WCF 用戶端可以使用 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="f9232-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="f9232-112">使用設定的 WCF 服務 <xref:System.ServiceModel.BasicHttpBinding> 可供 ASP.NET Web 服務用戶端使用。</span><span class="sxs-lookup"><span data-stu-id="f9232-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="f9232-113">ASP.NET Web 服務可以與 WCF 應用程式並存，而 WCF 甚至可以用來將功能新增至現有的 ASP.NET Web 服務。</span><span class="sxs-lookup"><span data-stu-id="f9232-113">ASP.NET Web services can coexist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="f9232-114">假設所有這些方法都可以搭配使用 WCF 和 ASP.NET Web 服務，則只有在需要 WCF 所提供的功能，而不是 ASP.NET Web 服務時，您才可以將 ASP.NET Web 服務遷移至 WCF。</span><span class="sxs-lookup"><span data-stu-id="f9232-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="f9232-115">即使在少數情況下，將程式碼從一項技術遷移至另一種技術並不是正確的方法。</span><span class="sxs-lookup"><span data-stu-id="f9232-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="f9232-116">採用新技術的原因，是為了滿足先前技術無法滿足的新需求，而在此情況下，正確的做法是設計新的解決方案，以符合新擴充的需求集。</span><span class="sxs-lookup"><span data-stu-id="f9232-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly expanded set of requirements.</span></span> <span data-ttu-id="f9232-117">新設計的優點在於採用現有系統的經驗，以及設計該系統以來所獲取的智慧。</span><span class="sxs-lookup"><span data-stu-id="f9232-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="f9232-118">新設計也可以使用新技術的完整功能，而不必在新平台上重現舊設計。</span><span class="sxs-lookup"><span data-stu-id="f9232-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="f9232-119">在建立新設計的關鍵元素原型之後，在新的系統內重複使用程式碼會變得更容易。</span><span class="sxs-lookup"><span data-stu-id="f9232-119">After prototyping key elements of the new design, it becomes easier to reuse code from the existing system within the new one.</span></span>
