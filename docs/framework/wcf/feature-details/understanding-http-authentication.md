## <a name="choosing-an-authentication-scheme"></a><span data-ttu-id="9451e-146">選擇驗證配置</span><span class="sxs-lookup"><span data-stu-id="9451e-146">Choosing an Authentication Scheme</span></span>  
 <span data-ttu-id="9451e-147">為 HTTP 伺服器選擇潛在驗證配置時，必須考量一些項目，包括：</span><span class="sxs-lookup"><span data-stu-id="9451e-147">When selecting the potential authentication schemes for an HTTP server, a few items to consider include the following:</span></span>  
  
- <span data-ttu-id="9451e-148">考量資源是否需要保護。</span><span class="sxs-lookup"><span data-stu-id="9451e-148">Consider whether the resource needs to be protected.</span></span> <span data-ttu-id="9451e-149">使用 HTTP 驗證需要傳輸更多資料，並且可限制與用戶端的互通性。</span><span class="sxs-lookup"><span data-stu-id="9451e-149">Using HTTP authentication requires transmitting more data and can limit interoperability with clients.</span></span> <span data-ttu-id="9451e-150">允許匿名存取不需要保護的資源。</span><span class="sxs-lookup"><span data-stu-id="9451e-150">Allow anonymous access to resources that do not need to be protected.</span></span>  
  
- <span data-ttu-id="9451e-151">如果資源需要保護，則考慮何種驗證配置提供所需的安全性等級。</span><span class="sxs-lookup"><span data-stu-id="9451e-151">If the resource needs to be protected, consider which authentication schemes provide the required level of security.</span></span> <span data-ttu-id="9451e-152">這裡所討論的驗證配置，最低標準是基礎驗證。</span><span class="sxs-lookup"><span data-stu-id="9451e-152">The weakest standard authentication scheme discussed here is Basic authentication.</span></span> <span data-ttu-id="9451e-153">基礎驗證不會保護使用者的認證資訊。</span><span class="sxs-lookup"><span data-stu-id="9451e-153">Basic authentication does not protect the user's credentials.</span></span> <span data-ttu-id="9451e-154">最高標準驗證配置為交涉驗證，也就是 Kerberos 通訊協定。</span><span class="sxs-lookup"><span data-stu-id="9451e-154">The strongest standard authentication scheme is Negotiate authentication, resulting in the Kerberos protocol.</span></span>
