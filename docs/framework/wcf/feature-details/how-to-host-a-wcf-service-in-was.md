# <a name="how-to-host-a-wcf-service-in-was"></a><span data-ttu-id="c1de4-102">HOW TO：在 WAS 中裝載 WCF 服務</span><span class="sxs-lookup"><span data-stu-id="c1de4-102">How to: Host a WCF Service in WAS</span></span>
<span data-ttu-id="c1de4-103">本主題概述建立 Windows 進程啟用服務（也稱為 WAS）主控的 Windows Communication Foundation （WCF）服務所需的基本步驟。</span><span class="sxs-lookup"><span data-stu-id="c1de4-103">This topic outlines the basic steps required to create a Windows Process Activation Services (also known as WAS) hosted Windows Communication Foundation (WCF) service.</span></span> <span data-ttu-id="c1de4-104">WAS 是新的處理序啟用服務，其為一般化的 Internet Information Services (IIS) 功能，與非 HTTP 傳輸通訊協定搭配使用。</span><span class="sxs-lookup"><span data-stu-id="c1de4-104">WAS is the new process activation service that is a generalization of Internet Information Services (IIS) features that work with non-HTTP transport protocols.</span></span> <span data-ttu-id="c1de4-105">WCF 使用接聽程式介面卡介面，來傳達透過 WCF 支援的非 HTTP 通訊協定（例如 TCP、具名管道和訊息佇列）接收的啟用要求。</span><span class="sxs-lookup"><span data-stu-id="c1de4-105">WCF uses the listener adapter interface to communicate activation requests that are received over the non-HTTP protocols supported by WCF, such as TCP, named pipes, and Message Queuing.</span></span>  
  
 <span data-ttu-id="c1de4-106">這個裝載選項要求 WAS 啟用元件必須正確安裝與設定，但不要求將任何裝載程式碼撰寫為應用程式的一部分。</span><span class="sxs-lookup"><span data-stu-id="c1de4-106">This hosting option requires that WAS activation components are properly installed and configured, but it does not require any hosting code to be written as part of the application.</span></span> <span data-ttu-id="c1de4-107">如需安裝和設定 WAS 的詳細資訊，請參閱[如何：安裝和設定 WCF 啟用元件](how-to-install-and-configure-wcf-activation-components.md)。</span><span class="sxs-lookup"><span data-stu-id="c1de4-107">For more information about installing and configuring WAS, see [How to: Install and Configure WCF Activation Components](how-to-install-and-configure-wcf-activation-components.md).</span></span>  
  
> [!WARNING]
> <span data-ttu-id="c1de4-108">如果 Web 伺服器的要求處理管線設定為「傳統」模式，就不支援 WAS 啟動。</span><span class="sxs-lookup"><span data-stu-id="c1de4-108">WAS activation is not supported if the web server’s request processing pipeline is set to Classic mode.</span></span> <span data-ttu-id="c1de4-109">若要使用 WAS 啟動，Web 伺服器的要求處理管線就必須設定為「整合」模式。</span><span class="sxs-lookup"><span data-stu-id="c1de4-109">The web server’s request processing pipeline must be set to Integrated mode if WAS activation is to be used.</span></span>  
  
 <span data-ttu-id="c1de4-110">在 WAS 中裝載 WCF 服務時，會以一般方式來使用標準系結。</span><span class="sxs-lookup"><span data-stu-id="c1de4-110">When a WCF service is hosted in WAS, the standard bindings are used in the usual way.</span></span> <span data-ttu-id="c1de4-111">但是，當透過 <xref:System.ServiceModel.NetTcpBinding> 和 <xref:System.ServiceModel.NetNamedPipeBinding> 來設定 WAS 裝載的服務時，就必須滿足下列限制。</span><span class="sxs-lookup"><span data-stu-id="c1de4-111">However, when using the <xref:System.ServiceModel.NetTcpBinding> and the <xref:System.ServiceModel.NetNamedPipeBinding> to configure a WAS-hosted service, a constraint must be satisfied.</span></span> <span data-ttu-id="c1de4-112">當不同的端點使用同一個傳輸，繫結設定必須符合下列七項屬性：</span><span class="sxs-lookup"><span data-stu-id="c1de4-112">When different endpoints use the same transport, the binding settings have to match on the following seven properties:</span></span>  
  
- <span data-ttu-id="c1de4-113">ConnectionBufferSize</span><span class="sxs-lookup"><span data-stu-id="c1de4-113">ConnectionBufferSize</span></span>  
  
- <span data-ttu-id="c1de4-114">ChannelInitializationTimeout</span><span class="sxs-lookup"><span data-stu-id="c1de4-114">ChannelInitializationTimeout</span></span>  
  
- <span data-ttu-id="c1de4-115">MaxPendingConnections</span><span class="sxs-lookup"><span data-stu-id="c1de4-115">MaxPendingConnections</span></span>  
  
- <span data-ttu-id="c1de4-116">MaxOutputDelay</span><span class="sxs-lookup"><span data-stu-id="c1de4-116">MaxOutputDelay</span></span>  
  
- <span data-ttu-id="c1de4-117">MaxPendingAccepts</span><span class="sxs-lookup"><span data-stu-id="c1de4-117">MaxPendingAccepts</span></span>  
  
- <span data-ttu-id="c1de4-118">ConnectionPoolSettings.IdleTimeout</span><span class="sxs-lookup"><span data-stu-id="c1de4-118">ConnectionPoolSettings.IdleTimeout</span></span>  
  
- <span data-ttu-id="c1de4-119">ConnectionPoolSettings.MaxOutboundConnectionsPerEndpoint</span><span class="sxs-lookup"><span data-stu-id="c1de4-119">ConnectionPoolSettings.MaxOutboundConnectionsPerEndpoint</span></span>  
  
 <span data-ttu-id="c1de4-120">否則，先初始化的端點一律直接決定這些屬性的值，而稍後新增的端點則會在這些屬性值未符合上述設定值時擲回 <xref:System.ServiceModel.ServiceActivationException>。</span><span class="sxs-lookup"><span data-stu-id="c1de4-120">Otherwise, the endpoint that is initialized first always determines the values of these properties, and endpoints added later throw a <xref:System.ServiceModel.ServiceActivationException> if they do not match those settings.</span></span>  
  
 <span data-ttu-id="c1de4-121">如需此範例的來源複本，請參閱[TCP 啟用](../samples/tcp-activation.md)。</span><span class="sxs-lookup"><span data-stu-id="c1de4-121">For the source copy of this example, see [TCP Activation](../samples/tcp-activation.md).</span></span>
