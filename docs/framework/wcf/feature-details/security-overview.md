 <span data-ttu-id="36c6c-173">傳輸安全性涵蓋三個主要的安全性功能：完整性、機密性和驗證。</span><span class="sxs-lookup"><span data-stu-id="36c6c-173">Transfer security encompasses three major security functions: integrity, confidentiality, and authentication.</span></span> <span data-ttu-id="36c6c-174">*完整性*是偵測訊息是否已遭修改的功能。</span><span class="sxs-lookup"><span data-stu-id="36c6c-174">*Integrity* is the ability to detect whether a message has been tampered with.</span></span> <span data-ttu-id="36c6c-175">*機密性*是指能夠讓非預期的收件者以外的任何人都能讀取訊息;這是透過密碼編譯來達成。</span><span class="sxs-lookup"><span data-stu-id="36c6c-175">*Confidentiality* is the ability to keep a message unreadable by anyone other than the intended recipient; this is achieved through cryptography.</span></span> <span data-ttu-id="36c6c-176">*驗證*是確認宣告的身分識別的能力。</span><span class="sxs-lookup"><span data-stu-id="36c6c-176">*Authentication* is the ability to verify a claimed identity.</span></span> <span data-ttu-id="36c6c-177">這三個功能合起來有助於確保訊息從某一點安全送達另一點。</span><span class="sxs-lookup"><span data-stu-id="36c6c-177">Together, these three functions help to ensure that messages securely arrive from one point to another.</span></span>  
  
#### <a name="transport-and-message-security-modes"></a><span data-ttu-id="36c6c-178">傳輸和訊息安全性模式</span><span class="sxs-lookup"><span data-stu-id="36c6c-178">Transport and Message Security Modes</span></span>  
 <span data-ttu-id="36c6c-179">有兩個主要的機制可用來在 WCF 中執行傳輸安全性：*傳輸*安全性模式和*訊息*安全性模式。</span><span class="sxs-lookup"><span data-stu-id="36c6c-179">Two main mechanisms are used to implement transfer security in WCF: *transport* security mode and *message* security mode.</span></span>  
  
- <span data-ttu-id="36c6c-180">*傳輸安全性模式*會使用傳輸層級通訊協定（例如 HTTPS）來達到傳輸安全性。</span><span class="sxs-lookup"><span data-stu-id="36c6c-180">*Transport security mode* uses a transport-level protocol, such as HTTPS, to achieve transfer security.</span></span> <span data-ttu-id="36c6c-181">傳輸模式的優點是已廣為採用、可使用於許多平台，且運算較不複雜。</span><span class="sxs-lookup"><span data-stu-id="36c6c-181">Transport mode has the advantage of being widely adopted, available on many platforms, and less computationally complex.</span></span> <span data-ttu-id="36c6c-182">但缺點是只確保訊息點對點安全性。</span><span class="sxs-lookup"><span data-stu-id="36c6c-182">However, it has the disadvantage of securing messages only from point-to-point.</span></span>  
  
- <span data-ttu-id="36c6c-183">另一方面，*訊息安全性模式*會使用 WS-security （和其他規格）來執行傳輸安全性。</span><span class="sxs-lookup"><span data-stu-id="36c6c-183">*Message security mode*, on the other hand, uses WS-Security (and other specifications) to implement transfer security.</span></span> <span data-ttu-id="36c6c-184">因為訊息安全性是直接套用至 SOAP 訊息，並且會與應用程式資料包含在 SOAP 封套中，所以它的優點是與傳輸通訊協定無關、更具擴充性和確保端對端安全性 (相對於點對點)。因為必須處理 SOAP 訊息的 XML 特性，它的缺點是比傳輸安全性慢上好幾倍。</span><span class="sxs-lookup"><span data-stu-id="36c6c-184">Because the message security is applied directly to the SOAP messages and is contained inside the SOAP envelopes, together with the application data, it has the advantage of being transport protocol-independent, more extensible, and ensuring end-to-end security (versus point-to-point); it has the disadvantage of being several times slower than transport security mode because it has to deal with the XML nature of the SOAP messages.</span></span>  
  
 <span data-ttu-id="36c6c-185">如需這些差異的詳細資訊，請參閱[保護服務和用戶端](securing-services-and-clients.md)。</span><span class="sxs-lookup"><span data-stu-id="36c6c-185">For more information about these differences, see [Securing Services and Clients](securing-services-and-clients.md).</span></span>  
  
 <span data-ttu-id="36c6c-186">第三個安全性模式同時使用前兩個模式，且結合兩者的優點。</span><span class="sxs-lookup"><span data-stu-id="36c6c-186">A third security mode uses both previous modes and brings advantages of both.</span></span> <span data-ttu-id="36c6c-187">這個模式稱為 `TransportWithMessageCredential`。</span><span class="sxs-lookup"><span data-stu-id="36c6c-187">This mode is called `TransportWithMessageCredential`.</span></span> <span data-ttu-id="36c6c-188">在這個模式中，訊息安全性會用來驗證用戶端，而傳輸安全性則會用來驗證伺服器及提供訊息機密性和完整性。</span><span class="sxs-lookup"><span data-stu-id="36c6c-188">In this mode, message security is used to authenticate the client and transport security is used to authenticate the server and provide message confidentiality and integrity.</span></span> <span data-ttu-id="36c6c-189">`TransportWithMessageCredential` 安全性模式因此受惠，幾乎與傳輸安全性模式一樣快速，並且以訊息安全性的相同方式提供用戶端驗證擴充性。</span><span class="sxs-lookup"><span data-stu-id="36c6c-189">Thanks to this, the `TransportWithMessageCredential` security mode is almost as fast as transport security mode and provides client authentication extensibility in the same way as message security.</span></span> <span data-ttu-id="36c6c-190">不過，不像訊息安全性模式，它不會提供完整的端對端安全性。</span><span class="sxs-lookup"><span data-stu-id="36c6c-190">However, unlike message security mode, it does not provide complete end-to-end security.</span></span>  
  
### <a name="access-control"></a><span data-ttu-id="36c6c-191">存取控制</span><span class="sxs-lookup"><span data-stu-id="36c6c-191">Access Control</span></span>  
 <span data-ttu-id="36c6c-192">*存取控制*也稱為授權。</span><span class="sxs-lookup"><span data-stu-id="36c6c-192">*Access control* is also known as authorization.</span></span> <span data-ttu-id="36c6c-193">*授權*可讓不同的使用者有不同的許可權來查看資料。</span><span class="sxs-lookup"><span data-stu-id="36c6c-193">*Authorization* allows different users to have different privileges to view data.</span></span> <span data-ttu-id="36c6c-194">例如，因為公司的人力資源檔案包含敏感的員工資料，所以只有經理才能檢視員工資料。</span><span class="sxs-lookup"><span data-stu-id="36c6c-194">For example, because a company's human resources files contain sensitive employee data, only managers are allowed to view employee data.</span></span> <span data-ttu-id="36c6c-195">而且，經理只能檢視其直屬員工的資料。</span><span class="sxs-lookup"><span data-stu-id="36c6c-195">Further, managers can view only data for their direct reports.</span></span> <span data-ttu-id="36c6c-196">在這種情況下，存取控制是根據角色 (「經理」) 和經理的特定身分識別 (以防止某個經理查看另一個經理的員工記錄)。</span><span class="sxs-lookup"><span data-stu-id="36c6c-196">In this case, access control is based on both the role ("manager") as well as the specific identity of the manager (to prevent one manager from looking at another manager's employee records).</span></span>  
  
 <span data-ttu-id="36c6c-197">在 WCF 中，存取控制功能是透過與 common language runtime （CLR）的整合 <xref:System.Security.Permissions.PrincipalPermissionAttribute> ，以及透過一組稱為「身分*識別模型*」的 api 來提供。</span><span class="sxs-lookup"><span data-stu-id="36c6c-197">In WCF, access control features are provided through integration with the common language runtime (CLR) <xref:System.Security.Permissions.PrincipalPermissionAttribute> and through a set of APIs known as the *identity model*.</span></span> <span data-ttu-id="36c6c-198">如需存取控制和宣告型授權的詳細資訊，請參閱[擴充安全性](../extending/extending-security.md)。</span><span class="sxs-lookup"><span data-stu-id="36c6c-198">For details about access control and claims-based authorization, see [Extending Security](../extending/extending-security.md).</span></span>  
  
### <a name="auditing"></a><span data-ttu-id="36c6c-199">稽核</span><span class="sxs-lookup"><span data-stu-id="36c6c-199">Auditing</span></span>
