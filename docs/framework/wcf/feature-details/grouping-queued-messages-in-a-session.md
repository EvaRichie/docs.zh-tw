# <a name="grouping-queued-messages-in-a-session"></a><span data-ttu-id="ce5a8-102">在工作階段中群組佇列訊息</span><span class="sxs-lookup"><span data-stu-id="ce5a8-102">Grouping Queued Messages in a Session</span></span>
<span data-ttu-id="ce5a8-103">Windows Communication Foundation （WCF）會提供一個會話，讓您將一組相關的訊息群組在一起，以供單一接收應用程式處理。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-103">Windows Communication Foundation (WCF) provides a session that allows you to group a set of related messages together for processing by a single receiving application.</span></span> <span data-ttu-id="ce5a8-104">本身是工作階段一部分的訊息，也必須是屬於相同的異動。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-104">Messages that are part of a session must be part of the same transaction.</span></span> <span data-ttu-id="ce5a8-105">由於所有訊息都屬於相同的異動，所以如果有任何一個訊息無法進行處理，就會回復整個工作階段。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-105">Because all messages are part of the same transaction, if one message fails to be processed the entire session is rolled back.</span></span> <span data-ttu-id="ce5a8-106">工作階段對於寄不出的信件佇列與有害佇列，會採取類似行為。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-106">Sessions have similar behaviors with regard to dead-letter queues and poison queues.</span></span> <span data-ttu-id="ce5a8-107">針對工作階段在佇列繫結上設定的存留時間 (TTL) 屬性會完整地套用到工作階段。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-107">The Time to Live (TTL) property set on a queued binding configured for sessions is applied to the session as a whole.</span></span> <span data-ttu-id="ce5a8-108">如果工作階段中只有部分訊息在 TTL 到期之前傳送出去，則整個工作階段將置於寄不出的信件佇列中。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-108">If only some of the messages in the session are sent before the TTL expires, the entire session is placed in the dead-letter queue.</span></span> <span data-ttu-id="ce5a8-109">同樣地，當工作階段中的訊息無法從應用程式佇列傳送到應用程式的話，則整個工作階段將置於有害佇列 (如果有的話)。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-109">Similarly, when messages in a session fail to be sent to an application from the application queue, the entire session is placed in the poison queue (if available).</span></span>

## <a name="procedures"></a><span data-ttu-id="ce5a8-119">程序</span><span class="sxs-lookup"><span data-stu-id="ce5a8-119">Procedures</span></span>  
  
#### <a name="to-set-up-a-service-contract-to-use-sessions"></a><span data-ttu-id="ce5a8-120">若要設定服務合約使用工作階段</span><span class="sxs-lookup"><span data-stu-id="ce5a8-120">To set up a service contract to use sessions</span></span>  
  
1. <span data-ttu-id="ce5a8-121">定義需要工作階段的服務合約。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-121">Define a service contract that requires a session.</span></span> <span data-ttu-id="ce5a8-122">使用屬性來執行此動作， <xref:System.ServiceModel.ServiceContractAttribute> 方法是指定：</span><span class="sxs-lookup"><span data-stu-id="ce5a8-122">Do this with the <xref:System.ServiceModel.ServiceContractAttribute> attribute by specifying:</span></span>  
  
    ```csharp

    ```

3. <span data-ttu-id="ce5a8-125">實作服務合約並指定 <xref:System.ServiceModel.ServiceBehaviorAttribute.InstanceContextMode> 的 <xref:System.ServiceModel.InstanceContextMode.PerSession?displayProperty=nameWithType>。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-125">Implement the service contract and specify an <xref:System.ServiceModel.ServiceBehaviorAttribute.InstanceContextMode> of <xref:System.ServiceModel.InstanceContextMode.PerSession?displayProperty=nameWithType>.</span></span> <span data-ttu-id="ce5a8-126">這樣只會針對每個工作階段產生服務一次。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-126">This instantiates the service only once for each session.</span></span>

5. <span data-ttu-id="ce5a8-130">設定使用系統提供之 `NetMsmqBinding` 繫結的端點。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-130">Configure an endpoint that uses the system-provided `NetMsmqBinding` binding.</span></span>  
  
6. <span data-ttu-id="ce5a8-131">使用 <xref:System.Messaging> 來建立交易式佇列。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-131">Create a transactional queue using <xref:System.Messaging>.</span></span> <span data-ttu-id="ce5a8-132">您也可以使用訊息佇列 (MSMQ) 或 MMC 來建立佇列。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-132">You can also create the queue by using Message Queuing (MSMQ) or MMC.</span></span> <span data-ttu-id="ce5a8-133">如果您要這麼做，請建立異動式佇列。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-133">If you do, create a transactional queue.</span></span>  
  
7. <span data-ttu-id="ce5a8-134">請使用 <xref:System.ServiceModel.ServiceHost> 來建立服務的服務主機。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-134">Create a service host for the service by using <xref:System.ServiceModel.ServiceHost>.</span></span>  
  
8. <span data-ttu-id="ce5a8-135">開啟服務主機來提供服務。</span><span class="sxs-lookup"><span data-stu-id="ce5a8-135">Open the service host to make the service available.</span></span>
