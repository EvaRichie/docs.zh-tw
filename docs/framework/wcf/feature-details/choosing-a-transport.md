 <span data-ttu-id="4d71e-121">本節將說明選擇這三種主要傳輸其中之一的主要原因，包括進行選擇的詳細決策圖表。</span><span class="sxs-lookup"><span data-stu-id="4d71e-121">This section describes the main reasons for choosing any one of the three main transports, including a detailed decision chart for choosing among them.</span></span>  
  
### <a name="when-to-use-http-transport"></a><span data-ttu-id="4d71e-122">使用 HTTP 傳輸的時機</span><span class="sxs-lookup"><span data-stu-id="4d71e-122">When to Use HTTP Transport</span></span>  

 <span data-ttu-id="4d71e-123">HTTP 是用戶端和伺服器之間的要求/回應通訊協定。</span><span class="sxs-lookup"><span data-stu-id="4d71e-123">HTTP is a request/response protocol between clients and servers.</span></span> <span data-ttu-id="4d71e-124">最常用的應用程式是由使用 Web 伺服器進行通訊的 Web 瀏覽器用戶端所組成的。</span><span class="sxs-lookup"><span data-stu-id="4d71e-124">The most common application consists of Web-browser clients that communicate with a Web server.</span></span> <span data-ttu-id="4d71e-125">用戶端會將要求傳送至伺服器，接聽用戶端要求訊息。</span><span class="sxs-lookup"><span data-stu-id="4d71e-125">The client sends a request to a server, which listens for client request messages.</span></span> <span data-ttu-id="4d71e-126">當伺服器收到要求時，會傳回一個回應，其中包含要求的狀態。</span><span class="sxs-lookup"><span data-stu-id="4d71e-126">When the server receives a request, it returns a response, which contains the status of the request.</span></span> <span data-ttu-id="4d71e-127">如果成功，會傳回選擇性資料 (例如網頁、錯誤訊息或其他資訊)。</span><span class="sxs-lookup"><span data-stu-id="4d71e-127">If successful, optional data, such as a Web page, an error message, or other information is returned.</span></span> <span data-ttu-id="4d71e-128">如需有關 HTTP 通訊協定的詳細資訊，請參閱 [Http 超文字傳輸通訊協定](https://www.w3.org/Protocols/)。</span><span class="sxs-lookup"><span data-stu-id="4d71e-128">For more information about the HTTP protocol, see [HTTP - Hypertext Transfer Protocol](https://www.w3.org/Protocols/).</span></span>

### <a name="when-to-use-the-tcp-transport"></a><span data-ttu-id="4d71e-134">使用 TCP 傳輸的時機</span><span class="sxs-lookup"><span data-stu-id="4d71e-134">When to Use the TCP Transport</span></span>  

 <span data-ttu-id="4d71e-135">TCP 是以連線為基礎、以資料流為導向的傳遞服務，具有端對端錯誤偵測和更正。</span><span class="sxs-lookup"><span data-stu-id="4d71e-135">TCP is a connection-based, stream-oriented delivery service with end-to-end error detection and correction.</span></span> <span data-ttu-id="4d71e-136">以 *連接為基礎* 表示在交換資料之前，會先建立主機之間的通訊會話。</span><span class="sxs-lookup"><span data-stu-id="4d71e-136">*Connection-based* means that a communication session between hosts is established before exchanging data.</span></span> <span data-ttu-id="4d71e-137">主機是由邏輯 IP 位址所識別的 TCP/IP 網路上的裝置。</span><span class="sxs-lookup"><span data-stu-id="4d71e-137">A host is any device on a TCP/IP network identified by a logical IP address.</span></span>  
  
 <span data-ttu-id="4d71e-138">TCP 提供可靠的資料傳遞且使用簡易。</span><span class="sxs-lookup"><span data-stu-id="4d71e-138">TCP provides reliable data delivery and ease of use.</span></span> <span data-ttu-id="4d71e-139">特別是，TCP 會通知傳送者有封包傳遞、保證封包會以和傳送時相同的順序傳遞、重新傳輸遺失的封包，以及確保資料封包沒有重複。</span><span class="sxs-lookup"><span data-stu-id="4d71e-139">Specifically, TCP notifies the sender of packet delivery, guarantees that packets are delivered in the same order in which they are sent, retransmits lost packets, and ensures that data packets are not duplicated.</span></span> <span data-ttu-id="4d71e-140">請注意，這種可靠的傳遞會套用在兩個 TCP/IP 節點之間，而且 *與 ws-reliablemessaging 不同* 的是，不論它們可能包含多少中繼節點，都是在端點之間套用。</span><span class="sxs-lookup"><span data-stu-id="4d71e-140">Note that this reliable delivery applies between two TCP/IP nodes, and is not the same thing as *WS-ReliableMessaging*, which applies between endpoints, no matter how many intermediate nodes they may include.</span></span>  
  
 <span data-ttu-id="4d71e-141">WCF TCP 傳輸已針對通訊兩端都使用 WCF 的案例進行優化。</span><span class="sxs-lookup"><span data-stu-id="4d71e-141">The WCF TCP transport is optimized for the scenario where both ends of the communication are using WCF.</span></span> <span data-ttu-id="4d71e-142">對於牽涉到不同電腦之間通訊的案例，此系結是最快的 WCF 系結。</span><span class="sxs-lookup"><span data-stu-id="4d71e-142">This binding is the fastest WCF binding for scenarios that involve communicating between different machines.</span></span> <span data-ttu-id="4d71e-143">訊息交換會對最佳化訊息傳輸使用 <xref:System.ServiceModel.Channels.BinaryMessageEncodingBindingElement>。</span><span class="sxs-lookup"><span data-stu-id="4d71e-143">The message exchanges use the <xref:System.ServiceModel.Channels.BinaryMessageEncodingBindingElement> for optimized message transfer.</span></span> <span data-ttu-id="4d71e-144">TCP 提供雙工通訊，因此即使用戶端位於網路位址轉譯 (NAT) 後，仍可用於實作雙工合約。</span><span class="sxs-lookup"><span data-stu-id="4d71e-144">TCP provides duplex communication and so can be used to implement duplex contracts, even if the client is behind network address translation (NAT).</span></span>
