<span data-ttu-id="5da63-110">ASP.NET HTTP 執行時間會處理 ASP.NET 要求，但不會參與處理以 WCF 服務為目標的要求，即使這些服務裝載于與 ASP.NET 內容相同的 AppDomain 中。</span><span class="sxs-lookup"><span data-stu-id="5da63-110">The ASP.NET HTTP runtime handles ASP.NET requests but does not participate in the processing of requests destined for WCF services, even though these services are hosted in the same AppDomain as is the ASP.NET content.</span></span> <span data-ttu-id="5da63-111">相反地，WCF 服務模型會攔截定址至 WCF 服務的訊息，並透過 WCF 傳輸/通道堆疊路由傳送。</span><span class="sxs-lookup"><span data-stu-id="5da63-111">Instead, the WCF Service Model intercepts messages addressed to WCF services and routes them through the WCF transport/channel stack.</span></span>

<span data-ttu-id="5da63-112">並存模型的結果如下：</span><span class="sxs-lookup"><span data-stu-id="5da63-112">The results of the side-by-side model are as follows:</span></span>

  - <span data-ttu-id="5da63-121">HttpCoNtext： <xref:System.Web.HttpContext.Current%2A> `null` 當從 WCF 服務記憶體取時，一律為。</span><span class="sxs-lookup"><span data-stu-id="5da63-121">HttpContext: <xref:System.Web.HttpContext.Current%2A> is always `null` when accessed from within a WCF service.</span></span> <span data-ttu-id="5da63-122">請改用 <xref:System.ServiceModel.Channels.RequestContext>。</span><span class="sxs-lookup"><span data-stu-id="5da63-122">Use <xref:System.ServiceModel.Channels.RequestContext> instead.</span></span>

## <a name="hosting-wcf-services-in-aspnet-compatibility-mode"></a><span data-ttu-id="5da63-137">在 ASP.NET 相容性模式中裝載 WCF 服務</span><span class="sxs-lookup"><span data-stu-id="5da63-137">Hosting WCF services in ASP.NET compatibility mode</span></span>

<span data-ttu-id="5da63-138">雖然 WCF 模型的設計是要在裝載環境和傳輸之間保持一致的行為，但通常應用程式不需要這麼多的彈性。</span><span class="sxs-lookup"><span data-stu-id="5da63-138">Although the WCF model is designed to behave consistently across hosting environments and transports, there are often scenarios where an application does not require this degree of flexibility.</span></span> <span data-ttu-id="5da63-139">WCF 的 ASP.NET 相容性模式適用于不需要在 IIS 外部裝載，或透過 HTTP 以外的通訊協定進行通訊，但卻使用 ASP.NET Web 應用程式平臺所有功能的情況。</span><span class="sxs-lookup"><span data-stu-id="5da63-139">WCF’s ASP.NET compatibility mode is suitable for scenarios that do not require the ability to host outside of IIS or to communicate over protocols other than HTTP, but that use all of features of the ASP.NET Web application platform.</span></span>

<span data-ttu-id="5da63-140">不同于預設並存設定，其中 WCF 裝載基礎結構會攔截 WCF 訊息並將其路由傳送至 HTTP 管線，在 ASP.NET 相容性模式中執行的 WCF 服務會完全參與 ASP.NET HTTP 要求生命週期。</span><span class="sxs-lookup"><span data-stu-id="5da63-140">Unlike the default side-by-side configuration, where the WCF hosting infrastructure intercepts WCF messages and routes them out of the HTTP pipeline, WCF services running in ASP.NET Compatibility Mode participate fully in the ASP.NET HTTP request lifecycle.</span></span> <span data-ttu-id="5da63-141">在相容性模式中，WCF 服務會透過執行來使用 HTTP 管線 <xref:System.Web.IHttpHandler> ，其方式類似于處理 ASPX 頁面和 .Asmx Web 服務的要求。</span><span class="sxs-lookup"><span data-stu-id="5da63-141">In compatibility mode, WCF services use the HTTP pipeline through an <xref:System.Web.IHttpHandler> implementation, similar to the way requests for ASPX pages and ASMX Web services are handled.</span></span> <span data-ttu-id="5da63-142">因此，WCF 的行為與與下列 ASP.NET 功能相關的 .ASMX 相同：</span><span class="sxs-lookup"><span data-stu-id="5da63-142">As a result, WCF behaves identically to ASMX with respect to the following ASP.NET features:</span></span>
