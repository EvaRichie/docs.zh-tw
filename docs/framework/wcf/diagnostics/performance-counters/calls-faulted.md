# <a name="calls-faulted"></a>發生錯誤的呼叫數

計數器名稱：發生錯誤的呼叫數  
  
## <a name="description"></a>描述  

 傳回錯誤之這個作業的呼叫次數。 在 Windows Communication Foundation (WCF) 應用程式中，服務方法會使用 SOAP 錯誤訊息來溝通處理錯誤資訊。 SOAP 錯誤是包含在服務作業之中繼資料內的訊息類型，因此會建立錯誤合約，而用戶端可使用該合約來讓其執行作業更強固或更具互動性。 由於 SOAP 錯誤會以 XML 表單的形式傳達給用戶端，所以其互通性很高。
