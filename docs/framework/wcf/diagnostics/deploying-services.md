# <a name="deploying-services"></a>部署服務

本主題說明如何將 Windows Communication Foundation 的 (WCF) 應用程式部署到執行時間環境。  
  
## <a name="choosing-the-hosting-environment-for-your-application"></a>為您的應用程式選擇裝載環境  

 WCF 服務的設計目的是要在任何支援 managed 程式碼的 Windows 進程中執行。 如果要成為作用中的服務，必須將服務裝載在建立及控制其內容和存留時間的執行階段環境中。 裝載選項的範圍從在最簡單的主控台應用程式到伺服器環境 (如 Windows 服務、Internet Information Services (IIS)) 內執行，或在由 Windows Activation Service (WAS) 管理的背景工作處理序中執行。 若要檢查 WCF 應用程式的不同裝載選項，請參閱 [裝載服務](../hosting-services.md)。
