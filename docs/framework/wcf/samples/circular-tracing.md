## <a name="circular-buffer-trace-listener"></a><span data-ttu-id="d223f-111">循環緩衝區追蹤接聽項</span><span class="sxs-lookup"><span data-stu-id="d223f-111">Circular Buffer Trace Listener</span></span>

<span data-ttu-id="d223f-112">實作「循環緩衝區追蹤接聽項」的基本概念是產生兩個檔案，而這兩個檔案可以各自儲存多達追蹤記錄資料總共所需數量的一半。</span><span class="sxs-lookup"><span data-stu-id="d223f-112">The concept behind the implementation of the Circular Buffer Trace Listener is to have two files that can each store up to half of the total desired trace log data.</span></span> <span data-ttu-id="d223f-113">接聽項會先建立一個檔案，再寫入該檔案直到資料大小上限的一半為止，然後於此時切換到第二個檔案。</span><span class="sxs-lookup"><span data-stu-id="d223f-113">The listener creates one file and writes to that file until it reaches the limit of half of the data size, at which point it switches to a second file.</span></span> <span data-ttu-id="d223f-114">當接聽項寫入到達第二個檔案的限制時，便會將新的追蹤覆寫到第一個檔案。</span><span class="sxs-lookup"><span data-stu-id="d223f-114">When the listener reaches the limit for the second file - it overwrites the first file with new traces.</span></span>

<span data-ttu-id="d223f-115">此接聽程式衍生自 `XmlWriteTraceListener` ，並可讓您使用[服務追蹤檢視器工具（svctraceviewer.exe）](../service-trace-viewer-tool-svctraceviewer-exe.md)來查看記錄。</span><span class="sxs-lookup"><span data-stu-id="d223f-115">This listener derives from the `XmlWriteTraceListener` and allows the logs to be viewed with the [Service Trace Viewer Tool (SvcTraceViewer.exe)](../service-trace-viewer-tool-svctraceviewer-exe.md).</span></span> <span data-ttu-id="d223f-116">當您嘗試檢視記錄時，只要在 [服務追蹤檢視器] 工具中同時開啟這兩個記錄檔，就可以輕易地將兩個記錄檔重新合併。</span><span class="sxs-lookup"><span data-stu-id="d223f-116">When attempting to view the logs, the two log files can be easily recombined by opening both of the log files at the same time in the Service Trace Viewer tool.</span></span> <span data-ttu-id="d223f-117">[服務追蹤檢視器] 工具會自動排序追蹤，讓它們依照正確的順序出現。</span><span class="sxs-lookup"><span data-stu-id="d223f-117">The Service Trace Viewer tool automatically takes care of sorting the traces so that they appear in the correct order.</span></span>

## <a name="configuration"></a><span data-ttu-id="d223f-118">組態</span><span class="sxs-lookup"><span data-stu-id="d223f-118">Configuration</span></span>

<span data-ttu-id="d223f-119">您可以為接聽項與來源項目新增下列程式碼，以便將服務設定為使用循環緩衝區追蹤接聽項。</span><span class="sxs-lookup"><span data-stu-id="d223f-119">A service can be configured to use the Circular Buffer Trace Listener by adding the following code for a listener and source elements.</span></span> <span data-ttu-id="d223f-120">在循環追蹤接聽項的組態中設定 `maxFileSizeKB` 屬性，可以指定檔案大小的上限。</span><span class="sxs-lookup"><span data-stu-id="d223f-120">The max file size is specified by setting the `maxFileSizeKB` attribute in the circular trace listener's configuration.</span></span> <span data-ttu-id="d223f-121">這點會在下列程式碼中示範。</span><span class="sxs-lookup"><span data-stu-id="d223f-121">This is demonstrated in the following code.</span></span>

```xml
<system.diagnostics>
  <sources>
    <source name="System.ServiceModel" switchValue="Information,ActivityTracing" propagateActivity="true">
      <listeners>
        <add name="CircularTraceListener" />
      </listeners>
    </source>
  </sources>
  <sharedListeners>
    <add name="CircularTraceListener" type="Microsoft. Samples.ServiceModel.CircularTraceListener,CircularTraceListener"
         initializeData="c:\logs\CircularTracing-service.svclog" maxFileSizeKB="100" />
  </sharedListeners>
  <trace autoflush="true" />
</system.diagnostics>
```
