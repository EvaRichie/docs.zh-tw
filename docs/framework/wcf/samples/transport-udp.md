## <a name="message-exchange-patterns"></a><span data-ttu-id="a2e9f-118">訊息交換模式</span><span class="sxs-lookup"><span data-stu-id="a2e9f-118">Message Exchange Patterns</span></span>  
 <span data-ttu-id="a2e9f-119">撰寫自訂傳輸時的第一個步驟是決定傳輸所需要的「訊息交換模式」(Message Exchange Patterns，MEP)。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-119">The first step in writing a custom transport is to decide which Message Exchange Patterns (MEPs) are required for the transport.</span></span> <span data-ttu-id="a2e9f-120">您可以從三個 MEP 中選擇：</span><span class="sxs-lookup"><span data-stu-id="a2e9f-120">There are three MEPs to choose from:</span></span>

     <span data-ttu-id="a2e9f-122">當使用資料包 (Datagram) MEP 時，用戶端會使用「射後不理」(Fire and Forget) 交換來傳送訊息。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-122">When using a datagram MEP, a client sends a message using a "fire and forget" exchange.</span></span> <span data-ttu-id="a2e9f-123">射後不理交換是一種需要以超出範圍之外的方式確認傳遞成功的交換。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-123">A fire and forget exchange is one that requires out-of-band confirmation of successful delivery.</span></span> <span data-ttu-id="a2e9f-124">訊息可能會在傳輸時遺失而永遠無法抵達服務。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-124">The message might be lost in transit and never reach the service.</span></span> <span data-ttu-id="a2e9f-125">即使傳送作業在用戶端已成功完成，也無法保證遠端端點已接收到該訊息。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-125">If the send operation completes successfully at the client end, it does not guarantee that the remote endpoint has received the message.</span></span> <span data-ttu-id="a2e9f-126">資料包是訊息的基本建置組塊，您可以在資料包的最上層建立自己的通訊協定，其中包括可靠的通訊協定和安全的通訊協定。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-126">The datagram is a fundamental building block for messaging, as you can build your own protocols on top of it—including reliable protocols and secure protocols.</span></span> <span data-ttu-id="a2e9f-127">用戶端資料包通道會實作 <xref:System.ServiceModel.Channels.IOutputChannel> 介面，服務資料包通道則會實作 <xref:System.ServiceModel.Channels.IInputChannel> 介面。</span><span class="sxs-lookup"><span data-stu-id="a2e9f-127">Client datagram channels implement the <xref:System.ServiceModel.Channels.IOutputChannel> interface and service datagram channels implement the <xref:System.ServiceModel.Channels.IInputChannel> interface.</span></span>  
  
- <span data-ttu-id="a2e9f-128">要求-回應 (IRequestChannel/IReplyChannel)</span><span class="sxs-lookup"><span data-stu-id="a2e9f-128">Request-Response (IRequestChannel/IReplyChannel)</span></span>
