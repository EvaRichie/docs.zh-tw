# <a name="weakly-typed-json-serialization-sample"></a><span data-ttu-id="1f27a-102">弱型別 JSON 序列化範例</span><span class="sxs-lookup"><span data-stu-id="1f27a-102">Weakly-typed JSON Serialization Sample</span></span>
<span data-ttu-id="1f27a-103">當將使用者定義的型別序列化為指定的 Wire 格式，或是將 Wire 格式還原序列化為使用者定義的型別時，服務和用戶端都必須提供指定的使用者定義型別，以供使用。</span><span class="sxs-lookup"><span data-stu-id="1f27a-103">When serializing a user-defined type to a given wire format, or deserializing a wire format back into a user-defined type, the given user-defined type must be available on both the service and the client.</span></span> <span data-ttu-id="1f27a-104">一般而言，若要完成這項操作， <xref:System.Runtime.Serialization.DataContractAttribute> 屬性會套用至這些使用者定義的型別，而 <xref:System.Runtime.Serialization.DataMemberAttribute> 屬性會套用至其成員。</span><span class="sxs-lookup"><span data-stu-id="1f27a-104">Usually to accomplish this, the <xref:System.Runtime.Serialization.DataContractAttribute> attribute is applied to these user-defined types and the <xref:System.Runtime.Serialization.DataMemberAttribute> attribute is applied to their members.</span></span> <span data-ttu-id="1f27a-105">這個機制也適用於使用 JavaScript Object Notation (JSON) 物件的情況，如主題 [How to: Serialize and Deserialize JSON Data](../feature-details/how-to-serialize-and-deserialize-json-data.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="1f27a-105">This mechanism also applies when working with JavaScript Object Notation (JSON) objects, as described in the topic [How to: Serialize and Deserialize JSON Data](../feature-details/how-to-serialize-and-deserialize-json-data.md).</span></span>  
  
 <span data-ttu-id="1f27a-106">在某些情況下，Windows Communication Foundation （WCF）服務或用戶端必須存取由開發人員控制之外的服務或用戶端所產生的 JSON 物件。</span><span class="sxs-lookup"><span data-stu-id="1f27a-106">In some scenarios, a Windows Communication Foundation (WCF) service or client must access JSON objects generated by a service or client that is outside of the control of the developer.</span></span> <span data-ttu-id="1f27a-107">當有更多 Web 服務公開 JSON Api 時，WCF 開發人員不一定要建立用來將任意 JSON 物件還原序列化的本機使用者定義類型。</span><span class="sxs-lookup"><span data-stu-id="1f27a-107">As more Web services publicly expose JSON APIs, it can become impractical for the WCF developer to construct local user-defined types into which to deserialize arbitrary JSON objects.</span></span> <span data-ttu-id="1f27a-108">這個範例提供一種機制，可讓 WCF 開發人員使用已還原序列化的任意 JSON 物件，而不需建立使用者定義型別。</span><span class="sxs-lookup"><span data-stu-id="1f27a-108">This sample provides a mechanism that enables WCF developers to work with deserialized, arbitrary JSON objects, without creating user-defined types.</span></span> <span data-ttu-id="1f27a-109">這稱為 JSON 物件的「 *弱型別序列化* 」(Weakly-Typed Serialization)，因為在編譯時期尚不知 JSON 物件要還原序列化成何種型別。</span><span class="sxs-lookup"><span data-stu-id="1f27a-109">This is known as *weakly-typed serialization* of JSON objects, because the type into which a JSON object deserializes is not known at compile time.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="1f27a-110">此範例的安裝程序與建置指示位於本主題的結尾。</span><span class="sxs-lookup"><span data-stu-id="1f27a-110">The setup procedure and build instructions for this sample are located at the end of this topic.</span></span>  
  
 <span data-ttu-id="1f27a-111">例如，公用 Web 服務 API 會傳回下列 JSON 物件，此物件描述有關服務的使用者的一些資訊。</span><span class="sxs-lookup"><span data-stu-id="1f27a-111">For example, a public Web service API returns the following JSON object, which describes some information about a user of the service.</span></span>  
  
```json  
{"personal": {"name": "Paul", "age": 23, "height": 1.7, "isSingle": true, "luckyNumbers": [5,17,21]}, "favoriteBands": ["Band ABC", "Band XYZ"]}  
```  
  
 <span data-ttu-id="1f27a-112">若要還原序列化這個物件，WCF 用戶端必須執行下列使用者定義類型。</span><span class="sxs-lookup"><span data-stu-id="1f27a-112">To deserialize this object, a WCF client must implement the following user-defined types.</span></span>  
  
```csharp  
[DataContract]  
 public class MemberProfile  
 {  
     [DataMember]  
     public PersonalInfo personal;  
  
     [DataMember]  
     public string[] favoriteBands;  
 }  
  
 [DataContract]  
 public class PersonalInfo  
 {  
     [DataMember]  
     public string name;  
  
     [DataMember]  
     public int age;  
  
     [DataMember]  
     public double height;  
  
     [DataMember]  
     public bool isSingle;  
  
     [DataMember]  
     public int[] luckyNumbers;  
 }  
```  
  
 <span data-ttu-id="1f27a-113">這可能會很麻煩，特別是如果用戶端必須處理一種以上的 JSON 物件型別。</span><span class="sxs-lookup"><span data-stu-id="1f27a-113">This can be cumbersome, especially if the client has to handle more than one type of JSON object.</span></span>  
  
 <span data-ttu-id="1f27a-114">這個範例提供的 `JsonObject` 型別引入還原序列化 JSON 物件的弱式型別表示。</span><span class="sxs-lookup"><span data-stu-id="1f27a-114">The `JsonObject` type provided by this sample introduces a weakly-typed representation of the deserialized JSON object.</span></span> <span data-ttu-id="1f27a-115">`JsonObject`依賴 JSON 物件與 .NET Framework 字典之間的自然對應，以及 JSON 陣列與 .NET Framework 陣列之間的對應。</span><span class="sxs-lookup"><span data-stu-id="1f27a-115">`JsonObject` relies on the natural mapping between JSON objects and .NET Framework dictionaries, and the mapping between JSON arrays and .NET Framework arrays.</span></span> <span data-ttu-id="1f27a-116">下列程式碼會顯示 `JsonObject` 型別。</span><span class="sxs-lookup"><span data-stu-id="1f27a-116">The following code shows the `JsonObject` type.</span></span>  
  
```csharp  
// Instantiation of JsonObject json omitted  
  
string name = json["root"]["personal"]["name"];  
int age = json["root"]["personal"]["age"];  
double height = json["root"]["personal"]["height"];  
bool isSingle = json["root"]["personal"]["isSingle"];  
int[] luckyNumbers = {  
                                     json["root"]["personal"]["luckyNumbers"][0],  
                                     json["root"]["personal"]["luckyNumbers"][1],  
                                     json["root"]["personal"]["luckyNumbers"][2]
                                 };  
string[] favoriteBands = {  
                                        json["root"]["favoriteBands"][0],  
                                        json["root"]["favoriteBands"][1]  
                                    };  
```  
  
 <span data-ttu-id="1f27a-117">請注意，您可以「瀏覽」JSON 物件和陣列，而不需要在編譯時宣告它們的型別。</span><span class="sxs-lookup"><span data-stu-id="1f27a-117">Note that you can "browse" JSON objects and arrays without the need to declare their type at compile time.</span></span> <span data-ttu-id="1f27a-118">如需最上層 `["root"]` 物件之需求的說明，請參閱主題 [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md)中所述。</span><span class="sxs-lookup"><span data-stu-id="1f27a-118">For an explanation of the requirement for the top-level `["root"]` object, see the topic [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md).</span></span>  
  
> [!NOTE]
> <span data-ttu-id="1f27a-119">`JsonObject` 類別僅供做為範例之用。</span><span class="sxs-lookup"><span data-stu-id="1f27a-119">The `JsonObject` class is provided as an example only.</span></span> <span data-ttu-id="1f27a-120">它尚未經過徹底的測試，不應用於實際執行環境中。</span><span class="sxs-lookup"><span data-stu-id="1f27a-120">It has not been thoroughly tested, and should not be used in production environments.</span></span> <span data-ttu-id="1f27a-121">弱型別 JSON 序列化有個明顯的含意存在，就是在使用 `JsonObject`時，缺少型別安全性。</span><span class="sxs-lookup"><span data-stu-id="1f27a-121">An obvious implication of weakly-typed JSON serialization is the lack of type-safety when working with `JsonObject`.</span></span>  
  
 <span data-ttu-id="1f27a-122">若要使用 `JsonObject` 型別，用戶端作業合約必須使用 <xref:System.ServiceModel.Channels.Message> 做為其傳回型別。</span><span class="sxs-lookup"><span data-stu-id="1f27a-122">To use the `JsonObject` type, the client operation contract must use <xref:System.ServiceModel.Channels.Message> as its return type.</span></span>  
  
```csharp  
[ServiceContract]  
    interface IClientSideProfileService  
    {  
        // There is no need to write a DataContract for the complex type returned by the service.  
        // The client will use a JsonObject to browse the JSON in the received message.  
  
        [OperationContract]  
        [WebGet(ResponseFormat = WebMessageFormat.Json)]  
        Message GetMemberProfile();  
    }  
```  
  
 <span data-ttu-id="1f27a-123">然後， `JsonObject` 會具現化，如下列程式碼所示。</span><span class="sxs-lookup"><span data-stu-id="1f27a-123">The `JsonObject` is then instantiated as shown in the following code.</span></span>  
  
```csharp  
// Code to instantiate IClientSideProfileService channel omitted…  
  
// Make a request to the service and obtain the Json response  
XmlDictionaryReader reader = channel.GetMemberProfile().GetReaderAtBodyContents();  
  
// Go through the Json as though it is a dictionary. There is no need to map it to a .NET CLR type.  
JsonObject json = new JsonObject(reader);  
```  
  
 <span data-ttu-id="1f27a-124">`JsonObject` 建構函式會採用透過 <xref:System.Xml.XmlDictionaryReader>方法取得的 <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> 。</span><span class="sxs-lookup"><span data-stu-id="1f27a-124">The `JsonObject` constructor takes a <xref:System.Xml.XmlDictionaryReader>, which is obtained through the <xref:System.ServiceModel.Channels.Message.GetReaderAtBodyContents%2A> method.</span></span> <span data-ttu-id="1f27a-125">讀取器包含用戶端所接收之 JSON 訊息的 XML 表示。</span><span class="sxs-lookup"><span data-stu-id="1f27a-125">The reader contains an XML representation of the JSON message received by the client.</span></span> <span data-ttu-id="1f27a-126">如需詳細資訊，請參閱[JSON 與 XML 之間的對應](../feature-details/mapping-between-json-and-xml.md)主題。</span><span class="sxs-lookup"><span data-stu-id="1f27a-126">For more information, see the topic [Mapping Between JSON and XML](../feature-details/mapping-between-json-and-xml.md).</span></span>  
  
 <span data-ttu-id="1f27a-127">此程式會產生下列輸出：</span><span class="sxs-lookup"><span data-stu-id="1f27a-127">The program produces the following output:</span></span>  
  
```console

To view the JSON output from the sample, navigate to http://localhost:8000/GetMemberProfile

I am single.  
My lucky numbers are 5, 17, and 21.  
My favorite bands are Band ABC and Band XYZ.  
```
