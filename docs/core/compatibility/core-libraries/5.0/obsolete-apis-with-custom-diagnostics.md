# <a name="api-obsoletions-with-non-default-diagnostic-ids"></a>使用非預設診斷識別碼的 API obsoletions

某些 Api 已標示為過時，從 .NET 5.0 開始。 這項重大變更是 *使用自訂診斷識別碼* 標示為已淘汰的 api 所特有。 隱藏 c # 編譯器所 [CS0618](../../../../csharp/language-reference/compiler-messages/cs0618.md) 的預設 OBSOLETION 診斷識別碼，不會隱藏編譯器在使用這些 api 時所產生的警告。

## <a name="change-description"></a>變更描述

在舊版的 .NET 中，您可以使用這些 Api，而不需要任何組建警告。 在 .NET 5.0 和更新版本中，使用這些 API 會產生具有自訂診斷識別碼的編譯時期警告或錯誤。 使用自訂診斷識別碼可讓您個別隱藏 obsoletion 警告，而非全面隱藏所有 obsoletion 警告。

下表列出已過時 Api 的自訂診斷識別碼及其對應的警告訊息。

| 診斷識別碼 | 描述 | Severity |
| - | - |
| `SYSLIB0001` | UTF-7 編碼並不安全，因此不應該使用。 請考慮改為使用 UTF-8。 | 警告 |
| `SYSLIB0002` | <xref:System.Security.Permissions.PrincipalPermissionAttribute> 執行時間不接受，而且不得使用。 | Error |
| `SYSLIB0003` | 執行時間不支援或不接受代碼啟用安全性 (CAS) 。 | 警告 |
| `SYSLIB0004` | 不支援 (CER) 功能的限制執列區域。 | 警告 |
| `SYSLIB0005` | 不支援全域組件快取 (GAC) 。 | 警告 |
| `SYSLIB0006` | <xref:System.Threading.Thread.Abort?displayProperty=nameWithType> 不受支援，而且會擲回 <xref:System.PlatformNotSupportedException> 。 | 警告 |
| `SYSLIB0007` | 不支援此密碼編譯演算法的預設執行。 | 警告 |
| `SYSLIB0008` | <xref:System.Runtime.CompilerServices.DebugInfoGenerator.CreatePdbGenerator>不支援並擲回 API <xref:System.PlatformNotSupportedException> 。 | 警告 |
| `SYSLIB0009` | <xref:System.Net.AuthenticationManager.Authenticate%2A?displayProperty=nameWithType>和 <xref:System.Net.AuthenticationManager.PreAuthenticate%2A?displayProperty=nameWithType> 方法不受支援，而且會擲回 <xref:System.PlatformNotSupportedException> 。 | 警告 |
| `SYSLIB0010`| 某些遠端 Api 不受支援且會擲回 <xref:System.PlatformNotSupportedException> 。 | 警告 |
| `SYSLIB0011` | <xref:System.Runtime.Serialization.Formatters.Binary.BinaryFormatter> 序列化已過時，不應該使用。 | 警告 |
| `SYSLIB0012` | <xref:System.Reflection.Assembly.CodeBase?displayProperty=nameWithType> 和 <xref:System.Reflection.Assembly.EscapedCodeBase?displayProperty=nameWithType> 只包含 .NET Framework 相容性。 請改用 <xref:System.Reflection.Assembly.Location?displayProperty=nameWithType>。 | 警告 |

## <a name="version-introduced"></a>引進的版本

.NET 5。0

## <a name="recommended-action"></a>建議的動作

- 使用警告上提供的 URL 連結，遵循針對每個診斷識別碼所提供的特定指引。

- 針對過時的類型或成員，無法使用標準診斷識別碼來抑制這些 obsoletions 的警告或錯誤;請改用自訂 `SYSLIBxxxx` 診斷識別碼值。
