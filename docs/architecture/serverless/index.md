## <a name="evolution-of-cloud-platforms"></a><span data-ttu-id="f8dfa-152">雲端平台的演進</span><span class="sxs-lookup"><span data-stu-id="f8dfa-152">Evolution of cloud platforms</span></span>

<span data-ttu-id="f8dfa-153">無伺服器是雲端平台多次反覆演進的高峰。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-153">Serverless is the culmination of several iterations of cloud platforms.</span></span> <span data-ttu-id="f8dfa-154">這項演進從資料中心內的實體電腦開始，逐漸發展為基礎結構即服務 (IaaS) 和平台即服務 (PaaS)。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-154">The evolution began with physical metal in the data center and progressed through Infrastructure as a Service (IaaS) and Platform as a Service (PaaS).</span></span>

<span data-ttu-id="f8dfa-156">在發展雲端以前，開發與作業之間的界限非常明確。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-156">Before the cloud, a discernible boundary existed between development and operations.</span></span> <span data-ttu-id="f8dfa-157">部署應用程式代表要回答無數個問題，如：</span><span class="sxs-lookup"><span data-stu-id="f8dfa-157">Deploying an application meant answering myriad questions like:</span></span>

- <span data-ttu-id="f8dfa-162">應該具有備用電力嗎？</span><span class="sxs-lookup"><span data-stu-id="f8dfa-162">Should there be redundant power?</span></span>

<span data-ttu-id="f8dfa-163">這份清單還不只如此，而且額外負荷也很龐大。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-163">The list goes on and the overhead was enormous.</span></span> <span data-ttu-id="f8dfa-164">在許多情況下，IT 部門不得已，必須處理許多浪費的資源。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-164">In many situations, IT departments were forced to deal with incredible waste.</span></span> <span data-ttu-id="f8dfa-165">浪費的原因是，過度配置的伺服器是損毀修復和待命伺服器的備份機器，以啟用相應放大。幸運的是，虛擬化技術的引進 (像是 [hyper-v](/virtualization/hyper-v-on-windows/about/)) 搭配虛擬機器 (vm) 帶來基礎結構即服務 (IaaS) 。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-165">The waste was due to over-allocation of servers as backup machines for disaster recovery and standby servers to enable scale-out. <span data-ttu-id="f8dfa-170">因為作業仍需負責處理各種工作，所以 IaaS 仍然需要龐大的額外負荷。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-170">IaaS still requires heavy overhead because operations is still responsible for various tasks.</span></span> <span data-ttu-id="f8dfa-171">這些工作包括：</span><span class="sxs-lookup"><span data-stu-id="f8dfa-171">These tasks include:</span></span>

- <span data-ttu-id="f8dfa-174">保持最新的作業系統。</span><span class="sxs-lookup"><span data-stu-id="f8dfa-174">Keeping the operating system up-to-date.</span></span>
