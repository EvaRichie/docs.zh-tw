<span data-ttu-id="01794-115">命令與查詢責任隔離 (CQRS) 是一種模式，可提供不同的介面來讀取 (或查詢修改資料的) 資料和作業。</span><span class="sxs-lookup"><span data-stu-id="01794-115">Command and Query Responsibility Segregation (CQRS) is a pattern that provides different interfaces for reading (or querying) data and operations that modify data.</span></span> <span data-ttu-id="01794-116">它解決了幾個常見的問題。</span><span class="sxs-lookup"><span data-stu-id="01794-116">It addresses several common problems.</span></span> <span data-ttu-id="01794-117">在傳統的 Create Read Update Delete (CRUD) 型系統中，從大量的讀取和寫入到相同的資料存放區都可能發生衝突。</span><span class="sxs-lookup"><span data-stu-id="01794-117">In traditional Create Read Update Delete (CRUD) based systems, conflicts can arise from high volume of both reads and writes to the same data store.</span></span> <span data-ttu-id="01794-118">鎖定可能會經常發生，並大幅降低讀取的速度。</span><span class="sxs-lookup"><span data-stu-id="01794-118">Locking may frequently occur and dramatically slow down reads.</span></span> <span data-ttu-id="01794-119">通常，資料會以數個網域物件的複合呈現，而讀取作業必須結合來自不同實體的資料。</span><span class="sxs-lookup"><span data-stu-id="01794-119">Often, data is presented as a composite of several domain objects and read operations must combine data from different entities.</span></span>
