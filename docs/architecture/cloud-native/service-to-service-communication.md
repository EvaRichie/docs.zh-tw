<span data-ttu-id="09bda-333">不過，EventGrid 是不同的。</span><span class="sxs-lookup"><span data-stu-id="09bda-333">EventGrid, however, is different.</span></span> <span data-ttu-id="09bda-334">它會執行*推送模型*，其中事件會以接收的方式傳送至 EventHandlers，提供近乎即時的事件傳遞。</span><span class="sxs-lookup"><span data-stu-id="09bda-334">It implements a *push model* in which events are sent to the EventHandlers as received, giving near real-time event delivery.</span></span> <span data-ttu-id="09bda-335">它也會降低成本，因為只有在需要取用事件時才會觸發服務，而不是隨著輪詢而持續使用。</span><span class="sxs-lookup"><span data-stu-id="09bda-335">It also reduces cost as the service is triggered only when it's needed to consume an event – not continually as with polling.</span></span> <span data-ttu-id="09bda-336">話雖如此，事件處理常式必須處理傳入的負載，並提供節流機制來保護自己免于被淹沒。</span><span class="sxs-lookup"><span data-stu-id="09bda-336">That said, an event handler must handle the incoming load and provide throttling mechanisms to protect itself from becoming overwhelmed.</span></span> <span data-ttu-id="09bda-337">許多使用這些事件的 Azure 服務，例如 Azure Functions 和 Logic Apps 會提供自動調整功能，以處理增加的負載。</span><span class="sxs-lookup"><span data-stu-id="09bda-337">Many Azure services that consume these events, such as Azure Functions and Logic Apps provide automatic autoscaling capabilities to handle increased loads.</span></span>  

<span data-ttu-id="09bda-338">事件方格是完全受控的無伺服器雲端服務。</span><span class="sxs-lookup"><span data-stu-id="09bda-338">Event Grid is a fully managed serverless cloud service.</span></span> <span data-ttu-id="09bda-339">它會根據您的流量動態調整，並只針對實際使用量收費，而不是預先購買的容量。</span><span class="sxs-lookup"><span data-stu-id="09bda-339">It dynamically scales based on your traffic and charges you only for your actual usage, not pre-purchased capacity.</span></span> <span data-ttu-id="09bda-340">每月前100000作業是免費的–作業會定義為事件輸入（傳入事件通知）、訂用帳戶傳遞嘗試、管理呼叫，以及依主旨篩選。</span><span class="sxs-lookup"><span data-stu-id="09bda-340">The first 100,000 operations per month are free – operations being defined as event ingress (incoming event notifications), subscription delivery attempts, management calls, and filtering by subject.</span></span> <span data-ttu-id="09bda-341">使用99.99% 的可用性，EventGrid 可保證在24小時期間內傳遞事件，並提供不成功傳遞的內建重試功能。</span><span class="sxs-lookup"><span data-stu-id="09bda-341">With 99.99% availability, EventGrid guarantees the delivery of an event within a 24-hour period, with built-in retry functionality for unsuccessful delivery.</span></span> <span data-ttu-id="09bda-342">未傳遞的訊息可以移至「寄不出的信件」佇列進行解析。</span><span class="sxs-lookup"><span data-stu-id="09bda-342">Undelivered messages can be moved to a "dead-letter" queue for resolution.</span></span>  <span data-ttu-id="09bda-343">不同于 Azure 服務匯流排，事件方格已針對快速效能進行微調，且不支援已排序訊息、交易和會話等功能。</span><span class="sxs-lookup"><span data-stu-id="09bda-343">Unlike Azure Service Bus, Event Grid is tuned for fast performance and doesn't support features like ordered messaging, transactions, and sessions.</span></span>

<span data-ttu-id="09bda-345">Azure 服務匯流排和事件方格針對公開單一獨立事件的應用程式提供絕佳的支援，例如新檔已插入 Cosmos DB。</span><span class="sxs-lookup"><span data-stu-id="09bda-345">Azure Service Bus and Event Grid provide great support for applications that expose single, discrete events like a new document has been inserted into a Cosmos DB.</span></span> <span data-ttu-id="09bda-346">但是，如果您的雲端原生系統需要處理*相關事件的資料流程*，該怎麼辦？</span><span class="sxs-lookup"><span data-stu-id="09bda-346">But, what if your cloud-native system needs to process a *stream of related events*?</span></span> <span data-ttu-id="09bda-347">[事件資料流程](https://docs.microsoft.com/archive/msdn-magazine/2015/february/microsoft-azure-the-rise-of-event-stream-oriented-systems)較複雜。</span><span class="sxs-lookup"><span data-stu-id="09bda-347">[Event streams](https://docs.microsoft.com/archive/msdn-magazine/2015/february/microsoft-azure-the-rise-of-event-stream-oriented-systems) are more complex.</span></span> <span data-ttu-id="09bda-348">它們通常是依時間排序、相互關聯，而且必須以群組的方式處理。</span><span class="sxs-lookup"><span data-stu-id="09bda-348">They're typically time-ordered, interrelated, and must be processed as a group.</span></span>
