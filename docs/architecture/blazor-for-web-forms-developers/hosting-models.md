## <a name="blazor-server-apps"></a>Blazor<span data-ttu-id="17ce1-133">伺服器應用程式</span><span class="sxs-lookup"><span data-stu-id="17ce1-133"> Server apps</span></span>

<span data-ttu-id="17ce1-134">回想[ Blazor 一下，](architecture-comparison.md#blazor)元件會將 Blazor 其輸出轉譯為名為的中繼抽象概念 `RenderTree` 。</span><span class="sxs-lookup"><span data-stu-id="17ce1-134">Recall from the [Blazor architecture](architecture-comparison.md#blazor) discussion that Blazor components render their output to an intermediate abstraction called a `RenderTree`.</span></span> <span data-ttu-id="17ce1-135">Blazor然後，架構會比較呈現的內容與先前轉譯的內容。</span><span class="sxs-lookup"><span data-stu-id="17ce1-135">The Blazor framework then compares what was rendered with what was previously rendered.</span></span> <span data-ttu-id="17ce1-136">這些差異會套用至 DOM。</span><span class="sxs-lookup"><span data-stu-id="17ce1-136">The differences are applied to the DOM.</span></span> Blazor<span data-ttu-id="17ce1-137">元件會與其呈現輸出的套用方式分離。</span><span class="sxs-lookup"><span data-stu-id="17ce1-137"> components are decoupled from how their rendered output is applied.</span></span> <span data-ttu-id="17ce1-138">因此，元件本身不需要與更新 UI 的進程在相同的進程中執行。</span><span class="sxs-lookup"><span data-stu-id="17ce1-138">Consequently, the components themselves don't have to run in the same process as the process updating the UI.</span></span> <span data-ttu-id="17ce1-139">事實上，它們甚至不需要在同一部電腦上執行。</span><span class="sxs-lookup"><span data-stu-id="17ce1-139">In fact, they don't even have to run on the same machine.</span></span>
