## <a name="what-about-microservices"></a><span data-ttu-id="41a4d-137">微服務呢？</span><span class="sxs-lookup"><span data-stu-id="41a4d-137">What about microservices?</span></span>

<span data-ttu-id="41a4d-138">當您考慮組織的雲端原生應用程式時，瞭解微服務及其運作方式是很重要的。</span><span class="sxs-lookup"><span data-stu-id="41a4d-138">Understanding microservices and how they work is important when you are considering cloud-native applications for your organization.</span></span>

<span data-ttu-id="41a4d-139">微服務架構是一種先進的方法，可讓您用於從頭建立的應用程式，或將現有的應用程式發展到雲端原生應用程式。</span><span class="sxs-lookup"><span data-stu-id="41a4d-139">The microservices architecture is an advanced approach that you can use for applications that are created from scratch or when you evolve existing applications toward cloud-native applications.</span></span> <span data-ttu-id="41a4d-140">您可以從將一些微服務新增至現有的應用程式開始，以瞭解新的微服務範例。</span><span class="sxs-lookup"><span data-stu-id="41a4d-140">You can start by adding a few microservices to existing applications to learn about the new microservices paradigms.</span></span> <span data-ttu-id="41a4d-141">但很明顯地，您需要架構設計和撰寫程式碼，特別是此類型的架構方法。</span><span class="sxs-lookup"><span data-stu-id="41a4d-141">But clearly, you need to architect and code, especially for this type of architectural approach.</span></span>

<span data-ttu-id="41a4d-142">不過，對於新的或新式應用程式來說，微服務並不是必要的。</span><span class="sxs-lookup"><span data-stu-id="41a4d-142">However, microservices are not mandatory for any new or modern application.</span></span> <span data-ttu-id="41a4d-143">微服務不是「魔術專案符號」，而是建立每個應用程式的單一、最佳方式。</span><span class="sxs-lookup"><span data-stu-id="41a4d-143">Microservices are not a "magic bullet," and they aren't the single, best way to create every application.</span></span> <span data-ttu-id="41a4d-144">使用微服務的方式和時機取決於您需要建立的應用程式類型。</span><span class="sxs-lookup"><span data-stu-id="41a4d-144">How and when you use microservices depends on the type of application that you need to build.</span></span>

<span data-ttu-id="41a4d-145">微服務架構即將成為分散式和大型或複雜的任務關鍵性應用程式的慣用方法，這些應用程式是以自發服務形式的多個獨立子系統為基礎。</span><span class="sxs-lookup"><span data-stu-id="41a4d-145">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple, independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="41a4d-146">在以微服務為基礎的架構中，應用程式會建立為服務的集合，這些服務可以獨立開發、測試、建立版本、部署及調整。</span><span class="sxs-lookup"><span data-stu-id="41a4d-146">In a microservices-based architecture, an application is built as a collection of services that can be independently developed, tested, versioned, deployed, and scaled.</span></span> <span data-ttu-id="41a4d-147">這種方法可以包含每個微服務的任何相關、自主資料庫。</span><span class="sxs-lookup"><span data-stu-id="41a4d-147">This approach can include any related, autonomous database per microservice.</span></span>
