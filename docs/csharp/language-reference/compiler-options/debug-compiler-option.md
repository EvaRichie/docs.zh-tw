<span data-ttu-id="aa12c-104">**-debug** 選項可讓編譯器產生偵錯資訊，並將它放在一或多個輸出檔案中。</span><span class="sxs-lookup"><span data-stu-id="aa12c-104">The **-debug** option causes the compiler to generate debugging information and place it in the output file or files.</span></span>

```console  
-debug[+ | -]  
-debug:{full | pdbonly}  
```

 <span data-ttu-id="aa12c-107">`+` &#124; `-`</span><span class="sxs-lookup"><span data-stu-id="aa12c-107">`+` &#124; `-`</span></span>  
 <span data-ttu-id="aa12c-108">指定 `+` 或 **-debug** 會讓編譯器產生偵錯資訊，並將其放在程式資料庫 (.pdb 檔案) 中。</span><span class="sxs-lookup"><span data-stu-id="aa12c-108">Specifying `+`, or just **-debug**, causes the compiler to generate debugging information and place it in a program database (.pdb file).</span></span> <span data-ttu-id="aa12c-109">指定 `-` (當您未指定 **-debug** 時，它就會生效) 將不會建立任何偵錯資訊。</span><span class="sxs-lookup"><span data-stu-id="aa12c-109">Specifying `-`, which is in effect if you do not specify **-debug**, causes no debug information to be created.</span></span>  
  
 <span data-ttu-id="aa12c-110">`full` &#124; `pdbonly`</span><span class="sxs-lookup"><span data-stu-id="aa12c-110">`full` &#124; `pdbonly`</span></span>
