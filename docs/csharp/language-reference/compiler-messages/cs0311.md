<span data-ttu-id="7b0f2-104">類型 ' type1 ' 無法當做泛型型別或方法 ' ' 中的類型參數 ' t ' 使用 \<name> 。</span><span class="sxs-lookup"><span data-stu-id="7b0f2-104">The type 'type1' cannot be used as type parameter 'T' in the generic type or method '\<name>'.</span></span> <span data-ttu-id="7b0f2-105">從 'type1' 到 'type2' 沒有隱含參考轉換。</span><span class="sxs-lookup"><span data-stu-id="7b0f2-105">There is no implicit reference conversion from 'type1' to 'type2'.</span></span>  
  
 <span data-ttu-id="7b0f2-106">將條件約束套用至泛型類型參數時，從實體引數到條件約束的類型必須存在隱含識別或參考轉換。</span><span class="sxs-lookup"><span data-stu-id="7b0f2-106">When a constraint is applied to a generic type parameter, an implicit identity or reference conversion must exist from the concrete argument to the type of the constraint.</span></span>

## <a name="example"></a><span data-ttu-id="7b0f2-111">範例</span><span class="sxs-lookup"><span data-stu-id="7b0f2-111">Example</span></span>  
  
```csharp  
// cs0311.cs  
class B {}  
class C {}  
class Test<T> where T : C  
{ }  
  
class Program  
{  
    static void Main()  
    {  
        Test<B> test = new Test<B>(); //CS0311  
    }  
}  
```  
  
 <span data-ttu-id="7b0f2-112">如果在嘗試使用實值類型引數時發生此錯誤，請注意，隱含數值轉換 (例如從 `short` 到 `int`) 並無法滿足泛型類型參數。</span><span class="sxs-lookup"><span data-stu-id="7b0f2-112">If this error occurs when trying to use a value-type argument, notice that an implicit numeric conversion, for example from `short` to `int`, does not satisfy a generic type parameter.</span></span>
