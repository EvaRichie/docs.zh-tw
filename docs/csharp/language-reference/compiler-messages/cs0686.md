<span data-ttu-id="3997c-104">存取子 'accessor' 無法為類型 'type' 實作介面成員 'member'。</span><span class="sxs-lookup"><span data-stu-id="3997c-104">Accessor 'accessor' cannot implement interface member 'member' for type 'type'.</span></span> <span data-ttu-id="3997c-105">請使用明確介面實作。</span><span class="sxs-lookup"><span data-stu-id="3997c-105">Use an explicit interface implementation.</span></span>  
  
 <span data-ttu-id="3997c-106">建議：當實作的介面中包含的方法名稱，與自動產生並與屬性或事件相關聯的方法發生衝突時，就會發生此錯誤。</span><span class="sxs-lookup"><span data-stu-id="3997c-106">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="3997c-107">屬性的 get/set 方法會產生為 get_property 和 set_property，而事件的 add/remove 方法會產生為 add_event 和 remove_event。</span><span class="sxs-lookup"><span data-stu-id="3997c-107">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="3997c-108">如果介面包含上述任一方法，就會發生衝突。</span><span class="sxs-lookup"><span data-stu-id="3997c-108">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="3997c-109">若要避免這個錯誤，請使用明確介面實作來實作這些方法。</span><span class="sxs-lookup"><span data-stu-id="3997c-109">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="3997c-110">若要執行這項操作，請將函式指定如下：</span><span class="sxs-lookup"><span data-stu-id="3997c-110">To do this, specify the function as:</span></span>

  
## <a name="example"></a><span data-ttu-id="3997c-111">範例</span><span class="sxs-lookup"><span data-stu-id="3997c-111">Example</span></span>  

 <span data-ttu-id="3997c-112">下列範例會產生 CS0686：</span><span class="sxs-lookup"><span data-stu-id="3997c-112">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="3997c-113">範例</span><span class="sxs-lookup"><span data-stu-id="3997c-113">Example</span></span>  

 <span data-ttu-id="3997c-114">宣告事件時也可能會發生此錯誤。</span><span class="sxs-lookup"><span data-stu-id="3997c-114">This error can also occur when declaring events.</span></span>  <span data-ttu-id="3997c-115">事件建構會自動產生 `add_event` 和 `remove_event` 方法，這些方法可能與介面中同名的方法發生衝突，如下列範例所示：</span><span class="sxs-lookup"><span data-stu-id="3997c-115">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
