# <a name="compiler-warning-level-1-cs4014"></a><span data-ttu-id="da357-102">編譯器警告 (層級 1) CS4014</span><span class="sxs-lookup"><span data-stu-id="da357-102">Compiler Warning (level 1) CS4014</span></span>

<span data-ttu-id="da357-103">因為未等候此呼叫，所以在呼叫完成之前會繼續執行目前的方法。</span><span class="sxs-lookup"><span data-stu-id="da357-103">Because this call is not awaited, execution of the current method continues before the call is completed.</span></span> <span data-ttu-id="da357-104">請考慮將 `await` 運算子套用至呼叫的結果。</span><span class="sxs-lookup"><span data-stu-id="da357-104">Consider applying the `await` operator to the result of the call.</span></span>

<span data-ttu-id="da357-105">目前的方法會呼叫傳回或的非同步方法， <xref:System.Threading.Tasks.Task> <xref:System.Threading.Tasks.Task%601> 而不會將 [await](../operators/await.md) 運算子套用至結果。</span><span class="sxs-lookup"><span data-stu-id="da357-105">The current method calls an async method that returns a <xref:System.Threading.Tasks.Task> or a <xref:System.Threading.Tasks.Task%601> and doesn't apply the [await](../operators/await.md) operator to the result.</span></span> <span data-ttu-id="da357-106">非同步方法的呼叫會啟動非同步工作。</span><span class="sxs-lookup"><span data-stu-id="da357-106">The call to the async method starts an asynchronous task.</span></span> <span data-ttu-id="da357-107">不過，由於不會套用任何 `await` 運算子，因此程式會繼續執行，而不等候工作完成。</span><span class="sxs-lookup"><span data-stu-id="da357-107">However, because no `await` operator is applied, the program continues without waiting for the task to complete.</span></span> <span data-ttu-id="da357-108">在大部分情況下，該行為並非您所預期。</span><span class="sxs-lookup"><span data-stu-id="da357-108">In most cases, that behavior isn't what you expect.</span></span> <span data-ttu-id="da357-109">通常呼叫方法的其他方面取決於呼叫的結果，或至少被呼叫的方法必須完成，才能從包含呼叫的方法傳回。</span><span class="sxs-lookup"><span data-stu-id="da357-109">Usually other aspects of the calling method depend on the results of the call or, minimally, the called method is expected to complete before you return from the method that contains the call.</span></span>

<span data-ttu-id="da357-110">同樣重要的問題是，在呼叫的非同步方法中引發的例外狀況會發生什麼情形。</span><span class="sxs-lookup"><span data-stu-id="da357-110">An equally important issue is what happens to exceptions that are raised in the called async method.</span></span> <span data-ttu-id="da357-111">在傳回 <xref:System.Threading.Tasks.Task> 或 <xref:System.Threading.Tasks.Task%601> 的方法中引發的例外狀況會儲存到傳回的工作中。</span><span class="sxs-lookup"><span data-stu-id="da357-111">An exception that's raised in a method that returns a <xref:System.Threading.Tasks.Task> or  <xref:System.Threading.Tasks.Task%601> is stored in the returned task.</span></span> <span data-ttu-id="da357-112">如果您未等候工作，也未明確檢查例外狀況，例外狀況就會遺失。</span><span class="sxs-lookup"><span data-stu-id="da357-112">If you don't await the task or explicitly check for exceptions, the exception is lost.</span></span> <span data-ttu-id="da357-113">如果您等候工作，則其例外狀況會再次擲回。</span><span class="sxs-lookup"><span data-stu-id="da357-113">If you await the task, its exception is rethrown.</span></span>

<span data-ttu-id="da357-114">因此最佳做法是一律等候呼叫。</span><span class="sxs-lookup"><span data-stu-id="da357-114">As a best practice, you should always await the call.</span></span>

<span data-ttu-id="da357-115">只有在您確定不要等候非同步呼叫完成，而且被呼叫的方法不會引發任何例外狀況時，才應考慮隱藏警告。</span><span class="sxs-lookup"><span data-stu-id="da357-115">You should consider suppressing the warning only if you're sure that you don't want to wait for the asynchronous call to complete and that the called method won't raise any exceptions.</span></span> <span data-ttu-id="da357-116">在這種情況下，您可以藉由將呼叫的工作結果指定至變數來隱藏警告。</span><span class="sxs-lookup"><span data-stu-id="da357-116">In that case, you can suppress the warning by assigning the task result of the call to a variable.</span></span>

<span data-ttu-id="da357-117">下列範例將示範如何產生警告、如何隱藏警告，以及如何等候呼叫。</span><span class="sxs-lookup"><span data-stu-id="da357-117">The following example shows how to cause the warning, how to suppress it, and how to await the call.</span></span>

```csharp
static async Task CallingMethodAsync(int millisecondsDelay)
{
    Console.WriteLine("  Entering calling method.");

    // Call #1.
    // Call an async method. Because you don't await it, its completion
    // isn't coordinated with the current method, CallingMethodAsync.
    // The following line causes warning CS4014.
    CalledMethodAsync(millisecondsDelay);

    // Call #2.
    // To suppress the warning without awaiting, you can assign the
    // returned task to a variable. The assignment doesn't change how
    // the program runs. However, recommended practice is always to
    // await a call to an async method.

    // Replace Call #1 with the following line.
    // Task delayTask = CalledMethodAsync(millisecondsDelay);

    // Call #3
    // To contrast with an awaited call, replace the unawaited call
    // (Call #1 or Call #2) with the following awaited call. Best
    // practice is to await the call.

    // await CalledMethodAsync(millisecondsDelay);

    Console.WriteLine("  Returning from calling method.");
}

static async Task CalledMethodAsync(int millisecondsDelay)
{
    Console.WriteLine("    Entering called method, starting and awaiting Task.Delay.");

    await Task.Delay(millisecondsDelay);

    Console.WriteLine("    Task.Delay is finished--returning from called method.");
}
```
