# <a name="compiler-error-cs1009"></a><span data-ttu-id="fd3f0-103">編譯器錯誤 CS1009</span><span class="sxs-lookup"><span data-stu-id="fd3f0-103">Compiler Error CS1009</span></span>
<span data-ttu-id="fd3f0-104">無法辨識的逸出序列</span><span class="sxs-lookup"><span data-stu-id="fd3f0-104">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="fd3f0-105">在[字串](../builtin-types/reference-types.md#the-string-type)中反斜線 (\\) 的後方出現未預期的字元。</span><span class="sxs-lookup"><span data-stu-id="fd3f0-105">An unexpected character follows a backslash (\\) in a [string](../builtin-types/reference-types.md#the-string-type).</span></span> <span data-ttu-id="fd3f0-106">編譯器需要其中一個有效的逸出字元。</span><span class="sxs-lookup"><span data-stu-id="fd3f0-106">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="fd3f0-107">如需詳細資訊，請參閱 [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md)。</span><span class="sxs-lookup"><span data-stu-id="fd3f0-107">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="fd3f0-108">下列範例會產生 CS1009。</span><span class="sxs-lookup"><span data-stu-id="fd3f0-108">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="fd3f0-109">通常發生此錯誤的原因是在檔案名稱中使用反斜線字元，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="fd3f0-109">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="fd3f0-110">若要解決這個錯誤，請使用 "\\\\" 或 @-quoted 字串常值，如下列範例所示。</span><span class="sxs-lookup"><span data-stu-id="fd3f0-110">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
