# <a name="compiler-error-cs0269"></a><span data-ttu-id="616d6-103">編譯器錯誤 CS0269</span><span class="sxs-lookup"><span data-stu-id="616d6-103">Compiler Error CS0269</span></span>
<span data-ttu-id="616d6-104">使用未指派的 out 參數 'parameter'</span><span class="sxs-lookup"><span data-stu-id="616d6-104">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="616d6-105">編譯器無法驗證 out 參數在被使用前是否已指派值，其值在指派時可能未定義。</span><span class="sxs-lookup"><span data-stu-id="616d6-105">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="616d6-106">請務必先在呼叫的方法中將值指派給 `out` 參數，再存取該值。</span><span class="sxs-lookup"><span data-stu-id="616d6-106">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="616d6-107">如果您需要使用傳入的變數值，請改用 `ref` 參數。</span><span class="sxs-lookup"><span data-stu-id="616d6-107">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="616d6-108">如需詳細資訊，請參閱[傳遞參數](../../programming-guide/classes-and-structs/passing-parameters.md)。</span><span class="sxs-lookup"><span data-stu-id="616d6-108">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="616d6-109">範例</span><span class="sxs-lookup"><span data-stu-id="616d6-109">Example</span></span>  
 <span data-ttu-id="616d6-110">下列範例會產生 CS0269：</span><span class="sxs-lookup"><span data-stu-id="616d6-110">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="616d6-111">範例</span><span class="sxs-lookup"><span data-stu-id="616d6-111">Example</span></span>  
 <span data-ttu-id="616d6-112">如果在 try 區塊中初始化變數，這也可能會發生，編譯器無法驗證是否會順利執行︰</span><span class="sxs-lookup"><span data-stu-id="616d6-112">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
