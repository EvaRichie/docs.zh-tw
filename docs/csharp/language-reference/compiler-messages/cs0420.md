# <a name="compiler-warning-level-1-cs0420"></a><span data-ttu-id="073bc-103">編譯器警告 (層級 1) CS0420</span><span class="sxs-lookup"><span data-stu-id="073bc-103">Compiler Warning (level 1) CS0420</span></span>
<span data-ttu-id="073bc-104">'identifier'：volatile 欄位的參考不會視為 volatile</span><span class="sxs-lookup"><span data-stu-id="073bc-104">'identifier': a reference to a volatile field will not be treated as volatile</span></span>  
  
 <span data-ttu-id="073bc-105">volatile 欄位通常不應使用 `ref` 或 **out** 參數傳遞，因為在函式範圍內不會將它視為 volatile。</span><span class="sxs-lookup"><span data-stu-id="073bc-105">A volatile field should not normally be passed using a `ref` or **out** parameter, since it will not be treated as volatile within the scope of the function.</span></span> <span data-ttu-id="073bc-106">不過這種情形有例外狀況，例如呼叫連鎖的 API 時。</span><span class="sxs-lookup"><span data-stu-id="073bc-106">There are exceptions to this, such as when calling an interlocked API.</span></span> <span data-ttu-id="073bc-107">如同任何警告，在您有意使用 volatile 欄位作為參考參數的罕見情況下，您可以使用 [#pragma warning](../preprocessor-directives/preprocessor-pragma-warning.md) 停用此警告。</span><span class="sxs-lookup"><span data-stu-id="073bc-107">As with any warning, you may use the [#pragma warning](../preprocessor-directives/preprocessor-pragma-warning.md) to disable this warning in those rare cases where you are intentionally using a volatile field as a reference parameter.</span></span>  
  
 <span data-ttu-id="073bc-108">下列範例會產生 CS0420：</span><span class="sxs-lookup"><span data-stu-id="073bc-108">The following sample generates CS0420:</span></span>  
  
```csharp  
// CS0420.cs  
// compile with: /W:1  
using System;  
  
class TestClass  
{  
   private volatile int i;  
  
   public void TestVolatile(ref int ii)  
   {  
   }  
  
   public static void Main()  
   {  
      TestClass x = new TestClass();  
      x.TestVolatile(ref x.i);   // CS0420
   }  
}  
```
