## <a name="design-goals-for-event-support"></a><span data-ttu-id="1a3d4-118">事件支援的設計目標</span><span class="sxs-lookup"><span data-stu-id="1a3d4-118">Design goals for event support</span></span>

## <a name="language-support-for-events"></a><span data-ttu-id="1a3d4-127">事件的語言支援</span><span class="sxs-lookup"><span data-stu-id="1a3d4-127">Language support for events</span></span>

<span data-ttu-id="1a3d4-128">定義事件以及訂閱或取消訂閱事件的語法，是委派的語法延伸模組。</span><span class="sxs-lookup"><span data-stu-id="1a3d4-128">The syntax for defining events, and subscribing or unsubscribing from events is an extension of the syntax for delegates.</span></span>

<span data-ttu-id="1a3d4-129">若要定義事件請使用 `event` 關鍵字︰</span><span class="sxs-lookup"><span data-stu-id="1a3d4-129">To define an event you use the `event` keyword:</span></span>

<span data-ttu-id="1a3d4-130">事件類型 (本例中為 `EventHandler<FileListArgs>`) 必須是委派類型。</span><span class="sxs-lookup"><span data-stu-id="1a3d4-130">The type of the event (`EventHandler<FileListArgs>` in this example) must be a delegate type.</span></span> <span data-ttu-id="1a3d4-131">宣告事件時應該遵循一些慣例。</span><span class="sxs-lookup"><span data-stu-id="1a3d4-131">There are a number of conventions that you should follow when declaring an event.</span></span> <span data-ttu-id="1a3d4-132">一般而言，事件委派類型具有 void 傳回。</span><span class="sxs-lookup"><span data-stu-id="1a3d4-132">Typically, the event delegate type has a void return.</span></span>
<span data-ttu-id="1a3d4-133">事件宣告應該是動詞或動詞片語。</span><span class="sxs-lookup"><span data-stu-id="1a3d4-133">Event declarations should be a verb, or a verb phrase.</span></span>
