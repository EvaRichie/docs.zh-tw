# <a name="compiler-error-cs1920"></a><span data-ttu-id="c3e75-102">編譯器錯誤 CS1920</span><span class="sxs-lookup"><span data-stu-id="c3e75-102">Compiler Error CS1920</span></span>

<span data-ttu-id="c3e75-103">項目初始設定式不可為空白。</span><span class="sxs-lookup"><span data-stu-id="c3e75-103">Element initializer cannot be empty.</span></span>  
  
 <span data-ttu-id="c3e75-104">集合初始設定式是由一連串的項目初始設定式所組成。</span><span class="sxs-lookup"><span data-stu-id="c3e75-104">A collection initializer consists of a sequence of element initializers.</span></span> <span data-ttu-id="c3e75-105">除非包含指派運算式，否則項目初始設定式不必括在大括弧中。</span><span class="sxs-lookup"><span data-stu-id="c3e75-105">The element initializers do not have to be enclosed in braces unless they contain an assignment expression.</span></span> <span data-ttu-id="c3e75-106">不過，如果您提供大括弧，它們不能是空的。</span><span class="sxs-lookup"><span data-stu-id="c3e75-106">However, if you do supply braces, they cannot be empty.</span></span> <span data-ttu-id="c3e75-107">如果項目初始設定式是物件初始設定式，那麼只要初始設定式包含新的物件建立運算式，就可能是空的大括弧。</span><span class="sxs-lookup"><span data-stu-id="c3e75-107">If the element initializer is an object initializer, the braces may be empty as long as the initializer contains a new object creation expression.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="c3e75-108">更正這個錯誤</span><span class="sxs-lookup"><span data-stu-id="c3e75-108">To correct this error</span></span>

## <a name="example"></a><span data-ttu-id="c3e75-111">範例</span><span class="sxs-lookup"><span data-stu-id="c3e75-111">Example</span></span>  

 <span data-ttu-id="c3e75-112">下列範例會產生 CS1920：</span><span class="sxs-lookup"><span data-stu-id="c3e75-112">The following example generates CS1920:</span></span>  
  
```csharp  
  // cs1920.cs  
using System.Collections.Generic;  
public class Test  
{  
    public static int Main()  
    {  
        // Error. Empty initializer
        // for inner list.  
        List<List<int>> collection =  
            new List<List<int>>() { { } }; // CS1920  
  
        // OK. No initializer for inner list.  
        List<List<int>> collection2 =  
            new List<List<int>>() {  };  
  
        // OK. Inner list is initialized
        // to one List<int> with zero elements.  
        List<List<int>> collection3 =  
            new List<List<int>>() { new List<int> { } };  
        return 0;  
    }  
}  
```
