 <span data-ttu-id="add78-106">有許多情況， `using` 或 `lock` 區塊會導致暫時性的資源流失。</span><span class="sxs-lookup"><span data-stu-id="add78-106">There are several scenarios where `using` or `lock` blocks will result in a temporary leak of resources.</span></span> <span data-ttu-id="add78-107">例如：</span><span class="sxs-lookup"><span data-stu-id="add78-107">Here is one example:</span></span>  
  
 `thisType f = null;`  
  
 `using (f)`  
  
 `{`  
  
 `f = new thisType();`  
  
 `...`  
  
 `}`  
  
 <span data-ttu-id="add78-108">在此例中，當 `thisType` 區塊完成執行，但在區塊內建立的 `using` 物件沒有被記憶體回收時 (雖然最後還是會回收)，會處置變數 `thisType` 的原始值，例如 null。</span><span class="sxs-lookup"><span data-stu-id="add78-108">In this case, the original value, such as null, of the variable `thisType` will be disposed of when the `using` block finishes executing, but the `thisType` object created inside the block will not be, although it will eventually get garbage collected.</span></span>

## <a name="example"></a><span data-ttu-id="add78-111">範例</span><span class="sxs-lookup"><span data-stu-id="add78-111">Example</span></span>  

 <span data-ttu-id="add78-112">下列程式碼會產生警告 CS0728。</span><span class="sxs-lookup"><span data-stu-id="add78-112">The following code will generate warning CS0728.</span></span>

```csharp
// CS0728.cs

public class ValidBase : IDisposable  
{  
    public void Dispose() {  }  
}  
  
public class Logger  
{  
    public static void dummy()  
    {  
        ValidBase vb = null;  
        using (vb)

            vb = null;  // CS0728  
        }  
        vb = null;  
    }  
    public static void Main() { }  
}  
```
