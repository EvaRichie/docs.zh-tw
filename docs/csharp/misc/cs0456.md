# <a name="compiler-error-cs0456"></a><span data-ttu-id="ab156-103">編譯器錯誤 CS0456</span><span class="sxs-lookup"><span data-stu-id="ab156-103">Compiler Error CS0456</span></span>
<span data-ttu-id="ab156-104">類型參數 '類型參數名稱 1' 具有 'struct' 條件約束，因此 '類型參數名稱 1' 不能作為 '類型參數名稱 2' 的條件約束</span><span class="sxs-lookup"><span data-stu-id="ab156-104">Type parameter 'Type Parameter Name 1' has the 'struct' constraint so 'Type Parameter Name 1' cannot be used as a constraint for 'Type Parameter Name 2'</span></span>  
  
 <span data-ttu-id="ab156-105">實值類型條件約束隱含地密封，因此這些條件約束不能作為第二個類型參數的條件約束。</span><span class="sxs-lookup"><span data-stu-id="ab156-105">Value type constraints are implicitly sealed so those constraints cannot be used as constraints on a second type parameter.</span></span> <span data-ttu-id="ab156-106">這是因為不能覆寫實值類型。</span><span class="sxs-lookup"><span data-stu-id="ab156-106">This is because value types cannot be overridden.</span></span> <span data-ttu-id="ab156-107">若要解決這個錯誤，請將實值類型條件約束直接放在第二個類型參數，而不要間接透過第一個類型參數進行。</span><span class="sxs-lookup"><span data-stu-id="ab156-107">To resolve this error, put a value type constraint directly on the second type parameter, instead of doing so indirectly by means of the first type parameter.</span></span>  
  
## <a name="example"></a><span data-ttu-id="ab156-108">範例</span><span class="sxs-lookup"><span data-stu-id="ab156-108">Example</span></span>  
 <span data-ttu-id="ab156-109">下列範例會產生 CS0456。</span><span class="sxs-lookup"><span data-stu-id="ab156-109">The following sample generates CS0456.</span></span>  
  
```csharp  
// CS0456.cs  
// compile with: /target:library  
public class GenericsErrors  
{  
   public class G5<T> where T : struct  
   {  
      public class N<U> where U : T {}   // CS0456  
      public class N2<U> where U : struct {}   // OK  
   }  
}  
```
