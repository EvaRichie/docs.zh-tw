# <a name="compiler-error-cs0172"></a><span data-ttu-id="a5816-103">編譯器錯誤 CS0172</span><span class="sxs-lookup"><span data-stu-id="a5816-103">Compiler Error CS0172</span></span>

<span data-ttu-id="a5816-104">無法判斷條件運算式的類型，因為 'type1' 和 'type2' 會互相隱含轉換。</span><span class="sxs-lookup"><span data-stu-id="a5816-104">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="a5816-105">在條件陳述式中，您必須能夠以隱含方式轉換 token 兩側的類型 `:` 。</span><span class="sxs-lookup"><span data-stu-id="a5816-105">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="a5816-106">此外，也不能有相互隱含的轉換;您只需要進行一次轉換。</span><span class="sxs-lookup"><span data-stu-id="a5816-106">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="a5816-107">下列範例會產生 CS0172：</span><span class="sxs-lookup"><span data-stu-id="a5816-107">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
