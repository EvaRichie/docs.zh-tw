# <a name="compiler-warning-level-1-cs0626"></a><span data-ttu-id="1cc52-103">編譯器警告 (層級 1) CS0626</span><span class="sxs-lookup"><span data-stu-id="1cc52-103">Compiler Warning (level 1) CS0626</span></span>

<span data-ttu-id="1cc52-104">方法、運算子或存取子 'method' 已標記為外部，但其上沒有屬性。</span><span class="sxs-lookup"><span data-stu-id="1cc52-104">Method, operator, or accessor 'method' is marked external and has no attributes on it.</span></span> <span data-ttu-id="1cc52-105">請考慮加入 DllImport 屬性來指定外部執行。</span><span class="sxs-lookup"><span data-stu-id="1cc52-105">Consider adding a DllImport attribute to specify the external implementation.</span></span>
  
 <span data-ttu-id="1cc52-106">標記為 `extern` 的方法也會以屬性標記，例如 [DllImport](xref:System.Runtime.InteropServices.DllImportAttribute) 屬性。</span><span class="sxs-lookup"><span data-stu-id="1cc52-106">A method marked `extern` should also be marked with an attribute, for example, the [DllImport](xref:System.Runtime.InteropServices.DllImportAttribute) attribute.</span></span>
  
 <span data-ttu-id="1cc52-107">此屬性會指定實作方法的位置。</span><span class="sxs-lookup"><span data-stu-id="1cc52-107">The attribute specifies where the method is implemented.</span></span> <span data-ttu-id="1cc52-108">在執行階段，程式將會需要這項資訊。</span><span class="sxs-lookup"><span data-stu-id="1cc52-108">At run time, the program will need this information.</span></span>  
  
 <span data-ttu-id="1cc52-109">下列範例會產生 CS0626：</span><span class="sxs-lookup"><span data-stu-id="1cc52-109">The following sample generates CS0626:</span></span>  
  
```csharp

public class MyClass  
{  
   static extern public void M(); // CS0626  
   // try the following line  
   // [DllImport("mydll.dll")] static extern public void M();  
  
   public static void Main()  
   {  
   }  
}  
```
