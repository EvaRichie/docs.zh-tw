# <a name="compiler-warning-level-3-cs0219"></a><span data-ttu-id="dfbbb-103">編譯器警告 (層級 3) CS0219</span><span class="sxs-lookup"><span data-stu-id="dfbbb-103">Compiler Warning (level 3) CS0219</span></span>
<span data-ttu-id="dfbbb-104">已指派變數 'variable'，但從未使用其值</span><span class="sxs-lookup"><span data-stu-id="dfbbb-104">The variable 'variable' is assigned but its value is never used</span></span>

 <span data-ttu-id="dfbbb-105">如果您宣告並指派變數，但未使用它，則編譯器會發出層級 3 警告。</span><span class="sxs-lookup"><span data-stu-id="dfbbb-105">The compiler issues a level-three warning, when you declare and assign a variable, but do not use it.</span></span>

 > [!NOTE]
 > <span data-ttu-id="dfbbb-106">只有當變數值為編譯時間常數時，編譯器才會產生此警告。</span><span class="sxs-lookup"><span data-stu-id="dfbbb-106">The compiler generates this warning only when the variable value is a compile-time constant.</span></span> <span data-ttu-id="dfbbb-107">將非常數運算式或方法結果指派給本機變數，可讓您更輕鬆地在偵錯工具中觀察這些運算式。</span><span class="sxs-lookup"><span data-stu-id="dfbbb-107">Assigning a non-constant expression or method result to a local variable makes it easier to observe those expressions in the debugger.</span></span> <span data-ttu-id="dfbbb-108">它也可讓結果成為可連接的，以防止在可連接該變數時進行垃圾收集。</span><span class="sxs-lookup"><span data-stu-id="dfbbb-108">It also makes the result reachable, preventing garbage collection while that variable is reachable.</span></span>

 <span data-ttu-id="dfbbb-109">下列範例顯示不會產生警告的情況和時機：</span><span class="sxs-lookup"><span data-stu-id="dfbbb-109">The following sample shows the cases when and when not the warning is generated:</span></span>

```csharp
// CS0219.cs
// compile with: /W:3
public class MyClass
{
   public static void Main()
   {
      var interpolated = "Interpolated";
      var a = 0; // CS0219
      int b = GetZero(); // Doesn't generate a warning.
      var c = "Regular string"; // CS0219
      var d = $"Constant interpolated string"; // Doesn't generate a warning.
      var e = $"{interpolated} string"; // Doesn't generate a warning.
   }

   private static int GetZero()
   {
       return 0;
   }
}  
```
