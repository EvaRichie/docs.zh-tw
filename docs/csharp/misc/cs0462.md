# <a name="compiler-error-cs0462"></a><span data-ttu-id="f3995-103">編譯器錯誤 CS0462</span><span class="sxs-lookup"><span data-stu-id="f3995-103">Compiler Error CS0462</span></span>
<span data-ttu-id="f3995-104">繼承的成員 'member1' 和 'member2'，在類型 'type' 中有相同的簽章，所以無法覆寫</span><span class="sxs-lookup"><span data-stu-id="f3995-104">The inherited members 'member1' and 'member2' have the same signature in type 'type', so they cannot be overridden</span></span>  
  
 <span data-ttu-id="f3995-105">此錯誤是引用泛型所造成。</span><span class="sxs-lookup"><span data-stu-id="f3995-105">This error arises with the introduction of generics.</span></span> <span data-ttu-id="f3995-106">一般來說，類別中方法的兩個版本不能有相同的簽章。</span><span class="sxs-lookup"><span data-stu-id="f3995-106">Normally, you cannot have two versions of a method in a class with the same signature.</span></span> <span data-ttu-id="f3995-107">但是，您可以使用泛型來指定可能會與另一種方法重複的泛型方法 (如果是使用特定類型進行具現化)。</span><span class="sxs-lookup"><span data-stu-id="f3995-107">But with generics, you can specify a generic method that might duplicate another method if it is instantiated with a particular type.</span></span>  
  
## <a name="example"></a><span data-ttu-id="f3995-108">範例</span><span class="sxs-lookup"><span data-stu-id="f3995-108">Example</span></span>  
 <span data-ttu-id="f3995-109">具現化 `C<int>` 時，會使用相同的簽章建立方法 `F` 的兩個版本，因此類別 `D` 中的覆寫無法決定要套用覆寫的版本。</span><span class="sxs-lookup"><span data-stu-id="f3995-109">When `C<int>` is instantiated, two versions of the method `F` are created with the same signature, so the override in class `D` cannot decide which one to apply the override to.</span></span>  
  
 <span data-ttu-id="f3995-110">下列範例會產生 CS0462。</span><span class="sxs-lookup"><span data-stu-id="f3995-110">The following sample generates CS0462.</span></span>  
  
```csharp  
// CS0462.cs  
// compile with: /target:library  
class C<T>
{  
   public virtual void F(T t) {}  
   public virtual void F(int t) {}  
}  
  
class D : C<int>
{  
   public override void F(int t) {}   // CS0462  
}  
```
