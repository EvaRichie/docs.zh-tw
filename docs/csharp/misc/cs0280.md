<span data-ttu-id="203a8-104">'type' 未實作 'pattern name' 模式。</span><span class="sxs-lookup"><span data-stu-id="203a8-104">'type' does not implement the 'pattern name' pattern.</span></span> <span data-ttu-id="203a8-105">'method name' 的簽章錯誤。</span><span class="sxs-lookup"><span data-stu-id="203a8-105">'method name' has the wrong signature.</span></span>  
  
 <span data-ttu-id="203a8-106">C# 中的兩個陳述式 ( **foreach** 和 **using**) 分別使用預先定義的模式："collection" 和 "resource"。</span><span class="sxs-lookup"><span data-stu-id="203a8-106">Two statements in C#, **foreach** and **using**, rely on predefined patterns, "collection" and "resource" respectively.</span></span> <span data-ttu-id="203a8-107">編譯器因方法的簽章不正確而無法符合其中一個陳述式與其模式時，就會發生這個警告。</span><span class="sxs-lookup"><span data-stu-id="203a8-107">This warning occurs when the compiler cannot match one of these statements to its pattern due to a method's incorrect signature.</span></span> <span data-ttu-id="203a8-108">例如，"collection" 模式需要一個稱為 <xref:System.Collections.IEnumerator.MoveNext%2A> 的方法，這個方法未採用任何參數，並且會傳回 `boolean`。</span><span class="sxs-lookup"><span data-stu-id="203a8-108">For example, the "collection" pattern requires that there be a method called <xref:System.Collections.IEnumerator.MoveNext%2A> which takes no parameters and returns a `boolean`.</span></span> <span data-ttu-id="203a8-109">您的程式碼可能包含具有參數或可能傳回物件的 <xref:System.Collections.IEnumerator.MoveNext%2A> 方法。</span><span class="sxs-lookup"><span data-stu-id="203a8-109">Your code might contain a <xref:System.Collections.IEnumerator.MoveNext%2A> method that has a parameter or perhaps returns an object.</span></span>  
  
 <span data-ttu-id="203a8-110">另一個範例是 "resource" 模式和 `using` 。</span><span class="sxs-lookup"><span data-stu-id="203a8-110">The "resource" pattern and `using` provide another example.</span></span> <span data-ttu-id="203a8-111">"resource" 模式需要 <xref:System.IDisposable.Dispose%2A> 方法；如果您定義同名的屬性，則會收到這個警告。</span><span class="sxs-lookup"><span data-stu-id="203a8-111">The "resource" pattern requires the <xref:System.IDisposable.Dispose%2A> method; if you define a property with the same name, you will get this warning.</span></span>  
  
 <span data-ttu-id="203a8-112">若要解決這個警告，請確定類型中的方法簽章符合模式中對應方法的簽章，並確定您沒有與模式所需方法同名的屬性。</span><span class="sxs-lookup"><span data-stu-id="203a8-112">To resolve this warning, ensure that the method signatures in your type match the signatures of the corresponding methods in the pattern, and ensure that you have no properties with the same name as a method required by the pattern.</span></span>  
  
## <a name="example"></a><span data-ttu-id="203a8-113">範例</span><span class="sxs-lookup"><span data-stu-id="203a8-113">Example</span></span>  
 <span data-ttu-id="203a8-114">下列範例會產生 CS0280。</span><span class="sxs-lookup"><span data-stu-id="203a8-114">The following sample generates CS0280.</span></span>  
  
```csharp  
// CS0280.cs  
using System;  
using System.Collections;  
  
public class ValidBase: IEnumerable  
{  
   IEnumerator IEnumerable.GetEnumerator()  
   {  
      yield return 0;  
   }  
  
   internal IEnumerator GetEnumerator()  
   {  
      yield return 0;  
   }  
}  
  
class Derived : ValidBase  
{  
   // field, not method  
   new public int GetEnumerator;  
}  
  
public class Test  
{  
   public static void Main()  
   {  
      foreach (int i in new Derived()) {}   // CS0280  
   }  
}  
```
