## <a name="c-language"></a><span data-ttu-id="747a8-112">C# 語言</span><span class="sxs-lookup"><span data-stu-id="747a8-112">C# language</span></span>

<span data-ttu-id="747a8-130">如果您必須與 COM 物件或原生 Win32 DLL 等其他 Windows 軟體互動，您可以在 C# 中透過稱為 "Interop" 的處理序來達到此目的。</span><span class="sxs-lookup"><span data-stu-id="747a8-130">If you have to interact with other Windows software such as COM objects or native Win32 DLLs, you can do this in C# through a process called "Interop".</span></span> <span data-ttu-id="747a8-131">Interop 幾乎可讓 C# 程式執行大部分原生 C++ 應用程式的功能。</span><span class="sxs-lookup"><span data-stu-id="747a8-131">Interop enables C# programs to do almost anything that a native C++ application can do.</span></span> <span data-ttu-id="747a8-132">C # 甚至還支援指標和「unsafe」程式碼的概念，在這些情況下，直接記憶體存取是很重要的。</span><span class="sxs-lookup"><span data-stu-id="747a8-132">C# even supports pointers and the concept of "unsafe" code for those cases in which direct memory access is critical.</span></span>

## <a name="net-framework-platform-architecture"></a><span data-ttu-id="747a8-140">.NET Framework 平台架構</span><span class="sxs-lookup"><span data-stu-id="747a8-140">.NET Framework Platform Architecture</span></span>

<span data-ttu-id="747a8-141">C# 程式在 .NET Framework 上執行，其為 Windows 不可或缺的元件，包括稱為通用語言執行平台 (CLR) 的虛擬執行系統和一組已整合類別庫。</span><span class="sxs-lookup"><span data-stu-id="747a8-141">C# programs run on the .NET Framework, an integral component of Windows that includes a virtual execution system called the common language runtime (CLR) and a unified set of class libraries.</span></span> <span data-ttu-id="747a8-142">CLR 是由 Microsoft 通用語言基礎結構 (CLI) 的商業實作，此國際標準是建立各種語言和程式庫都能一起順暢執行和開發環境的基礎。</span><span class="sxs-lookup"><span data-stu-id="747a8-142">The CLR is the commercial implementation by Microsoft of the common language infrastructure (CLI), an international standard that is the basis for creating execution and development environments in which languages and libraries work together seamlessly.</span></span>

<span data-ttu-id="747a8-143">以 c # 撰寫的原始程式碼會編譯成符合 CLI 規格 [ (IL) 的中繼語言 ](../../standard/managed-code.md) 。</span><span class="sxs-lookup"><span data-stu-id="747a8-143">Source code written in C# is compiled into an [intermediate language (IL)](../../standard/managed-code.md) that conforms to the CLI specification.</span></span> <span data-ttu-id="747a8-144">IL 程式碼和像是點陣圖和字串的資源，會以稱為組件的可執行檔儲存在磁碟上，副檔名通常為 .exe 或 .dll。</span><span class="sxs-lookup"><span data-stu-id="747a8-144">The IL code and resources, such as bitmaps and strings, are stored on disk in an executable file called an assembly, typically with an extension of .exe or .dll.</span></span> <span data-ttu-id="747a8-145">組件包含的資訊清單提供有關組件的型別、版本、文化特性及安全性需求資訊。</span><span class="sxs-lookup"><span data-stu-id="747a8-145">An assembly contains a manifest that provides information about the assembly's types, version, culture, and security requirements.</span></span>

<span data-ttu-id="747a8-146">C# 程式執行時，組件會載入至 CLR，根據資訊清單中的資訊執行各種不同動作。</span><span class="sxs-lookup"><span data-stu-id="747a8-146">When the C# program is executed, the assembly is loaded into the CLR, which might take various actions based on the information in the manifest.</span></span> <span data-ttu-id="747a8-147">然後，如果符合安全性需求，CLR 會執行即時 (JIT) 編譯，以將 IL 程式碼轉換成原生電腦指令。</span><span class="sxs-lookup"><span data-stu-id="747a8-147">Then, if the security requirements are met, the CLR performs Just-In-Time (JIT) compilation to convert the IL code to native machine instructions.</span></span> <span data-ttu-id="747a8-148">CLR 也提供有關自動記憶體回收、例外狀況處理和資源管理的其他服務。</span><span class="sxs-lookup"><span data-stu-id="747a8-148">The CLR also provides other services related to automatic garbage collection, exception handling, and resource management.</span></span> <span data-ttu-id="747a8-149">CLR 所執行的程式碼有時稱為「managed 程式碼」，相較于「非受控碼」，它會編譯成以特定系統為目標的原生電腦語言。</span><span class="sxs-lookup"><span data-stu-id="747a8-149">Code that's executed by the CLR is sometimes referred to as "managed code", in contrast to "unmanaged code", which is compiled into native machine language that targets a specific system.</span></span> <span data-ttu-id="747a8-150">下圖說明 C# 原始程式碼檔、.NET Framework 類別庫、組件及 CLR 的編譯時期和執行階段關聯性。</span><span class="sxs-lookup"><span data-stu-id="747a8-150">The following diagram illustrates the compile-time and run-time relationships of C# source code files, the .NET Framework class libraries, assemblies, and the CLR.</span></span>

![從 C# 原始程式碼到電腦執行](./media/introduction-to-the-csharp-language-and-the-net-framework/net-architecture-relationships.png)

<span data-ttu-id="747a8-152">語言互通性是 .NET Framework 的一項重要功能。</span><span class="sxs-lookup"><span data-stu-id="747a8-152">Language interoperability is a key feature of the .NET Framework.</span></span> <span data-ttu-id="747a8-153">由於 C# 編譯器所產生的 IL 程式碼符合通用型別規格 (CTS)，所以從 C# 產生的 IL 程式碼可以和從 Visual Basic、Visual C++ 的 .NET 版本所產生的程式碼互動，或和 20 種以上的任何其他 CTS 相容語言互動。</span><span class="sxs-lookup"><span data-stu-id="747a8-153">Because the IL code produced by the C# compiler conforms to the Common Type Specification (CTS), IL code generated from C# can interact with code that was generated from the .NET versions of Visual Basic, Visual C++, or any of more than 20 other CTS-compliant languages.</span></span> <span data-ttu-id="747a8-154">單一元件可能包含多個以不同 .NET 語言撰寫的模組，而類型可以彼此參考，如同以相同語言撰寫的一樣。</span><span class="sxs-lookup"><span data-stu-id="747a8-154">A single assembly may contain multiple modules written in different .NET languages, and the types can reference each other as if they were written in the same language.</span></span>
