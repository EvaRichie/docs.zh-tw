# <a name="how-to-define-value-equality-for-a-type-c-programming-guide"></a><span data-ttu-id="dbd38-104">如何定義類型的實值相等（c # 程式設計手冊）</span><span class="sxs-lookup"><span data-stu-id="dbd38-104">How to define value equality for a type (C# Programming Guide)</span></span>

<span data-ttu-id="dbd38-105">當您定義類別或結構時，需判斷是否有必要為類型建立實值相等 (或等價) 的自訂定義。</span><span class="sxs-lookup"><span data-stu-id="dbd38-105">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="dbd38-106">通常，當該類型的物件必須新增至某種集合，或物件的主要目的是為了儲存一組欄位或屬性時，就會實作實值相等。</span><span class="sxs-lookup"><span data-stu-id="dbd38-106">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="dbd38-107">您可以根據對該類型中所有欄位和屬性的比較來定義實值相等，也可以根據子集來進行定義。</span><span class="sxs-lookup"><span data-stu-id="dbd38-107">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span>

<span data-ttu-id="dbd38-108">不論是哪一種情況，以及在類別和結構中，您的執行都應該遵循等價的五項保證（針對下列規則，假設 `x` 、 `y` 和 `z` 都不是 null）：</span><span class="sxs-lookup"><span data-stu-id="dbd38-108">In either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence (For the following rules, assume that `x`, `y` and `z` are not null):</span></span>  
  
1. <span data-ttu-id="dbd38-109">`x.Equals(x)` 會傳回 `true`。</span><span class="sxs-lookup"><span data-stu-id="dbd38-109">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="dbd38-110">這稱為自反屬性。</span><span class="sxs-lookup"><span data-stu-id="dbd38-110">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="dbd38-111">`x.Equals(y)` 會傳回與 `y.Equals(x)` 相同的值。</span><span class="sxs-lookup"><span data-stu-id="dbd38-111">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="dbd38-112">這稱為對稱屬性。</span><span class="sxs-lookup"><span data-stu-id="dbd38-112">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="dbd38-113">如果 `(x.Equals(y) && y.Equals(z))` 傳回 `true`，則 `x.Equals(z)` 會傳回 `true`。</span><span class="sxs-lookup"><span data-stu-id="dbd38-113">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="dbd38-114">這稱為可轉移屬性。</span><span class="sxs-lookup"><span data-stu-id="dbd38-114">This is called the transitive property.</span></span>  
  
4. <span data-ttu-id="dbd38-115">只要 x 和 y 所參考的物件沒有經過修改，後續叫用 `x.Equals(y)` 就會傳回相同的值。</span><span class="sxs-lookup"><span data-stu-id="dbd38-115">Successive invocations of `x.Equals(y)` return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5. <span data-ttu-id="dbd38-116">任何非 null 值都不等於 null。</span><span class="sxs-lookup"><span data-stu-id="dbd38-116">Any non-null value is not equal to null.</span></span> <span data-ttu-id="dbd38-117">不過，CLR 會在所有方法呼叫上檢查是否有 null， `NullReferenceException` 如果此 `this` 參考為 null，則會擲回。</span><span class="sxs-lookup"><span data-stu-id="dbd38-117">However, the CLR checks for null on all method calls and throws a `NullReferenceException` if the `this` reference would be null.</span></span> <span data-ttu-id="dbd38-118">因此， `x.Equals(y)` 當為 null 時，會擲回例外狀況 `x` 。</span><span class="sxs-lookup"><span data-stu-id="dbd38-118">Therefore, `x.Equals(y)` throws an exception when `x` is null.</span></span> <span data-ttu-id="dbd38-119">這會中斷規則1或2，視的引數而定 `Equals` 。</span><span class="sxs-lookup"><span data-stu-id="dbd38-119">That breaks rules 1 or 2, depending on the argument to `Equals`.</span></span>

 <span data-ttu-id="dbd38-120">任何您已定義的結構，皆有繼承自 <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> 方法的 <xref:System.ValueType?displayProperty=nameWithType> 覆寫的預設實作實值相等。</span><span class="sxs-lookup"><span data-stu-id="dbd38-120">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="dbd38-121">此實作使用反映來檢查類型中的所有欄位和屬性。</span><span class="sxs-lookup"><span data-stu-id="dbd38-121">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="dbd38-122">雖然此實作會產生正確的結果，但相較於您針對該類型特別撰寫的自訂實作卻慢得多。</span><span class="sxs-lookup"><span data-stu-id="dbd38-122">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="dbd38-123">對類別和結構而言，實值相等的實作細節並不同。</span><span class="sxs-lookup"><span data-stu-id="dbd38-123">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="dbd38-124">不過，類別和結構都需要相同的基本步驟來實作相等：</span><span class="sxs-lookup"><span data-stu-id="dbd38-124">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1. <span data-ttu-id="dbd38-125">覆寫[虛擬](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> 方法。</span><span class="sxs-lookup"><span data-stu-id="dbd38-125">Override the [virtual](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="dbd38-126">在大部分情況下，實作 `bool Equals( object obj )` 應該只會呼叫特定類型的 `Equals` 方法，這是 <xref:System.IEquatable%601?displayProperty=nameWithType> 介面的實作。</span><span class="sxs-lookup"><span data-stu-id="dbd38-126">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="dbd38-127">(請參閱步驟 2)。</span><span class="sxs-lookup"><span data-stu-id="dbd38-127">(See step 2.)</span></span>  
  
2. <span data-ttu-id="dbd38-128">透過提供類型專屬的 `Equals` 方法實作 <xref:System.IEquatable%601?displayProperty=nameWithType> 介面。</span><span class="sxs-lookup"><span data-stu-id="dbd38-128">Implement the <xref:System.IEquatable%601?displayProperty=nameWithType> interface by providing a type-specific `Equals` method.</span></span> <span data-ttu-id="dbd38-129">實際的等價比較是在這裡執行。</span><span class="sxs-lookup"><span data-stu-id="dbd38-129">This is where the actual equivalence comparison is performed.</span></span> <span data-ttu-id="dbd38-130">例如，您可能決定只比較類型中的一個或兩個欄位，以定義相等。</span><span class="sxs-lookup"><span data-stu-id="dbd38-130">For example, you might decide to define equality by comparing only one or two fields in your type.</span></span> <span data-ttu-id="dbd38-131">不會從 `Equals`擲回例外狀況。</span><span class="sxs-lookup"><span data-stu-id="dbd38-131">Do not throw exceptions from `Equals`.</span></span> <span data-ttu-id="dbd38-132">僅適用於類別：此方法只會檢查在類別中宣告的欄位。</span><span class="sxs-lookup"><span data-stu-id="dbd38-132">For classes only: This method should examine only fields that are declared in the class.</span></span> <span data-ttu-id="dbd38-133">它應該呼叫 `base.Equals` 以檢查基底類別中的欄位</span><span class="sxs-lookup"><span data-stu-id="dbd38-133">It should call `base.Equals` to examine fields that are in the base class.</span></span> <span data-ttu-id="dbd38-134">(如果類型直接繼承自 <xref:System.Object>，請不要這樣做，因為 <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> 的 <xref:System.Object> 實作會執行參考相等檢查。)</span><span class="sxs-lookup"><span data-stu-id="dbd38-134">(Do not do this if the type inherits directly from <xref:System.Object>, because the <xref:System.Object> implementation of <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> performs a reference equality check.)</span></span>  
  
3. <span data-ttu-id="dbd38-135">選擇性但建議：多載 [==](../../language-reference/operators/equality-operators.md#equality-operator-) 和[！ =](../../language-reference/operators/equality-operators.md#inequality-operator-)運算子。</span><span class="sxs-lookup"><span data-stu-id="dbd38-135">Optional but recommended: Overload the [==](../../language-reference/operators/equality-operators.md#equality-operator-) and [!=](../../language-reference/operators/equality-operators.md#inequality-operator-) operators.</span></span>  
  
4. <span data-ttu-id="dbd38-136">覆寫 <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType>，以便有實值相等的兩個物件產生相同的雜湊碼。</span><span class="sxs-lookup"><span data-stu-id="dbd38-136">Override <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> so that two objects that have value equality produce the same hash code.</span></span>  
  
5. <span data-ttu-id="dbd38-137">選擇性：若要支援「大於」或「小於」的定義，請為您的型別執行 <xref:System.IComparable%601> 介面，並同時多載 [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) 和 [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-) 運算子。</span><span class="sxs-lookup"><span data-stu-id="dbd38-137">Optional: To support definitions for "greater than" or "less than," implement the <xref:System.IComparable%601> interface for your type, and also overload the [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) and [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-) operators.</span></span>
