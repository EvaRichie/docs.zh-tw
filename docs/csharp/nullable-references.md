# <a name="nullable-reference-types"></a><span data-ttu-id="fae33-104">可為 Null 的參考型別</span><span class="sxs-lookup"><span data-stu-id="fae33-104">Nullable reference types</span></span>

<span data-ttu-id="fae33-105">C# 8.0 引進**可為 Null 的參考型別**及**不可為 Null 的參考型別**，可讓您針對參考型別變數的屬性撰寫相關重要陳述式：</span><span class="sxs-lookup"><span data-stu-id="fae33-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="fae33-106">**參考不**應為 null。</span><span class="sxs-lookup"><span data-stu-id="fae33-106">**A reference isn't supposed to be null**.</span></span> <span data-ttu-id="fae33-107">當變數不應該是 null 時，編譯器會強制執行規則，確保在不先檢查變數不是 null 的情況下，就能安全地取值這些變數：</span><span class="sxs-lookup"><span data-stu-id="fae33-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it's safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="fae33-108">變數必須初始化為非 Null 值。</span><span class="sxs-lookup"><span data-stu-id="fae33-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="fae33-109">變數永遠不可指派 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="fae33-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="fae33-110">**參考可能為 Null**。</span><span class="sxs-lookup"><span data-stu-id="fae33-110">**A reference may be null**.</span></span> <span data-ttu-id="fae33-111">當變數可為 Null 時，編譯器會實施不同的規則，確保您已針對 Null 參考正確地進行檢查：</span><span class="sxs-lookup"><span data-stu-id="fae33-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="fae33-112">只有在編譯器能保證該值並非為 Null 時，才能對該變數進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="fae33-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>
  - <span data-ttu-id="fae33-113">這些變數可使用預設的 `null` 值初始化，也可以在其他程式碼中指派 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="fae33-113">These variables may be initialized with the default `null` value and may be assigned the value `null` in other code.</span></span>

<span data-ttu-id="fae33-114">這項新功能對舊版 c # 中的參考變數的處理提供了顯著的好處，因為無法從變數宣告判斷設計意圖。</span><span class="sxs-lookup"><span data-stu-id="fae33-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent can't be determined from the variable declaration.</span></span> <span data-ttu-id="fae33-115">編譯器不針對參考型別的 Null 參考例外狀況提供安全：</span><span class="sxs-lookup"><span data-stu-id="fae33-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="fae33-116">**參考可為 Null**。</span><span class="sxs-lookup"><span data-stu-id="fae33-116">**A reference can be null**.</span></span> <span data-ttu-id="fae33-117">當參考型別初始化為 null，或稍後指派給 null 時，編譯器不會發出警告。</span><span class="sxs-lookup"><span data-stu-id="fae33-117">The compiler doesn't issue warnings when a reference type is initialized to null, or later assigned to null.</span></span> <span data-ttu-id="fae33-118">當參考這些變數時，編譯器會發出警告，但不含 null 檢查。</span><span class="sxs-lookup"><span data-stu-id="fae33-118">The compiler issues warnings when these variables are dereferenced without null checks.</span></span>
- <span data-ttu-id="fae33-119">**參考已假設為並非 Null**。</span><span class="sxs-lookup"><span data-stu-id="fae33-119">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="fae33-120">編譯器不會在對參考型別進行取值 (Dereference) 時發出任何警告。</span><span class="sxs-lookup"><span data-stu-id="fae33-120">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="fae33-121">如果變數設定為可能是 null 的運算式，則編譯器會發出警告。</span><span class="sxs-lookup"><span data-stu-id="fae33-121">The compiler issues warnings if a variable is set to an expression that may be null.</span></span>

<span data-ttu-id="fae33-122">這些警告會在編譯時期發出。</span><span class="sxs-lookup"><span data-stu-id="fae33-122">These warnings are emitted at compile time.</span></span> <span data-ttu-id="fae33-123">編譯器不會在可為 null 的內容中加入任何 null 檢查或其他執行時間結構。</span><span class="sxs-lookup"><span data-stu-id="fae33-123">The compiler doesn't add any null checks or other runtime constructs in a nullable context.</span></span> <span data-ttu-id="fae33-124">在執行時間，可為 null 的參考和不可為 null 的參考都是相等的。</span><span class="sxs-lookup"><span data-stu-id="fae33-124">At runtime, a nullable reference and a non-nullable reference are equivalent.</span></span>

<span data-ttu-id="fae33-125">透過新增的可為 Null 參考型別，您可以更清楚地宣告您的意圖。</span><span class="sxs-lookup"><span data-stu-id="fae33-125">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="fae33-126">`null` 值是表示變數並非指向某個值的正確方式。</span><span class="sxs-lookup"><span data-stu-id="fae33-126">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="fae33-127">請不要使用這項功能來從您的程式碼中移除所有 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="fae33-127">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="fae33-128">相反地，您應該對編譯器及其他閱讀您程式碼之開發人員宣告您的意圖。</span><span class="sxs-lookup"><span data-stu-id="fae33-128">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="fae33-129">藉由宣告您的意圖，編譯器會在您撰寫的程式碼與該意圖不一致時通知您。</span><span class="sxs-lookup"><span data-stu-id="fae33-129">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="fae33-130">**可為 Null 參考型別**的語法與[可為 Null 實值型別](language-reference/builtin-types/nullable-value-types.md)語法相同：將 `?` 附加至變數的型別。</span><span class="sxs-lookup"><span data-stu-id="fae33-130">A **nullable reference type** is noted using the same syntax as [nullable value types](language-reference/builtin-types/nullable-value-types.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="fae33-131">例如，下列變數宣告代表可為 Null 字串變數，`name`：</span><span class="sxs-lookup"><span data-stu-id="fae33-131">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>

<span data-ttu-id="fae33-132">`?`未附加至類型名稱的任何變數都是**不可為 null 的參考型別**。</span><span class="sxs-lookup"><span data-stu-id="fae33-132">Any variable where the `?` isn't appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="fae33-133">當您啟用這項功能時，其中包括現有程式碼中的所有參考型別變數。</span><span class="sxs-lookup"><span data-stu-id="fae33-133">That includes all reference type variables in existing code when you've enabled this feature.</span></span>

<span data-ttu-id="fae33-134">編譯器會使用靜態分析來判斷可為 Null 參考是否已知並非為 Null。</span><span class="sxs-lookup"><span data-stu-id="fae33-134">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="fae33-135">若您在可為 Null 參考可能為 Null 時對其進行取值 (Dereference)，編譯器會警告您。</span><span class="sxs-lookup"><span data-stu-id="fae33-135">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="fae33-136">您可以使用[容許運算子](language-reference/operators/null-forgiving.md)後面的變數名稱來覆寫此行為 `!` 。</span><span class="sxs-lookup"><span data-stu-id="fae33-136">You can override this behavior by using the [null-forgiving operator](language-reference/operators/null-forgiving.md) `!` following a variable name.</span></span> <span data-ttu-id="fae33-137">例如，若您知道 `name` 變數並非為 Null，但編譯器卻發出警告，您可以撰寫下列程式碼來覆寫編譯器的分析：</span><span class="sxs-lookup"><span data-stu-id="fae33-137">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>


## <a name="nullability-of-types"></a><span data-ttu-id="fae33-138">型別的可 NULL 性</span><span class="sxs-lookup"><span data-stu-id="fae33-138">Nullability of types</span></span>

<span data-ttu-id="fae33-139">任何參考型別都可擁有四種「可 NULL 性」\*\* 中的其中一種，其描述警告產生的時機：</span><span class="sxs-lookup"><span data-stu-id="fae33-139">Any reference type can have one of four *nullabilities*, which describes when warnings are generated:</span></span>

- <span data-ttu-id="fae33-140">*不可為 null*： Null 無法指派給這個類型的變數。</span><span class="sxs-lookup"><span data-stu-id="fae33-140">*Nonnullable*: Null can't be assigned to variables of this type.</span></span> <span data-ttu-id="fae33-141">此型別的變數不需進行 Null 檢查即可進行取值 (Dereference)。</span><span class="sxs-lookup"><span data-stu-id="fae33-141">Variables of this type don't need to be null-checked before dereferencing.</span></span>
- <span data-ttu-id="fae33-142">*Nullable*： null 可以指派給這個類型的變數。</span><span class="sxs-lookup"><span data-stu-id="fae33-142">*Nullable*: Null can be assigned to variables of this type.</span></span> <span data-ttu-id="fae33-143">若沒有事先檢查是否為 `null` 即對此型別的變數進行取值 (Dereference)，即會產生警告。</span><span class="sxs-lookup"><span data-stu-id="fae33-143">Dereferencing variables of this type without first checking for `null` causes a warning.</span></span>

1. <span data-ttu-id="fae33-210">變數已確定指派為並非 Null 的值。</span><span class="sxs-lookup"><span data-stu-id="fae33-210">The variable has been definitely assigned to a non-null value.</span></span>
