## <a name="cause"></a><span data-ttu-id="82d01-111">原因</span><span class="sxs-lookup"><span data-stu-id="82d01-111">Cause</span></span>

<span data-ttu-id="82d01-112">使用 <xref:System.Net.Http.HttpClient?displayProperty=fullName> 時，提供平臺特定的處理常式 (<xref:System.Net.Http.WinHttpHandler?displayProperty=fullName> 或 <xref:System.Net.Http.HttpClientHandler?displayProperty=fullName>) 其 `CheckCertificateRevocationList` 屬性可能會設定為，以允許將已 `false` 撤銷的憑證被 <xref:System.Net.Http.HttpClient> 視為有效。</span><span class="sxs-lookup"><span data-stu-id="82d01-112">Using <xref:System.Net.Http.HttpClient?displayProperty=fullName> while providing a platform specific handler (<xref:System.Net.Http.WinHttpHandler?displayProperty=fullName> or <xref:System.Net.Http.HttpClientHandler?displayProperty=fullName>) whose `CheckCertificateRevocationList` property is possibly set to `false` will allow revoked certificates to be accepted by the <xref:System.Net.Http.HttpClient> as valid.</span></span>

<span data-ttu-id="82d01-113">這項規則類似于 [CA5400](ca5400.md)，但分析無法判斷 `CheckCertificateRevocationList` 屬性是否為絕對 `false` 或未設定。</span><span class="sxs-lookup"><span data-stu-id="82d01-113">This rule is similar to [CA5400](ca5400.md), but analysis can't determine that the `CheckCertificateRevocationList` property is definitely `false` or not set.</span></span>

## <a name="pseudo-code-examples"></a><span data-ttu-id="82d01-128">虛擬程式碼範例</span><span class="sxs-lookup"><span data-stu-id="82d01-128">Pseudo-code examples</span></span>

```csharp
using System;
using System.Net.Http;

class ExampleClass
{
    void ExampleMethod(bool checkCertificateRevocationList)
    {
        WinHttpHandler winHttpHandler = new WinHttpHandler();
        winHttpHandler.CheckCertificateRevocationList = checkCertificateRevocationList;
        Random r = new Random();

        if (r.Next(6) == 4)
        {
            winHttpHandler.CheckCertificateRevocationList = true;
        }

        HttpClient httpClient = new HttpClient(winHttpHandler);
    }
}
```
