## <a name="cause"></a><span data-ttu-id="8cb66-111">原因</span><span class="sxs-lookup"><span data-stu-id="8cb66-111">Cause</span></span>

<span data-ttu-id="8cb66-112">型別宣告了具有字串參數的方法，其名稱包含 "uri"、"Uri"、"urn"、"Urn"、"url" 或 "Url"，且類型未宣告採用參數的對應多載 <xref:System.Uri?displayProperty=fullName> 。</span><span class="sxs-lookup"><span data-stu-id="8cb66-112">A type declares a method with a string parameter whose name contains "uri", "Uri", "urn", "Urn", "url", or "Url" and the type does not declare a corresponding overload that takes a <xref:System.Uri?displayProperty=fullName> parameter.</span></span>

<span data-ttu-id="8cb66-113">根據預設，此規則只會查看外部可見的類型，但這是 [可](#configure-code-to-analyze)設定的。</span><span class="sxs-lookup"><span data-stu-id="8cb66-113">By default, this rule only looks at externally visible types, but this is [configurable](#configure-code-to-analyze).</span></span>

## <a name="rule-description"></a><span data-ttu-id="8cb66-114">規則描述</span><span class="sxs-lookup"><span data-stu-id="8cb66-114">Rule description</span></span>

<span data-ttu-id="8cb66-115">此規則會根據 camel 大小寫慣例，將參數名稱分割成權杖，並檢查每個權杖是否等於 "uri"、"Uri"、"urn"、"Urn"、"url" 或 "Url"。</span><span class="sxs-lookup"><span data-stu-id="8cb66-115">This rule splits the parameter name into tokens based on the camel casing convention and checks whether each token equals "uri", "Uri", "urn", "Urn", "url", or "Url".</span></span> <span data-ttu-id="8cb66-116">如果有相符的規則，此規則會假設參數代表 (URI) 的統一資源識別項。</span><span class="sxs-lookup"><span data-stu-id="8cb66-116">If there is a match, the rule assumes that the parameter represents a uniform resource identifier (URI).</span></span> <span data-ttu-id="8cb66-117">URI 的字串表示方式容易發生剖析和編碼錯誤，並且可能因此產生安全性弱點。</span><span class="sxs-lookup"><span data-stu-id="8cb66-117">A string representation of a URI is prone to parsing and encoding errors, and can lead to security vulnerabilities.</span></span> <span data-ttu-id="8cb66-118">如果方法採用 URI 的字串標記法，則應該提供對應的多載，以取得類別的實例 <xref:System.Uri> ，這會以安全且安全的方式提供這些服務。</span><span class="sxs-lookup"><span data-stu-id="8cb66-118">If a method takes a string representation of a URI, a corresponding overload should be provided that takes an instance of the <xref:System.Uri> class, which provides these services in a safe and secure manner.</span></span>

## <a name="how-to-fix-violations"></a><span data-ttu-id="8cb66-119">如何修正違規</span><span class="sxs-lookup"><span data-stu-id="8cb66-119">How to fix violations</span></span>

<span data-ttu-id="8cb66-120">若要修正此規則的違規情形，請將參數變更為 <xref:System.Uri> 類型; 這是中斷性變更。</span><span class="sxs-lookup"><span data-stu-id="8cb66-120">To fix a violation of this rule, change the parameter to a <xref:System.Uri> type; this is a breaking change.</span></span> <span data-ttu-id="8cb66-121">或者，提供接受參數之方法的多載， <xref:System.Uri> 這是不中斷的變更。</span><span class="sxs-lookup"><span data-stu-id="8cb66-121">Alternately, provide an overload of the method that takes a <xref:System.Uri> parameter; this is a non-breaking change.</span></span>

## <a name="when-to-suppress-warnings"></a><span data-ttu-id="8cb66-122">隱藏警告的時機</span><span class="sxs-lookup"><span data-stu-id="8cb66-122">When to suppress warnings</span></span>
