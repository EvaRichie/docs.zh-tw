## <a name="how-to-fix-violations"></a><span data-ttu-id="0dd23-118">如何修正違規</span><span class="sxs-lookup"><span data-stu-id="0dd23-118">How to fix violations</span></span>

<span data-ttu-id="0dd23-119">若要修正違規，請叫 <xref:System.Object.Equals%2A> 用實作為覆寫並執行 <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> 。</span><span class="sxs-lookup"><span data-stu-id="0dd23-119">To fix a violation, override <xref:System.Object.Equals%2A> and implement it by invoking the <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> implementation.</span></span> <span data-ttu-id="0dd23-120">例如，下列兩個程式碼片段會顯示規則違規以及如何修正此問題：</span><span class="sxs-lookup"><span data-stu-id="0dd23-120">For example, the following two code snippets show a violation of the rule and how to fix it:</span></span>

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public bool Equals(S other)
        => _value == other._value;

    public override bool Equals(object obj)
        => obj is S objS && Equals(objS);

    public override int GetHashCode()
        => _value.GetHashCode();
}
```
