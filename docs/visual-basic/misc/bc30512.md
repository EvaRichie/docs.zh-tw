# <a name="option-strict-on-disallows-implicit-conversions-from-type1-to-type2"></a><span data-ttu-id="3986e-102">Option Strict 為 On 時，不允許從 ' \<type1> ' 到 ' ' 的隱含轉換 \<type2></span><span class="sxs-lookup"><span data-stu-id="3986e-102">Option Strict On disallows implicit conversions from '\<type1>' to '\<type2>'</span></span>
<span data-ttu-id="3986e-103">您嘗試在類型檢查參數 ( `Long` ) 設定為 `Integer`時，將某個類型轉換成另一個不能包含值的類型，例如將[Long](../language-reference/statements/option-strict-statement.md)轉換成 `On`。</span><span class="sxs-lookup"><span data-stu-id="3986e-103">You have tried to convert a type to another type that may not be able to contain the value, such as a `Long` to an `Integer`, while the type checking switch ([Option Strict Statement](../language-reference/statements/option-strict-statement.md)) is set to `On`.</span></span>  
  
 <span data-ttu-id="3986e-104">這種轉換類型稱為 *「縮小轉換」*(narrowing conversion)，可能會在執行階段失敗。</span><span class="sxs-lookup"><span data-stu-id="3986e-104">This type of conversion is called a *narrowing conversion*, and it is possible for it to fail at run time.</span></span> <span data-ttu-id="3986e-105">因此， `Option Strict On` 不允許隱含的縮小轉換。</span><span class="sxs-lookup"><span data-stu-id="3986e-105">For this reason, `Option Strict On` disallows implicit narrowing conversions.</span></span>  
  
 <span data-ttu-id="3986e-106">**錯誤 ID︰** BC30512</span><span class="sxs-lookup"><span data-stu-id="3986e-106">**Error ID:** BC30512</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="3986e-107">更正這個錯誤</span><span class="sxs-lookup"><span data-stu-id="3986e-107">To correct this error</span></span>  
  
1. <span data-ttu-id="3986e-108">判斷是否有從 `<type1>` 到 `<type2>`的任何類型轉換。</span><span class="sxs-lookup"><span data-stu-id="3986e-108">Determine whether a conversion of any type exists from `<type1>` to `<type2>`.</span></span> <span data-ttu-id="3986e-109">如果兩者都是 Visual Basic 的基本類型，或兩者都是類別的實例，您通常可以藉由在[擴展和縮小轉換](../programming-guide/language-features/data-types/widening-and-narrowing-conversions.md)中查閱資料表來進行這項判斷。</span><span class="sxs-lookup"><span data-stu-id="3986e-109">If both are Visual Basic elementary types, or if both are instances of classes, you can usually make this determination by consulting the table in [Widening and Narrowing Conversions](../programming-guide/language-features/data-types/widening-and-narrowing-conversions.md).</span></span>  
  
2. <span data-ttu-id="3986e-110">如果只有從 `<type1>` 到 `<type2>`的縮小轉換，您應該使用明確轉型。</span><span class="sxs-lookup"><span data-stu-id="3986e-110">If only a narrowing conversion exists from `<type1>` to `<type2>`, you should use explicit casting.</span></span> <span data-ttu-id="3986e-111">如果轉換失敗， [CType 函數](../language-reference/functions/ctype-function.md)和[DirectCast 運算子](../language-reference/operators/directcast-operator.md)關鍵字會擲回執行時間例外狀況。</span><span class="sxs-lookup"><span data-stu-id="3986e-111">The [CType Function](../language-reference/functions/ctype-function.md) and [DirectCast Operator](../language-reference/operators/directcast-operator.md) keywords throw a run-time exception if the conversion fails.</span></span> <span data-ttu-id="3986e-112">[TryCast Operator](../language-reference/operators/trycast-operator.md)關鍵字僅適用于參考型別，如果轉換失敗，則不傳回[任何內容](../language-reference/nothing.md)。</span><span class="sxs-lookup"><span data-stu-id="3986e-112">The [TryCast Operator](../language-reference/operators/trycast-operator.md) keyword applies only to reference types and returns [Nothing](../language-reference/nothing.md) if the conversion fails.</span></span>  
  
3. <span data-ttu-id="3986e-113">如果有縮小轉換，而且您的程式可以容許執行階段失敗，或您確定執行階段不可能失敗，您可以在原始程式碼開頭指定 `Option Strict Off` 。</span><span class="sxs-lookup"><span data-stu-id="3986e-113">If a narrowing conversion exists and your program can tolerate a run-time failure, or you are confident that a run-time failure is not possible, you can specify `Option Strict Off` at the beginning of your source code.</span></span> <span data-ttu-id="3986e-114">但您仍然應該將轉換放在[Try .。。Catch .。。Finally 語句](../language-reference/statements/try-catch-finally-statement.md)區塊，以避免非預期的結果或程式提前終止。</span><span class="sxs-lookup"><span data-stu-id="3986e-114">But you should still enclose the conversion in a [Try...Catch...Finally Statement](../language-reference/statements/try-catch-finally-statement.md) block to avoid unexpected results or early termination of your program.</span></span>  
  
4. <span data-ttu-id="3986e-115">如果沒有從 `<type1>` 到 `<type2>`的轉換，您必須重新評估程式邏輯。</span><span class="sxs-lookup"><span data-stu-id="3986e-115">If no conversion exists from `<type1>` to `<type2>`, you must re-evaluate your program logic.</span></span> <span data-ttu-id="3986e-116">您或許可以撰寫程式碼，將值指派給對應至 `<type2>` 之預期值的 `<type1>`。</span><span class="sxs-lookup"><span data-stu-id="3986e-116">You might be able to write code that can assign values to `<type2>` corresponding to anticipated values of `<type1>`.</span></span>  
  
5. <span data-ttu-id="3986e-117">如果沒有從 `<type1>` 到 `<type2>` 的轉換，而且其中一個類型是已定義的類別或結構，您或許可以定義轉換運算子，在該類型與其他類型之間進行轉換。</span><span class="sxs-lookup"><span data-stu-id="3986e-117">If no conversion exists from `<type1>` to `<type2>` and one of the types is a class or structure you have defined, you might be able to define a conversion operator from that type to or from the other type.</span></span> <span data-ttu-id="3986e-118">如需詳細資訊，請參閱 [How to: Define a Conversion Operator](../programming-guide/language-features/procedures/how-to-define-a-conversion-operator.md)。</span><span class="sxs-lookup"><span data-stu-id="3986e-118">For more information, see [How to: Define a Conversion Operator](../programming-guide/language-features/procedures/how-to-define-a-conversion-operator.md).</span></span>  
  
6. <span data-ttu-id="3986e-119">一般來說，在所有情況下，除非您可以攔截 `Catch` 區塊中的失敗並有效地加以處理，否則您應該避免使用縮小轉換。</span><span class="sxs-lookup"><span data-stu-id="3986e-119">In all cases and as a general guideline, you should avoid using narrowing conversions unless you can trap failures in a `Catch` block and deal with them effectively.</span></span>
