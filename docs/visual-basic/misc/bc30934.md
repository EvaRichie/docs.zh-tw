# <a name="conversion-from-type1-to-type2-cannot-occur-in-a-constant-expression-used-as-an-argument-to-an-attribute"></a><span data-ttu-id="6cdba-102">在當做屬性引數使用的常數運算式中，不可以發生從 '\<type1>' 到 '\<type2>' 的轉換</span><span class="sxs-lookup"><span data-stu-id="6cdba-102">Conversion from '\<type1>' to '\<type2>' cannot occur in a constant expression used as an argument to an attribute</span></span>

<span data-ttu-id="6cdba-103">屬性引數所使用的運算式會評估為與對應屬性參數不同的資料類型，而 Visual Basic 不允許屬性引數的必要型別轉換。</span><span class="sxs-lookup"><span data-stu-id="6cdba-103">An expression used for an attribute argument evaluates to a data type different from that of the corresponding attribute parameter, and Visual Basic does not allow the required type conversion for attribute arguments.</span></span>  
  
 <span data-ttu-id="6cdba-104">屬性提供所套用項目的中繼資料，而且編譯器必須可以在編譯時期建構所有中繼資料。</span><span class="sxs-lookup"><span data-stu-id="6cdba-104">An attribute provides metadata for the element it is applied to, and the compiler must be able to construct all the metadata at compile time.</span></span> <span data-ttu-id="6cdba-105">因此，每個屬性都必須使用在編譯時期為常數的值，因此，每個屬性引數都必須評估為編譯時期常數值。</span><span class="sxs-lookup"><span data-stu-id="6cdba-105">For this reason, every attribute must use values that are constant at compile time, and therefore every attribute argument must evaluate to a compile-time constant value.</span></span>  
  
 <span data-ttu-id="6cdba-106">特定類型轉換無法產生在編譯時期為常數的值。</span><span class="sxs-lookup"><span data-stu-id="6cdba-106">Certain type conversions cannot produce values that are constant at compile time.</span></span> <span data-ttu-id="6cdba-107">例如，將 `String` 轉換為 `Double` 或 `Date` 取決於執行階段的地區設定。</span><span class="sxs-lookup"><span data-stu-id="6cdba-107">For example, converting a `String` to a `Double` or a `Date` depends on the locale setting at run time.</span></span> <span data-ttu-id="6cdba-108">其他轉換 (例如衍生類型陣列到 `Object`陣列) 會呈現各種問題，而不允許編譯器在屬性引數上允許它們。</span><span class="sxs-lookup"><span data-stu-id="6cdba-108">Other conversions, such as an array of a derived type to an array of `Object`, present a variety of problems that do not permit the compiler to allow them on attribute arguments.</span></span>
