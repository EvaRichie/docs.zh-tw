# <a name="type-typename1-cannot-be-marked-cls-compliant-because-its-containing-type-typename2-is-not-cls-compliant"></a>類型 '\<typename1>' 無法標記為符合 CLS 規範，因為其包含類型 '\<typename2>' 不符合 CLS 規範

當類別或介面巢狀包含於標記為 `<CLSCompliant(True)>` 或未標記的類型時，已標記為 `<CLSCompliant(False)>` 。  
  
 為了讓類別或介面符合 [語言獨立性以及與語言無關的元件](../../standard/language-independence-and-language-independent-components.md) (cls) ，其整個內含專案階層必須符合規範。 也就是說，其巢狀所在的每種類型都必須符合規範。  
  
 將 <xref:System.CLSCompliantAttribute> 套用至程式設計項目時，請將屬性的 `isCompliant` 參數設定為 `True` 或 `False` ，表示符合標準或不符合標準。 這個參數沒有預設值，您必須提供值。  
  
 如果您未將 <xref:System.CLSCompliantAttribute> 套用至項目，則視為不符合標準。  
  
 根據預設，這個訊息是一個警告。 如需隱藏警告或將警告視為錯誤的相關資訊，請參閱 [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic)。  
  
 **錯誤 ID︰** BC40030  
  
## <a name="to-correct-this-error"></a>更正這個錯誤  
  
- 如果您必須符合 CLS 標準，請在不同的內含項目階層中定義這個類型。  
  
- 如果您想要將這個類型保留在其目前的內含項目階層中，請從其定義移除 <xref:System.CLSCompliantAttribute> 或將其標記為 `<CLSCompliant(False)>`。
