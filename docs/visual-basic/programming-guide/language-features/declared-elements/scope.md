<span data-ttu-id="5ddc9-173">針對任何類型的暫時計算而言，本機變數都是不錯的選擇，原因如下：</span><span class="sxs-lookup"><span data-stu-id="5ddc9-173">Local variables are a good choice for any kind of temporary calculation, for the following reasons:</span></span>

- <span data-ttu-id="5ddc9-174">**避免名稱衝突。**</span><span class="sxs-lookup"><span data-stu-id="5ddc9-174">**Name Conflict Avoidance.**</span></span> <span data-ttu-id="5ddc9-175">本機變數名稱不容易發生衝突。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-175">Local variable names are not susceptible to conflict.</span></span> <span data-ttu-id="5ddc9-176">例如，您可以建立數個不同的程式，其中包含名為的變數 `intTemp` 。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-176">For example, you can create several different procedures containing a variable called `intTemp`.</span></span> <span data-ttu-id="5ddc9-177">只要每個 `intTemp` 都宣告為區域變數，每個程式就只會辨識自己的版本 `intTemp` 。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-177">As long as each `intTemp` is declared as a local variable, each procedure recognizes only its own version of `intTemp`.</span></span> <span data-ttu-id="5ddc9-178">任何一個程式都可以在其本機中改變值， `intTemp` 而不會影響 `intTemp` 其他程式中的變數。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-178">Any one procedure can alter the value in its local `intTemp` without affecting `intTemp` variables in other procedures.</span></span>

- <span data-ttu-id="5ddc9-179">**記憶體耗用量。**</span><span class="sxs-lookup"><span data-stu-id="5ddc9-179">**Memory Consumption.**</span></span> <span data-ttu-id="5ddc9-180">本機變數只有在其程式正在執行時才會耗用記憶體。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-180">Local variables consume memory only while their procedure is running.</span></span> <span data-ttu-id="5ddc9-181">當程式返回呼叫的程式碼時，就會釋放其記憶體。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-181">Their memory is released when the procedure returns to the calling code.</span></span> <span data-ttu-id="5ddc9-182">相反地，[共用](../../../language-reference/modifiers/shared.md)和[靜態](../../../language-reference/modifiers/static.md)變數會耗用記憶體資源，直到您的應用程式停止執行為止，因此只有在必要時才使用它們。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-182">By contrast, [Shared](../../../language-reference/modifiers/shared.md) and [Static](../../../language-reference/modifiers/static.md) variables consume memory resources until your application stops running, so use them only when necessary.</span></span> <span data-ttu-id="5ddc9-183">*執行個體變數*會耗用記憶體，而其實例會繼續存在，讓它們比區域變數更有效率，但可能比 `Shared` 或變數更有效率 `Static` 。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-183">*Instance variables* consume memory while their instance continues to exist, which makes them less efficient than local variables, but potentially more efficient than `Shared` or `Static` variables.</span></span>

### <a name="minimizing-scope"></a><span data-ttu-id="5ddc9-184">最小化範圍</span><span class="sxs-lookup"><span data-stu-id="5ddc9-184">Minimizing Scope</span></span>

<span data-ttu-id="5ddc9-185">一般來說，在宣告任何變數或常數時，最好的程式設計作法是讓範圍盡可能縮小（區塊範圍是最窄的）。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-185">In general, when declaring any variable or constant, it is good programming practice to make the scope as narrow as possible (block scope is the narrowest).</span></span> <span data-ttu-id="5ddc9-186">這有助於節省記憶體，並將程式碼錯誤地參考錯誤變數的機率降到最低。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-186">This helps conserve memory and minimizes the chances of your code erroneously referring to the wrong variable.</span></span> <span data-ttu-id="5ddc9-187">同樣地，您應該將變數宣告為[靜態](../../../language-reference/modifiers/static.md)，只有在必要時，才必須在程序呼叫之間保留其值。</span><span class="sxs-lookup"><span data-stu-id="5ddc9-187">Similarly, you should declare a variable to be [Static](../../../language-reference/modifiers/static.md) only when it is necessary to preserve its value between procedure calls.</span></span>
