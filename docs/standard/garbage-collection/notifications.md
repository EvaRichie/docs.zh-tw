 <span data-ttu-id="834e1-106"><xref:System.GC.RegisterForFullGCNotification%2A> 方法會註冊一個當執行階段偵測到接近完整記憶體回收時要引發的通知。</span><span class="sxs-lookup"><span data-stu-id="834e1-106">The <xref:System.GC.RegisterForFullGCNotification%2A> method registers for a notification to be raised when the runtime senses that a full garbage collection is approaching.</span></span> <span data-ttu-id="834e1-107">通知有兩個部分：當接近完整記憶體回收時，以及當完整記憶體回收完成時。</span><span class="sxs-lookup"><span data-stu-id="834e1-107">There are two parts to this notification: when the full garbage collection is approaching and when the full garbage collection has completed.</span></span>

 <span data-ttu-id="834e1-110">若要判斷引發通知的時機，請使用 <xref:System.GC.WaitForFullGCApproach%2A> 和 <xref:System.GC.WaitForFullGCComplete%2A> 方法。</span><span class="sxs-lookup"><span data-stu-id="834e1-110">To determine when a notification has been raised, use the <xref:System.GC.WaitForFullGCApproach%2A> and <xref:System.GC.WaitForFullGCComplete%2A> methods.</span></span> <span data-ttu-id="834e1-111">一般來說，您會在 `while` 迴圈中使用這些方法，以持續取得可顯示通知狀態的 <xref:System.GCNotificationStatus> 列舉。</span><span class="sxs-lookup"><span data-stu-id="834e1-111">Typically, you use these methods in a `while` loop to continually obtain a <xref:System.GCNotificationStatus> enumeration that shows the status of the notification.</span></span> <span data-ttu-id="834e1-112">如果值為 <xref:System.GCNotificationStatus.Succeeded>，您可以執行以下動作：</span><span class="sxs-lookup"><span data-stu-id="834e1-112">If that value is <xref:System.GCNotificationStatus.Succeeded>, you can do the following:</span></span>

## <a name="full-garbage-collection"></a><span data-ttu-id="834e1-119">完整記憶體回收</span><span class="sxs-lookup"><span data-stu-id="834e1-119">Full Garbage Collection</span></span>  
 <span data-ttu-id="834e1-120">當下列任何案例都成立時，執行階段會導致完整記憶體回收：</span><span class="sxs-lookup"><span data-stu-id="834e1-120">The runtime causes a full garbage collection when any of the following scenarios are true:</span></span>  
  
- <span data-ttu-id="834e1-121">足夠的記憶體已升階至層代 2，導致下一個層代 2 回收。</span><span class="sxs-lookup"><span data-stu-id="834e1-121">Enough memory has been promoted into generation 2 to cause the next generation 2 collection.</span></span>  
  
- <span data-ttu-id="834e1-122">足夠的記憶體已升階至大型物件堆積，導致下一個層代 2 回收。</span><span class="sxs-lookup"><span data-stu-id="834e1-122">Enough memory has been promoted into the large object heap to cause the next generation 2 collection.</span></span>  
  
- <span data-ttu-id="834e1-123">因為其他因素，層代 1 回收已擴大為層代 2 回收。</span><span class="sxs-lookup"><span data-stu-id="834e1-123">A collection of generation 1 is escalated to a collection of generation 2 due to other factors.</span></span>  
  
 <span data-ttu-id="834e1-124">您在 <xref:System.GC.RegisterForFullGCNotification%2A> 方法中指定的閾值會套用到前兩個案例。</span><span class="sxs-lookup"><span data-stu-id="834e1-124">The thresholds you specify in the <xref:System.GC.RegisterForFullGCNotification%2A> method apply to the first two scenarios.</span></span> <span data-ttu-id="834e1-125">不過，在第一個案例中，您不一定會在指定的閾值成比例的時間收到通知，原因有兩個：</span><span class="sxs-lookup"><span data-stu-id="834e1-125">However, in the first scenario you will not always receive the notification at the time proportional to the threshold values you specify for two reasons:</span></span>  
  
- <span data-ttu-id="834e1-126">執行階段不會檢查每個小型物件配置 (基於效能考量)。</span><span class="sxs-lookup"><span data-stu-id="834e1-126">The runtime does not check each small object allocation (for performance reasons).</span></span>  
  
- <span data-ttu-id="834e1-127">只有層代 1 回收會將記憶體升階為層代 2。</span><span class="sxs-lookup"><span data-stu-id="834e1-127">Only generation 1 collections promote memory into generation 2.</span></span>  
  
 <span data-ttu-id="834e1-128">第三個案例也會造成您收到通知之時間的不確定性。</span><span class="sxs-lookup"><span data-stu-id="834e1-128">The third scenario also contributes to the uncertainty of when you will receive the notification.</span></span> <span data-ttu-id="834e1-129">雖然這不是一種保證，但是透過在這段期間將要求重新導向或在更能容納的時候自行引發，確實已證明這是一種減輕不適當的完整記憶體回收之效果的有用方法。</span><span class="sxs-lookup"><span data-stu-id="834e1-129">Although this is not a guarantee, it does prove to be a useful way to mitigate the effects of an inopportune full garbage collection by redirecting the requests during this time or inducing the collection yourself when it can be better accommodated.</span></span>
