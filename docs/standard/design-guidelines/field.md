# <a name="field-design"></a><span data-ttu-id="d7db3-102">欄位設計</span><span class="sxs-lookup"><span data-stu-id="d7db3-102">Field Design</span></span>
<span data-ttu-id="d7db3-103">封裝的原則是物件導向設計中最重要的概念之一。</span><span class="sxs-lookup"><span data-stu-id="d7db3-103">The principle of encapsulation is one of the most important notions in object-oriented design.</span></span> <span data-ttu-id="d7db3-104">此原則指出，儲存在物件內的資料應該只能供該物件存取。</span><span class="sxs-lookup"><span data-stu-id="d7db3-104">This principle states that data stored inside an object should be accessible only to that object.</span></span>

 <span data-ttu-id="d7db3-105">解讀原則的一個實用方法是說，型別應該設計成讓該類型的欄位變更（名稱或類型變更），而不需要中斷程式碼，而不是針對該類型的成員進行。</span><span class="sxs-lookup"><span data-stu-id="d7db3-105">A useful way to interpret the principle is to say that a type should be designed so that changes to fields of that type (name or type changes) can be made without breaking code other than for members of the type.</span></span> <span data-ttu-id="d7db3-106">這種轉譯會立即暗示所有欄位都必須是私用的。</span><span class="sxs-lookup"><span data-stu-id="d7db3-106">This interpretation immediately implies that all fields must be private.</span></span>

 <span data-ttu-id="d7db3-107">我們會從這項嚴格的限制排除常數和靜態唯讀欄位，因為這類欄位幾乎是定義的，因此永遠不需要變更。</span><span class="sxs-lookup"><span data-stu-id="d7db3-107">We exclude constant and static read-only fields from this strict restriction, because such fields, almost by definition, are never required to change.</span></span>

 <span data-ttu-id="d7db3-108">❌請勿提供公用或受保護的實例欄位。</span><span class="sxs-lookup"><span data-stu-id="d7db3-108">❌ DO NOT provide instance fields that are public or protected.</span></span>

 <span data-ttu-id="d7db3-109">您應該提供屬性來存取欄位，而不是將它們設為公用或保護。</span><span class="sxs-lookup"><span data-stu-id="d7db3-109">You should provide properties for accessing fields instead of making them public or protected.</span></span>

 <span data-ttu-id="d7db3-110">✔️對於永遠不會變更的常數，請使用常數位段。</span><span class="sxs-lookup"><span data-stu-id="d7db3-110">✔️ DO use constant fields for constants that will never change.</span></span>

 <span data-ttu-id="d7db3-111">編譯器會將 const 欄位的值直接帶到呼叫程式碼中。</span><span class="sxs-lookup"><span data-stu-id="d7db3-111">The compiler burns the values of const fields directly into calling code.</span></span> <span data-ttu-id="d7db3-112">因此，如果沒有中斷相容性的風險，就永遠不會變更 const 值。</span><span class="sxs-lookup"><span data-stu-id="d7db3-112">Therefore, const values can never be changed without the risk of breaking compatibility.</span></span>

 <span data-ttu-id="d7db3-113">✔️ `readonly` 針對預先定義的物件實例使用公用靜態欄位。</span><span class="sxs-lookup"><span data-stu-id="d7db3-113">✔️ DO use public static `readonly` fields for predefined object instances.</span></span>

 <span data-ttu-id="d7db3-114">如果有預先定義的類型實例，請將其宣告為類型本身的公用唯讀靜態欄位。</span><span class="sxs-lookup"><span data-stu-id="d7db3-114">If there are predefined instances of the type, declare them as public read-only static fields of the type itself.</span></span>

 <span data-ttu-id="d7db3-115">❌請勿將可變類型的實例指派給 `readonly` 欄位。</span><span class="sxs-lookup"><span data-stu-id="d7db3-115">❌ DO NOT assign instances of mutable types to `readonly` fields.</span></span>

 <span data-ttu-id="d7db3-116">可變型別是具有實例的型別，可在具現化之後加以修改。</span><span class="sxs-lookup"><span data-stu-id="d7db3-116">A mutable type is a type with instances that can be modified after they are instantiated.</span></span> <span data-ttu-id="d7db3-117">例如，陣列、大部分的集合和資料流程都是可變動的類型，但 <xref:System.Int32?displayProperty=nameWithType> 、 <xref:System.Uri?displayProperty=nameWithType> 和 <xref:System.String?displayProperty=nameWithType> 都是不可變的。</span><span class="sxs-lookup"><span data-stu-id="d7db3-117">For example, arrays, most collections, and streams are mutable types, but <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Uri?displayProperty=nameWithType>, and <xref:System.String?displayProperty=nameWithType> are all immutable.</span></span> <span data-ttu-id="d7db3-118">[參考型別] 欄位上的唯讀修飾詞會防止儲存在欄位中的實例被取代，但它不會藉由呼叫變更實例的成員來防止修改欄位的實例資料。</span><span class="sxs-lookup"><span data-stu-id="d7db3-118">The read-only modifier on a reference type field prevents the instance stored in the field from being replaced, but it does not prevent the field’s instance data from being modified by calling members changing the instance.</span></span>
