## <a name="dont-assume-that-iterations-of-foreach-for-and-forall-always-execute-in-parallel"></a><span data-ttu-id="15aa8-154">請勿假設 ForEach、For 和 ForAll 的反復專案一律以平行方式執行</span><span class="sxs-lookup"><span data-stu-id="15aa8-154">Don't assume that iterations of ForEach, For, and ForAll always execute in parallel</span></span>

```vb
Dim mre = New ManualResetEventSlim()

   If j = Environment.ProcessorCount Then
       Console.WriteLine("Set on {0} with value of {1}", Thread.CurrentThread.ManagedThreadId, j)
       mre.Set()
   Else
       Console.WriteLine("Waiting on {0} with value of {1}", Thread.CurrentThread.ManagedThreadId, j)
       mre.Wait()
   End If
End Sub) ' deadlocks
```

```csharp

    if (j == Environment.ProcessorCount)

        Console.WriteLine("Set on {0} with value of {1}", Thread.CurrentThread.ManagedThreadId, j);

    }

    }
```

<span data-ttu-id="15aa8-158">在此範例中，一個反覆項目會設定事件，而其他所有反覆項目則是等候事件。</span><span class="sxs-lookup"><span data-stu-id="15aa8-158">In this example, one iteration sets an event, and all other iterations wait on the event.</span></span> <span data-ttu-id="15aa8-159">在事件設定反覆項目完成之前，沒有任何等候中的反覆項目可以完成。</span><span class="sxs-lookup"><span data-stu-id="15aa8-159">None of the waiting iterations can complete until the event-setting iteration has completed.</span></span> <span data-ttu-id="15aa8-160">不過，等待中的反覆項目有可能會在事件設定反覆項目有機會執行之前，就封鎖所有用來執行平行迴圈的執行緒。</span><span class="sxs-lookup"><span data-stu-id="15aa8-160">However, it is possible that the waiting iterations block all threads that are used to execute the parallel loop, before the event-setting iteration has had a chance to execute.</span></span> <span data-ttu-id="15aa8-161">這會導致死結，事件設定反覆項目永遠不會執行，而等待中的反覆項目也永遠不會醒來。</span><span class="sxs-lookup"><span data-stu-id="15aa8-161">This results in a deadlock – the event-setting iteration will never execute, and the waiting iterations will never wake up.</span></span>

<span data-ttu-id="15aa8-162">特別是，平行迴圈的反覆項目永遠不應該等候該迴圈的另一個反覆項目才能繼續。</span><span class="sxs-lookup"><span data-stu-id="15aa8-162">In particular, one iteration of a parallel loop should never wait on another iteration of the loop to make progress.</span></span> <span data-ttu-id="15aa8-163">如果平行迴圈決定以循序方式排程反覆項目，但順序相反，系統就會發生死結。</span><span class="sxs-lookup"><span data-stu-id="15aa8-163">If the parallel loop decides to schedule the iterations sequentially but in the opposite order, a deadlock will occur.</span></span>
