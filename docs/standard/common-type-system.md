## <a name="common-type-system"></a><span data-ttu-id="5ef00-105">一般類型系統</span><span class="sxs-lookup"><span data-stu-id="5ef00-105">Common Type System</span></span>

<span data-ttu-id="5ef00-109">若要透明地執行這項操作，必須要有通用的方式來描述所有支援的類型。</span><span class="sxs-lookup"><span data-stu-id="5ef00-109">In order to do this transparently, there has to be a common way to describe all supported types.</span></span> <span data-ttu-id="5ef00-110">這是一般類型系統 (CTS) 負責執行的作業。</span><span class="sxs-lookup"><span data-stu-id="5ef00-110">This is what the Common Type System (CTS) is in charge of doing.</span></span> <span data-ttu-id="5ef00-111">它是用來執行下列事項︰</span><span class="sxs-lookup"><span data-stu-id="5ef00-111">It was made to do several things:</span></span>

* <span data-ttu-id="5ef00-112">建立跨語言執行的架構。</span><span class="sxs-lookup"><span data-stu-id="5ef00-112">Establish a framework for cross-language execution.</span></span>

<span data-ttu-id="5ef00-116">CTS 定義應該支援的兩種主要類型︰參考和實值類型。</span><span class="sxs-lookup"><span data-stu-id="5ef00-116">CTS defines two main kinds of types that should be supported: reference and value types.</span></span> <span data-ttu-id="5ef00-117">它們的名稱會指向其定義。</span><span class="sxs-lookup"><span data-stu-id="5ef00-117">Their names point to their definitions.</span></span>

<span data-ttu-id="5ef00-118">參考型別的物件是以物件實際值的參考來表示。這裡的參考類似于 C/c + + 中的指標。</span><span class="sxs-lookup"><span data-stu-id="5ef00-118">Reference types' objects are represented by a reference to the object's actual value; a reference here is similar to a pointer in C/C++.</span></span> <span data-ttu-id="5ef00-119">它只是指物件值所在的記憶體位置。</span><span class="sxs-lookup"><span data-stu-id="5ef00-119">It simply refers to a memory location where the objects' values are.</span></span> <span data-ttu-id="5ef00-120">這對這些類型的使用方式具有深遠影響。</span><span class="sxs-lookup"><span data-stu-id="5ef00-120">This has a profound impact on how these types are used.</span></span> <span data-ttu-id="5ef00-121">如果您將參考類型指派給變數，然後將該變數傳遞給方法，例如，物件的任何變更都會反映在主要物件中；未進行複製。</span><span class="sxs-lookup"><span data-stu-id="5ef00-121">If you assign a reference type to a variable and then pass that variable into a method, for instance, any changes to the object will be reflected on the main object; there is no copying.</span></span>

<span data-ttu-id="5ef00-122">如果物件是由物件的值來表示，則實值類型就是相反值。</span><span class="sxs-lookup"><span data-stu-id="5ef00-122">Value types are the opposite, where the objects are represented by their values.</span></span> <span data-ttu-id="5ef00-123">如果您將實值類型指派給變數，則基本上會複製物件的值。</span><span class="sxs-lookup"><span data-stu-id="5ef00-123">If you assign a value type to a variable, you are essentially copying a value of the object.</span></span>

<span data-ttu-id="5ef00-124">CTS 定義數種分類的類型，各有其特定語意和用法︰</span><span class="sxs-lookup"><span data-stu-id="5ef00-124">CTS defines several categories of types, each with their specific semantics and usage:</span></span>

* <span data-ttu-id="5ef00-125">類別</span><span class="sxs-lookup"><span data-stu-id="5ef00-125">Classes</span></span>
* <span data-ttu-id="5ef00-126">結構</span><span class="sxs-lookup"><span data-stu-id="5ef00-126">Structures</span></span>

* <span data-ttu-id="5ef00-128">介面</span><span class="sxs-lookup"><span data-stu-id="5ef00-128">Interfaces</span></span>
* <span data-ttu-id="5ef00-129">委派</span><span class="sxs-lookup"><span data-stu-id="5ef00-129">Delegates</span></span>

<span data-ttu-id="5ef00-130">CTS 也會定義類型的所有其他屬性 (例如存取修飾詞)、什麼是有效類型成員、繼承和多載的運作方式等。</span><span class="sxs-lookup"><span data-stu-id="5ef00-130">CTS also defines all other properties of the types, such as access modifiers, what are valid type members, how inheritance and overloading works and so on.</span></span> <span data-ttu-id="5ef00-131">不幸的是，深入探討其中任何項目不在簡介文章的範圍 (例如這項)，但您可以查閱涵蓋這些主題之更深入內容連結結尾的[更多資源](#more-resources)一節。</span><span class="sxs-lookup"><span data-stu-id="5ef00-131">Unfortunately, going deep into any of those is beyond the scope of an introductory article such as this, but you can consult [More resources](#more-resources) section at the end for links to more in-depth content that covers these topics.</span></span>

## <a name="common-language-specification"></a><span data-ttu-id="5ef00-132">Common Language Specification</span><span class="sxs-lookup"><span data-stu-id="5ef00-132">Common Language Specification</span></span>

<span data-ttu-id="5ef00-133">若要啟用完整互通性案例，則透過程式碼建立的所有物件都必須依賴使用它們之語言的一些共通性 (即其 _呼叫端_)。</span><span class="sxs-lookup"><span data-stu-id="5ef00-133">To enable full interoperability scenarios, all objects that are created in code must rely on some commonality in the languages that are consuming them (are their _callers_).</span></span> <span data-ttu-id="5ef00-134">因為有多種不同的語言，所以 .NET 已透過 **Common Language Specification** (CLS) 指定這些共通性。</span><span class="sxs-lookup"><span data-stu-id="5ef00-134">Since there are numerous different languages, .NET has specified those commonalities in something called the **Common Language Specification** (CLS).</span></span> <span data-ttu-id="5ef00-135">CLS 定義許多常見應用程式所需的一組功能。</span><span class="sxs-lookup"><span data-stu-id="5ef00-135">CLS defines a set of features that are needed by many common applications.</span></span> <span data-ttu-id="5ef00-136">它也提供一種方法，以在需要支援的 .NET 上實作任何語言。</span><span class="sxs-lookup"><span data-stu-id="5ef00-136">It also provides a sort of recipe for any language that is implemented on top of .NET on what it needs to support.</span></span>

<span data-ttu-id="5ef00-137">CLS 是 CTS 的子集。</span><span class="sxs-lookup"><span data-stu-id="5ef00-137">CLS is a subset of the CTS.</span></span> <span data-ttu-id="5ef00-138">除非 CLS 規則更嚴格，否則這表示 CTS 中的所有規則也適用於 CLS。</span><span class="sxs-lookup"><span data-stu-id="5ef00-138">This means that all of the rules in the CTS also apply to the CLS, unless the CLS rules are more strict.</span></span> <span data-ttu-id="5ef00-139">如果元件只是使用 CLS 中的規則所建置 (亦即，它只會在其 API 中公開 CLS 功能)，表示 **符合 CLS 規範**。</span><span class="sxs-lookup"><span data-stu-id="5ef00-139">If a component is built using only the rules in the CLS, that is, it exposes only the CLS features in its API, it is said to be **CLS-compliant**.</span></span> <span data-ttu-id="5ef00-140">例如，`<framework-librares>` 完全符合 CLS 標準，原因是它們需要作用於 .NET 上支援的所有語言。</span><span class="sxs-lookup"><span data-stu-id="5ef00-140">For instance, the `<framework-librares>` are CLS-compliant precisely because they need to work across all of the languages that are supported on .NET.</span></span>
