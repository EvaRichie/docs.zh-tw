## <a name="security-considerations-for-xmlserializer-applications"></a><span data-ttu-id="dbb9b-139">XmlSerializer 應用程式的安全性考量</span><span class="sxs-lookup"><span data-stu-id="dbb9b-139">Security Considerations for XmlSerializer Applications</span></span>

<span data-ttu-id="dbb9b-140">建立使用**XmlSerializer**的應用程式時，請注意下列專案及其含意：</span><span class="sxs-lookup"><span data-stu-id="dbb9b-140">When creating an application that uses the **XmlSerializer**, be aware of the following items and their implications:</span></span>

- <span data-ttu-id="dbb9b-141">**XmlSerializer** 建立 C# (.cs) 檔案並在以 TEMP 環境變數命名的目錄中，將它們編譯成為 .dll 檔案；那些 DLL 會發生序列化。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-141">The **XmlSerializer** creates C# (.cs) files and compiles them into .dll files in the directory named by the TEMP environment variable; serialization occurs with those DLLs.</span></span>

  > [!NOTE]
  > <span data-ttu-id="dbb9b-142">這些序列化組件可預先產生，並且使用 SGen.exe 工具簽署。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-142">These serialization assemblies can be generated in advance and signed by using the SGen.exe tool.</span></span> <span data-ttu-id="dbb9b-143">這並不是當做 Web 服務的伺服器。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-143">This does not work a server of Web services.</span></span> <span data-ttu-id="dbb9b-144">換句話說，它只是供用戶端使用以及手動序列化。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-144">In other words, it is only for client use and for manual serialization.</span></span>

  <span data-ttu-id="dbb9b-145">在建立與編譯時，程式碼與 DLL 容易受到惡意處理序的攻擊。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-145">The code and the DLLs are vulnerable to a malicious process at the time of creation and compilation.</span></span> <span data-ttu-id="dbb9b-146">使用執行 Microsoft Windows NT 4.0 或以上版本的電腦時，有可能讓兩位以上的使用者共用 TEMP 目錄。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-146">When using a computer running Microsoft Windows NT 4.0 or later, it might be possible for two or more users to share the TEMP directory.</span></span> <span data-ttu-id="dbb9b-147">如果兩個帳戶擁有不同的安全性權限，而且權限較高的帳戶使用 **XmlSerializer** 執行應用程式，共用 TEMP 目錄會有危險性。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-147">Sharing a TEMP directory is dangerous if the two accounts have different security privileges and the higher-privilege account runs an application using the **XmlSerializer**.</span></span> <span data-ttu-id="dbb9b-148">在此例中，一個使用者只要取代編譯的 .cs 或 .dll 檔案，就會破壞電腦的安全性。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-148">In this case, one user can breach the computer's security by replacing either the .cs or .dll file that is compiled.</span></span> <span data-ttu-id="dbb9b-149">若要去除此顧慮，請確定電腦上各帳戶皆有個別的設定檔。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-149">To eliminate this concern, always be sure that each account on the computer has its own profile.</span></span> <span data-ttu-id="dbb9b-150">根據預設，TEMP 環境變數是每個帳戶指向不同目錄。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-150">By default, the TEMP environment variable points to a different directory for each account.</span></span>

- <span data-ttu-id="dbb9b-151">若惡意使用者對網頁伺服器發送持續 XML 資料流 (阻絕服務攻擊)，那麼 **XmlSerializer** 會持續處理資料直到電腦資源消耗殆盡。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-151">If a malicious user sends a continuous stream of XML data to a Web server (a denial of service attack), then the **XmlSerializer** continues to process the data until the computer runs low on resources.</span></span>

  <span data-ttu-id="dbb9b-152">若您使用執行 Internet Information Services (IIS) 的電腦，而且您的應用程式是在 IIS 裡執行，就可排除這類攻擊。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-152">This kind of attack is eliminated if you are using a computer running Internet Information Services (IIS), and your application is running within IIS.</span></span> <span data-ttu-id="dbb9b-153">IIS 採用閘道，超過設定量 (預設為 4 KB) 的資料流就不處理。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-153">IIS features a gate that does not process streams longer than a set amount (the default is 4 KB).</span></span> <span data-ttu-id="dbb9b-154">若您建立不使用 IIS 的應用程式，而且以 **XmlSerializer** 還原序列化，那應實作類似閘道以防止阻絕服務攻擊。</span><span class="sxs-lookup"><span data-stu-id="dbb9b-154">If you create an application that does not use IIS and deserializes with the **XmlSerializer**, you should implement a similar gate that prevents a denial of service attack.</span></span>
