# <a name="backtracking-in-regular-expressions"></a><span data-ttu-id="9a6fb-103">規則運算式中的回溯</span><span class="sxs-lookup"><span data-stu-id="9a6fb-103">Backtracking in Regular Expressions</span></span>
<span data-ttu-id="9a6fb-104">回溯 (Backtracking) 會在規則運算式模式包含選擇性的[數量詞](quantifiers-in-regular-expressions.md)或[替代建構](alternation-constructs-in-regular-expressions.md)，且規則運算式引擎返回之前儲存的狀態繼續搜尋相符項目時發生。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-104">Backtracking occurs when a regular expression pattern contains optional [quantifiers](quantifiers-in-regular-expressions.md) or [alternation constructs](alternation-constructs-in-regular-expressions.md), and the regular expression engine returns to a previous saved state to continue its search for a match.</span></span> <span data-ttu-id="9a6fb-105">回溯是規則運算式的核心能力，可讓運算式功能強大且靈活，並且比對非常複雜的模式。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-105">Backtracking is central to the power of regular expressions; it makes it possible for expressions to be powerful and flexible, and to match very complex patterns.</span></span> <span data-ttu-id="9a6fb-106">但同時，這項強大功能需付出相當的代價。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-106">At the same time, this power comes at a cost.</span></span> <span data-ttu-id="9a6fb-107">回溯經常是影響規則運算式引擎之效能最重要的一項因素。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-107">Backtracking is often the single most important factor that affects the performance of the regular expression engine.</span></span> <span data-ttu-id="9a6fb-108">幸好開發人員能夠掌控規則運算式引擎的行為，以及其使用回溯的方式。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-108">Fortunately, the developer has control over the behavior of the regular expression engine and how it uses backtracking.</span></span> <span data-ttu-id="9a6fb-109">本主題將說明回溯運作的方式，以及如何進行控制。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-109">This topic explains how backtracking works and how it can be controlled.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="9a6fb-110">大致而言，像是 .NET 規則運算式引擎這類非決定性有限自動化 (NFA) 引擎將設計有效率且快速之規則運算式的責任交給了開發人員。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-110">In general, a Nondeterministic Finite Automaton (NFA) engine like .NET regular expression engine places the responsibility for crafting efficient, fast regular expressions on the developer.</span></span>

 <span data-ttu-id="9a6fb-112">如果規則運算式模式沒有選擇性數量詞或替代建構，則規則運算式引擎會以線性時間執行。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-112">If a regular expression pattern has no optional quantifiers or alternation constructs, the regular expression engine executes in linear time.</span></span> <span data-ttu-id="9a6fb-113">也就是說，規則運算式引擎比對模式中的第一個語言項目與輸入字串中的文字之後，會嘗試比對模式中的下一個語言項目與輸入字串中的下一個字元或字元群組。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-113">That is, after the regular expression engine matches the first language element in the pattern with text in the input string, it tries to match the next language element in the pattern with the next character or group of characters in the input string.</span></span> <span data-ttu-id="9a6fb-114">這項作業會繼續進行，直到比對成功或失敗為止。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-114">This continues until the match either succeeds or fails.</span></span> <span data-ttu-id="9a6fb-115">無論成功或失敗，規則運算式引擎都會在輸入字串中一次前進一個字元。</span><span class="sxs-lookup"><span data-stu-id="9a6fb-115">In either case, the regular expression engine advances by one character at a time in the input string.</span></span>
