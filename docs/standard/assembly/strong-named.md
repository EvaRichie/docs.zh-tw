# <a name="strong-named-assemblies"></a><span data-ttu-id="c001d-103">強式名稱組件</span><span class="sxs-lookup"><span data-stu-id="c001d-103">Strong-named assemblies</span></span>

<span data-ttu-id="c001d-104">強式命名組件會為組件建立唯一識別，並可防止發生組件衝突。</span><span class="sxs-lookup"><span data-stu-id="c001d-104">Strong-naming an assembly creates a unique identity for the assembly, and can prevent assembly conflicts.</span></span>

## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="c001d-105">構成強式名稱的組件之項目為何？</span><span class="sxs-lookup"><span data-stu-id="c001d-105">What makes a strong-named assembly?</span></span>

<span data-ttu-id="c001d-106">強式名稱的組件是透過私密金鑰所產生，該私密金鑰對應至隨組件或當做組件本身散發的公開金鑰。</span><span class="sxs-lookup"><span data-stu-id="c001d-106">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="c001d-107">該組件包含組件資訊清單，其中包含構成組件之所有檔案的名稱和雜湊。</span><span class="sxs-lookup"><span data-stu-id="c001d-107">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="c001d-108">具有相同強式名稱的組件應該是相同的。</span><span class="sxs-lookup"><span data-stu-id="c001d-108">Assemblies that have the same strong name should be identical.</span></span>

<span data-ttu-id="c001d-109">您可以使用 Visual Studio 或命令列工具強式命名組件。</span><span class="sxs-lookup"><span data-stu-id="c001d-109">You can strong-name assemblies by using Visual Studio or a command-line tool.</span></span> <span data-ttu-id="c001d-110">如需詳細資訊，請參閱[如何：使用強式名稱簽署元件](sign-strong-name.md)或[Sn.exe （強式名稱工具）](../../framework/tools/sn-exe-strong-name-tool.md)。</span><span class="sxs-lookup"><span data-stu-id="c001d-110">For more information, see [How to: Sign an assembly with a strong name](sign-strong-name.md) or [Sn.exe (Strong Name tool)](../../framework/tools/sn-exe-strong-name-tool.md).</span></span>

<span data-ttu-id="c001d-111">建立強式名稱的組件時，會包含組件的簡單文字名稱、版本號碼、文化特性資訊 (選擇性)、數位簽章，以及對應至用於簽署之私密金鑰的公開金鑰。</span><span class="sxs-lookup"><span data-stu-id="c001d-111">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

> [!WARNING]
> <span data-ttu-id="c001d-112">請勿依賴強式名稱提供安全性。</span><span class="sxs-lookup"><span data-stu-id="c001d-112">Do not rely on strong names for security.</span></span> <span data-ttu-id="c001d-113">強式名稱僅提供唯一識別。</span><span class="sxs-lookup"><span data-stu-id="c001d-113">They provide a unique identity only.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="c001d-114">為什麼需要強式命名您的組件？</span><span class="sxs-lookup"><span data-stu-id="c001d-114">Why strong-name your assemblies?</span></span>

<span data-ttu-id="c001d-115">當您參考強式名稱的組件時，您會希望取得某些優點，例如版本控制和命名保護。</span><span class="sxs-lookup"><span data-stu-id="c001d-115">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="c001d-116">在 .NET Framework 中，可將強式名稱的元件安裝在全域組件快取中，這是啟用某些案例的必要條件。</span><span class="sxs-lookup"><span data-stu-id="c001d-116">In the .NET Framework, strong-named assemblies can be installed in the global assembly cache, which is required to enable some scenarios.</span></span>

<span data-ttu-id="c001d-117">強式名稱的組件在下列情況下很實用：</span><span class="sxs-lookup"><span data-stu-id="c001d-117">Strong-named assemblies are useful in the following scenarios:</span></span>

- <span data-ttu-id="c001d-118">您想要啟用組件以供強式名稱的組件參考，或者您想要授與 `friend` 存取權，以從其他強式名稱的組件存取您的組件。</span><span class="sxs-lookup"><span data-stu-id="c001d-118">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>

- <span data-ttu-id="c001d-119">應用程式需要存取相同組件的不同版本。</span><span class="sxs-lookup"><span data-stu-id="c001d-119">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="c001d-120">這表示您需要在不起衝突的情況下，在同一個應用程式網域中並存載入組件的不同版本。</span><span class="sxs-lookup"><span data-stu-id="c001d-120">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="c001d-121">例如，如果組件中存在具有相同簡單名稱的不同應用程式開發介面擴充模組，強式命名會為每個組件版本提供唯一識別。</span><span class="sxs-lookup"><span data-stu-id="c001d-121">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>

- <span data-ttu-id="c001d-122">您不希望在使用組件時，對應用程式效能造成負面影響，因此想要使用領域中性的組件。</span><span class="sxs-lookup"><span data-stu-id="c001d-122">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="c001d-123">由於領域中性的組件必須安裝在全域組件快取中，因此需要對其強式命名。</span><span class="sxs-lookup"><span data-stu-id="c001d-123">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>

- <span data-ttu-id="c001d-124">您想要藉由套用發行者原則來為您的應用程式集中化服務，這表示元件必須安裝在全域組件快取中。</span><span class="sxs-lookup"><span data-stu-id="c001d-124">You want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the global assembly cache.</span></span>

<span data-ttu-id="c001d-125">如果您是開放原始碼開發人員，而且想要強名稱元件的身分識別優點，請考慮將與元件相關聯的私密金鑰簽入到您的原始檔控制系統。</span><span class="sxs-lookup"><span data-stu-id="c001d-125">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly to your source control system.</span></span>
