### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="26e6c-101">SSE 和 SSE2 CompareGreaterThan 方法會適當地處理 NaN 輸入</span><span class="sxs-lookup"><span data-stu-id="26e6c-101">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="26e6c-102">下列 <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> 和 <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> 方法已修正，可適當地處理 `NaN` 輸入，並符合類別中對等方法的硬體行為 <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> ：</span><span class="sxs-lookup"><span data-stu-id="26e6c-102">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a><span data-ttu-id="26e6c-103">變更描述</span><span class="sxs-lookup"><span data-stu-id="26e6c-103">Change description</span></span>

<span data-ttu-id="26e6c-104">先前 `NaN` 列出的 <xref:System.Runtime.Intrinsics.X86.Sse> 和方法的輸入 <xref:System.Runtime.Intrinsics.X86.Sse2> 傳回不正確的結果。</span><span class="sxs-lookup"><span data-stu-id="26e6c-104">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="26e6c-105">結果也不會與類別中對應方法所產生的結果有差異 <xref:System.Runtime.Intrinsics.X86.Avx> 。</span><span class="sxs-lookup"><span data-stu-id="26e6c-105">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="26e6c-106">從 .NET 5.0 開始，這些方法會正確地處理 `NaN` 輸入，並傳回與類別中對應方法相同的結果 <xref:System.Runtime.Intrinsics.X86.Avx> 。</span><span class="sxs-lookup"><span data-stu-id="26e6c-106">Starting in .NET 5.0, these methods correctly handle `NaN` inputs and return the same results as the corresponding methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="26e6c-107">Streaming SIMD Extensions （SSE）和 Streaming SIMD Extensions 2 （SSE2）產業標準架構（ISAs）不會針對這些比較方法提供直接硬體支援，因此它們會實作為軟體。</span><span class="sxs-lookup"><span data-stu-id="26e6c-107">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="26e6c-108">之前，方法未正確地執行，而且不正確地處理 `NaN` 輸入。</span><span class="sxs-lookup"><span data-stu-id="26e6c-108">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="26e6c-109">如果是從原生移植的程式碼，則不正確的行為可能會導致錯誤。</span><span class="sxs-lookup"><span data-stu-id="26e6c-109">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="26e6c-110">若是256位程式碼路徑，方法也可以在類別中對等的方法產生不同的結果 <xref:System.Runtime.Intrinsics.X86.Avx> 。</span><span class="sxs-lookup"><span data-stu-id="26e6c-110">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="26e6c-111">如需方法先前不正確的範例，您可以將當做 `CompareNotGreaterThan(x,y)` `CompareLessThanOrEqual(x,y)` 一般整數來執行。</span><span class="sxs-lookup"><span data-stu-id="26e6c-111">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="26e6c-112">不過，針對 `NaN` 輸入，該邏輯會計算錯誤的結果。</span><span class="sxs-lookup"><span data-stu-id="26e6c-112">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="26e6c-113">相反地，使用 `CompareNotLessThan(y,x)` 會正確比較數位 *，並*將 `NaN` 輸入納入考慮。</span><span class="sxs-lookup"><span data-stu-id="26e6c-113">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>
