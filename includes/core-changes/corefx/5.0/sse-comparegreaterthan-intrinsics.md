### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a>SSE 和 SSE2 CompareGreaterThan 方法會正確處理 NaN 輸入

<xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> 已修正下列和方法來適當地處理 `NaN` 輸入，並符合類別中對等方法的硬體行為 <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> ：

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`
* `CompareScalarGreaterThanOrEqual`
* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a>變更描述

先前所 `NaN` 列 <xref:System.Runtime.Intrinsics.X86.Sse> 和方法的輸入 <xref:System.Runtime.Intrinsics.X86.Sse2> 傳回不正確的結果。 結果也與類別中的對應方法所產生的結果不相同 <xref:System.Runtime.Intrinsics.X86.Avx> 。

從 .NET 5.0 開始，這些方法會正確處理 `NaN` 輸入，並傳回與類別中的對應方法相同的結果 <xref:System.Runtime.Intrinsics.X86.Avx> 。

Streaming SIMD Extensions (SSE) 和 Streaming SIMD Extensions 2 (SSE2) 業界標準架構 (Isa) 不會為這些比較方法提供直接硬體支援，所以會在軟體中執行。 先前，方法未正確地完成，而且它們不正確地處理 `NaN` 輸入。 針對從原生移植的程式碼，不正確的行為可能會導致 bug。 若為256位的程式碼路徑，方法也可以在類別中對等的方法產生不同的結果 <xref:System.Runtime.Intrinsics.X86.Avx> 。

如需方法先前不正確的範例，您可以 `CompareNotGreaterThan(x,y)` `CompareLessThanOrEqual(x,y)` 針對一般整數執行 as。 不過，對於 `NaN` 輸入，該邏輯會計算錯誤的結果。 相反地，使用 `CompareNotLessThan(y,x)` 會正確地比較數位 *，並* 將 `NaN` 輸入列入考慮。

#### <a name="version-introduced"></a>引進的版本

5.0 Preview 4

#### <a name="recommended-action"></a>建議的動作

- 如果先前的行為是錯誤，則不需要進行任何變更。

- 如果需要先前的行為，您可以藉由變更相關的調用來維持該行為，如下所示：

  * `CompareGreaterThan(x,y)` -> `CompareNotLessThanOrEqual(x,y)`
  * `CompareGreaterThanOrEqual(x,y)` -> `CompareNotLessThan(x,y)`
  * `CompareNotGreaterThan(x,y)` -> `CompareLessThanOrEqual(x,y)`
  * `CompareNotGreaterThanOrEqual(x,y)` -> `CompareLessThan(x,y)`
  * `CompareScalarGreaterThan(x,y)` -> `CompareScalarNotLessThanOrEqual(x,y)`
  * `CompareScalarGreaterThanOrEqual(x,y)` -> `CompareScalarNotLessThan(x,y)`
  * `CompareScalarNotGreaterThan(x,y)` -> `CompareScalarLessThanOrEqual(x,y)`
  * `CompareScalarNotGreaterThanOrEqual(x,y)` -> `CompareScalarLessThan(x,y)`
