### <a name="sse-and-sse2-comparegreaterthan-methods-properly-handle-nan-inputs"></a><span data-ttu-id="b0654-101">SSE 和 SSE2 CompareGreaterThan 方法會正確處理 NaN 輸入</span><span class="sxs-lookup"><span data-stu-id="b0654-101">SSE and SSE2 CompareGreaterThan methods properly handle NaN inputs</span></span>

<span data-ttu-id="b0654-102"><xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> 已修正下列和方法來適當地處理 `NaN` 輸入，並符合類別中對等方法的硬體行為 <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> ：</span><span class="sxs-lookup"><span data-stu-id="b0654-102">The following <xref:System.Runtime.Intrinsics.X86.Sse?displayProperty=nameWithType> and <xref:System.Runtime.Intrinsics.X86.Sse2?displayProperty=nameWithType> methods have been fixed to properly handle `NaN` inputs and match the hardware behavior of the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx?displayProperty=nameWithType> class:</span></span>

* `CompareGreaterThan`
* `CompareGreaterThanOrEqual`
* `CompareNotGreaterThan`
* `CompareNotGreaterThanOrEqual`
* `CompareScalarGreaterThan`

* `CompareScalarNotGreaterThan`
* `CompareScalarNotGreaterThanOrEqual`

#### <a name="change-description"></a><span data-ttu-id="b0654-103">變更描述</span><span class="sxs-lookup"><span data-stu-id="b0654-103">Change description</span></span>

<span data-ttu-id="b0654-104">先前所 `NaN` 列 <xref:System.Runtime.Intrinsics.X86.Sse> 和方法的輸入 <xref:System.Runtime.Intrinsics.X86.Sse2> 傳回不正確的結果。</span><span class="sxs-lookup"><span data-stu-id="b0654-104">Previously, `NaN` inputs to the listed <xref:System.Runtime.Intrinsics.X86.Sse> and <xref:System.Runtime.Intrinsics.X86.Sse2> methods returned an incorrect result.</span></span> <span data-ttu-id="b0654-105">結果也與類別中的對應方法所產生的結果不相同 <xref:System.Runtime.Intrinsics.X86.Avx> 。</span><span class="sxs-lookup"><span data-stu-id="b0654-105">The result also differed from the result generated by the corresponding method in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="b0654-107">Streaming SIMD Extensions (SSE) 和 Streaming SIMD Extensions 2 (SSE2) 業界標準架構 (Isa) 不會為這些比較方法提供直接硬體支援，所以會在軟體中執行。</span><span class="sxs-lookup"><span data-stu-id="b0654-107">The Streaming SIMD Extensions (SSE) and Streaming SIMD Extensions 2 (SSE2) industry standard architectures (ISAs) don't provide direct hardware support for these comparison methods, so they're implemented in software.</span></span> <span data-ttu-id="b0654-108">先前，方法未正確地完成，而且它們不正確地處理 `NaN` 輸入。</span><span class="sxs-lookup"><span data-stu-id="b0654-108">Previously, the methods were improperly implemented, and they incorrectly handled `NaN` inputs.</span></span> <span data-ttu-id="b0654-109">針對從原生移植的程式碼，不正確的行為可能會導致 bug。</span><span class="sxs-lookup"><span data-stu-id="b0654-109">For code ported from native, the incorrect behavior may introduce bugs.</span></span> <span data-ttu-id="b0654-110">若為256位的程式碼路徑，方法也可以在類別中對等的方法產生不同的結果 <xref:System.Runtime.Intrinsics.X86.Avx> 。</span><span class="sxs-lookup"><span data-stu-id="b0654-110">For a 256-bit code path, the methods can also produce different results to the equivalent methods in the <xref:System.Runtime.Intrinsics.X86.Avx> class.</span></span>

<span data-ttu-id="b0654-111">如需方法先前不正確的範例，您可以 `CompareNotGreaterThan(x,y)` `CompareLessThanOrEqual(x,y)` 針對一般整數執行 as。</span><span class="sxs-lookup"><span data-stu-id="b0654-111">As an example of how the methods were previously incorrect, you can implement `CompareNotGreaterThan(x,y)` as `CompareLessThanOrEqual(x,y)` for regular integers.</span></span> <span data-ttu-id="b0654-112">不過，對於 `NaN` 輸入，該邏輯會計算錯誤的結果。</span><span class="sxs-lookup"><span data-stu-id="b0654-112">However, for `NaN` inputs, that logic computes the wrong result.</span></span> <span data-ttu-id="b0654-113">相反地，使用 `CompareNotLessThan(y,x)` 會正確地比較數位 *，並* 將 `NaN` 輸入列入考慮。</span><span class="sxs-lookup"><span data-stu-id="b0654-113">Instead, using `CompareNotLessThan(y,x)` compares the numbers correctly *and* takes `NaN` inputs into consideration.</span></span>
